# Claude Code 进阶 · 5月16日

## 今日主题：复杂工作流编排 —— 多文件批处理与跨项目管道

当任务涉及"读10个文件 → 分析 → 生成5个新文件"或跨越多个项目目录时，逐条发指令效率太低。Claude Code 支持**脚本管道 + 会话复用 + 批量文件操作**三种编排方式，今天学会把零散指令组合成自动化工作流。

核心思路：Claude Code 的 `/compact`（压缩上下文）、`--resume`（恢复会话）配合 shell 管道，能把 20 步的手动操作压缩成一条命令。

## 操作步骤

### 第1步：用 `/compact` 管理长会话上下文

Claude Code 的上下文窗口有限。当你在一个会话中完成了多步操作，继续前先压缩：

```
# 在 Claude Code 对话中输入
/compact
```

效果：Claude Code 会丢弃非关键对话历史，仅保留当前状态和未完成任务。**不会丢失文件变更和工作进度**。

> 规律：每完成一个阶段（例如每处理完3-5个文件），执行一次 `/compact`。尤其在批量操作中途一定要用，避免上下文膨胀后模型开始"遗忘"之前的指令。

### 第2步：用 `--resume` 跨天继续工作

大型工作流（比如批量萃取100篇公众号文章）需要跨多个会话完成：

```bash
# 第一次会话：处理前50篇
claude -p "处理收件箱/信号/2026-05-16/*.md，提取实体和概念" --resume

# 第二天继续：后50篇（--resume 会自动加载上次会话状态）
claude -p "继续处理收件箱/信号/2026-05-16/下一批" --resume
```

`--resume` 会从上一次会话的断点继续，保留已完成文件的状态，只处理未完成的。

**进阶技巧**：结合文件名标记法实现手动断点恢复
```bash
# 在批量操作中，每完成一个文件，写入一个标记
echo "_processed/2026-05-16_article03.md" >> /tmp/batch_checkpoint.txt

# 下次启动时读取 checkpoint
claude -p "加载/tmp/batch_checkpoint.txt，跳过已处理文件，继续处理剩余的"
```

### 第3步：Shell 管道 + Claude Code 批处理

**模式A：逐文件管道**（适合每个文件需要独立分析）

```bash
# 批量萃取Obsidian笔记中的实体（逐个文件处理）
for f in ~/wiki/LLM\ Wiki知识应用/企业出海/信号/*.md; do
  echo "=== 处理: $f ==="
  claude -p "阅读文件 $f，提取其中的公司名、人物名、关键事件，以表格输出" \
    --no-print-rules \
    --allowedTools "Read,Edit" \
    2>&1 | tail -20
done
```

**模式B：聚合式管道**（适合所有文件统一分析）

```bash
# 一次性分析本周所有信号，输出趋势报告
claude -p "
阅读以下全部文件：
$(ls ~/wiki/LLM\ Wiki知识应用/企业出海/信号/2026-05-1*.md)

分析任务：
1. 找出本周出现的重复主题和趋势
2. 哪些公司被提到最多
3. 输出一份200字以内的周报摘要
" \
  --output-format markdown > ~/wiki/周报/信号趋势-$(date +%F).md
```

**模式C：生成 + 修改两步管道**（最常用模式）

```bash
# 第一步：Claude Code 生成批量文件（课程大纲模板）
claude -p "
根据R.I.S.E.课程体系模板，为以下5个主题各生成一个课程大纲文件：
- 跨文化谈判技巧
- 国际商务邮件写作
- 海外项目汇报演讲
- 跨文化冲突管理
- 全球团队领导力

每个文件保存到 ~/projects/课程体系-RISE/大纲/ 目录，命名格式：序号-主题.md
大纲包含：课程目标、模块划分（4-6个模块）、每个模块的3个学习目标。
"

# 第二步：不另开会话，直接继续
/compact

# 然后执行修改
claude -p "
对 ~/projects/课程体系-RISE/大纲/ 下所有文件做统一修改：
1. 每个大纲末尾添加\"评估方式\"一节（笔试/案例/实操三选一）
2. 在所有文件开头添加统一的YAML frontmatter（tags, type, date）
3. 文件名统一改成英文 slug 格式（如 01-cross-cultural-negotiation.md）
"
```

### 第5步：深度模式 —— 环境变量 + Claude Code 脚本管道

```bash
# 定义一个可以重复调用的 shell 函数
cc_batch() {
  local dir="$1"
  local prompt="$2"
  local output="$3"
  
  claude -p "
当前目录: $dir
任务: $prompt
把所有结果保存到: $output
请列出你处理了哪些文件。
" --print-raw
}

# 使用：批量萃取Obsidian笔记
cc_batch \
  "~/wiki/LLM Wiki知识应用/企业出海/行业报告" \
  "阅读所有Markdown文件，为每个文件生成一个摘要卡片（100字内）保存到同目录下 *.summary.md" \
  "自动保存"
```

## 注意事项

1. **`/compact` 不是无损操作**：它会丢弃对话历史，如果后续需要回看中间结果，压缩前先用 `--output-format markdown > checkpoint.md` 保存输出日志
2. **`--resume` 的局限**：它恢复的是文件状态和会话上下文，但不恢复终端状态（环境变量、当前目录等）。跨会话工作流建议在每个循环中加 `pwd` 和 `ls` 验证当前目录
3. **批量操作超时**：Claude Code 每次调用有约5分钟的超时限制。处理超过20个文件时，建议分成3-5批，每批之间用 `/compact` 压缩
4. **管道安全**：`claude -p "指令"` 模式中的指令会被直接执行。不要在 `-p` 参数中使用未过滤的用户输入或变量，可能引发 prompt injection
5. **`--allowedTools` 限制**：在管道模式中限制工具集（如 `--allowedTools "Read,Edit"`）可以防止 Claude Code 意外执行网络请求或系统命令
6. **文件路径中的空格**：shell 管道中处理文件名时，始终用双引号包裹变量（`"$f"`），尤其是 Obsidian 笔记中常含中文和特殊字符
7. **模型选择影响**：复杂工作流（多文件交叉分析、跨项目操作）建议用 Claude Opus 模型；大批量简单格式化任务用 Sonnet 即可。在 `.claude/CLAUDE.md` 中设置 `model:` 字段针对性选择

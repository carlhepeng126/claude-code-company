# Claude Code 进阶 · 5月15日

## 今日主题：MCP 集成实战 —— 让 Claude Code 操作你的工具箱

你已经用 Claude Code 处理本地文件，但它的真正威力在于通过 **MCP（Model Context Protocol）** 连接外部工具。Claude Code 原生支持 MCP 服务器，像 GBrain 知识图谱、飞书文档、Linear 项目管理、Airtable 数据库等，都可以通过 MCP 直接在对话中被读写操作。今天学会配置和调试 MCP，让 Claude Code 接入你已有的工具生态。

核心原理：MCP 是一个开放协议，Claude Code 通过 stdio 或 HTTP 与 MCP Server 通信。你只需要在 `~/.claude/settings.json` 中注册 Server，Claude Code 启动时自动加载其工具列表。

## 操作步骤

### 1. 查看当前已配置的 MCP 服务器

```bash
# Claude Code 的 MCP 配置文件
cat ~/.claude/settings.json 2>/dev/null || echo "文件不存在"
```

如果你一直在用 Hermes Agent 的 MCP（如 gbrain 相关工具），这个文件可能已有内容。验证加载状态：

```bash
# 用 Claude Code 查看已注册的工具（非交互模式）
# 启动 Claude Code 后输入 /tools 即可看到所有可用工具
echo "执行后手动验证：进入 Claude Code 后输入 /tools 查看工具列表"
```

### 2. 注册一个新的 MCP Server —— 以飞书文档为例

```bash
# 编辑 Claude Code 的 MCP 配置
cat >> ~/.claude/settings.json << 'MCPCONFIG'
{
  "mcpServers": {
    "feishu-docs": {
      "command": "python3",
      "args": ["/path/to/feishu-mcp-server/index.py"],
      "env": {
        "FEISHU_APP_ID": "cli_xxxxxxxxxxxxx",
        "FEISHU_APP_SECRET": "xxxxxxxxxxxxxxxxxxxxxxxxxx"
      },
      "disabled": false,
      "autoApprove": []
    },
    "linear": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-linear"],
      "env": {
        "LINEAR_API_KEY": "lin_api_xxxxxxxxxxxxxxxxxxxx"
      },
      "disabled": false,
      "autoApprove": []
    },
    "obsidian": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-obsidian"],
      "env": {
        "OBSIDIAN_API_TOKEN": "your_obsidian_local_rest_api_token",
        "OBSIDIAN_VAULT_PATH": "/root/wiki"
      },
      "disabled": false,
      "autoApprove": []
    }
  }
}
MCPCONFIG
```

> **注意：** 上面的配置只是模板。如果你没有对应的 API Key，不要直接复制——**先确认工具是否可用再配置**。

### 3. 本地 MCP Server 开发 —— 自定义你自己的工具

如果需要 Claude Code 操作自定义脚本或内部系统，可以快速搭建一个本地 MCP Server：

```bash
# 最简单的 stdio MCP Server 示例 (Python)
mkdir -p ~/mcp-servers/custom-reports
cat > ~/mcp-servers/custom-reports/server.py << 'PYTHON'
import sys, json

def handle_request(request):
    method = request.get("method", "")
    
    if method == "initialize":
        # 返回服务器信息
        return {
            "protocolVersion": "2024-11-05",
            "capabilities": {
                "tools": {
                    "listChanged": False
                }
            },
            "serverInfo": {
                "name": "custom-reports",
                "version": "1.0.0"
            }
        }
    
    elif method == "tools/list":
        return {
            "tools": [
                {
                    "name": "generate_weekly_report",
                    "description": "从日日志文件生成周报 Markdown",
                    "inputSchema": {
                        "type": "object",
                        "properties": {
                            "week_number": {
                                "type": "integer",
                                "description": "ISO 周数，默认当前周"
                            },
                            "output_path": {
                                "type": "string",
                                "description": "输出路径，默认 ~/wiki/周报/"
                            }
                        }
                    }
                },
                {
                    "name": "search_customer_projects",
                    "description": "按关键字搜索客户项目文档",
                    "inputSchema": {
                        "type": "object",
                        "properties": {
                            "keyword": {"type": "string", "description": "搜索关键词"},
                            "limit": {"type": "integer", "description": "返回条数上限"}
                        },
                        "required": ["keyword"]
                    }
                }
            ]
        }
    
    elif method == "tools/call":
        tool_name = request.get("params", {}).get("name", "")
        args = request.get("params", {}).get("arguments", {})
        
        if tool_name == "generate_weekly_report":
            week = args.get("week_number", "current")
            output = f"# 第 {week} 周工作周报\n\n(周报内容生成逻辑...)\n"
            return {"content": [{"type": "text", "text": output}]}
        
        elif tool_name == "search_customer_projects":
            keyword = args.get("keyword", "")
            limit = args.get("limit", 10)
            return {
                "content": [{
                    "type": "text",
                    "text": f"搜索 '{keyword}' 找到相关项目文档 {limit} 条 (示例返回)"
                }]
            }
    
    return {"error": {"code": -32601, "message": f"Method not found: {method}"}}

# JSON-RPC over stdio
for line in sys.stdin:
    try:
        request = json.loads(line.strip())
        response = handle_request(request)
        response["id"] = request.get("id")
        print(json.dumps(response), flush=True)
    except Exception as e:
        print(json.dumps({"error": {"message": str(e)}, "id": None}), flush=True)
PYTHON

# 注册到 Claude Code
cat >> ~/.claude/settings.json << 'MCPADD'
,    "custom-reports": {
      "command": "python3",
      "args": ["/root/mcp-servers/custom-reports/server.py"],
      "disabled": false,
      "autoApprove": ["generate_weekly_report", "search_customer_projects"]
    }
MCPADD
```

## 今日练习 (5分钟)

检查你当前的 MCP 配置状态，并添加一个可操作的自定义 MCP Server：

```bash
# 1. 查看当前是否已有 MCP 配置
cat ~/.claude/settings.json 2>/dev/null

# 2. 检查是否存在 ~/.claude/ 目录，不存在则创建

# 3. 创建一个最简单的"项目文档查询"MCP Server
cat > ~/mcp-servers/project-query/server.py << 'PYTHON'
import sys, json, os, glob
from pathlib import Path

def handle(req):
    if req.get("method") == "initialize":
        return {"protocolVersion":"2024-11-05","capabilities":{"tools":{"listChanged":False}},"serverInfo":{"name":"project-query","version":"1.0.0"}}
    if req.get("method") == "tools/list":
        return {"tools":[
            {"name":"find_project_docs","description":"搜索项目文档","inputSchema":{"type":"object","properties":{"keyword":{"type":"string"},"path":{"type":"string","description":"搜索根目录，默认 ~/wiki/ACT笔记-深度思考"}},"required":["keyword"]}}
        ]}
    if req.get("method") == "tools/call" and req.get("params",{}).get("name") == "find_project_docs":
        args = req.get("params",{}).get("arguments",{})
        keyword = args.get("keyword","")
        search_path = args.get("path", os.path.expanduser("~/wiki/ACT笔记-深度思考"))
        results = []
        for f in Path(search_path).rglob("*.md"):
            if keyword.lower() in f.read_text(encoding="utf-8", errors="ignore").lower():
                results.append(str(f))
        return {"content":[{"type":"text","text":f"找到 {len(results)} 个文档:\n" + "\n".join(results[:10])}]}
    return {"error":{"code":-32601,"message":"unknown method"}}

for line in sys.stdin:
    try:
        r = handle(json.loads(line.strip()))
        r["id"] = json.loads(line.strip()).get("id")
        print(json.dumps(r), flush=True)
    except: pass
PYTHON

# 4. 验证 MCP Server 能正常响应
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{}}' | python3 ~/mcp-servers/project-query/server.py
```

**完成了？** 你已经亲手写了一个 MCP Server！下次启动 Claude Code 时，可以直接说 *"查带'当升科技'的项目文档"*——Claude Code 会自动调用你的 MCP 做全文搜索。

## 注意事项

- **autoApprove 白名单：** `autoApprove: ["tool_name"]` 可以让某些工具执行时无需你确认。只对低风险工具（查、读）开启，写/删操作不要 autoApprove。
- **MCP Server 的生命周期：** Claude Code 启动时启动所有 MCP Server，退出时关闭。长时间运行建议使用 HTTP 模式的 MCP Server（而非 stdio），减少进程重启开销。
- **环境变量泄露风险：** `env` 字段中的 API Key 会暴露在 `settings.json` 文件里。考虑用 `.env` 文件 + dotenv 加载，或设置文件权限 `chmod 600 ~/.claude/settings.json`。
- **NPM 依赖：** 使用 `npx` 启动的 MCP Server（如 `@anthropic/mcp-linear`）需要 Node.js ≥ 18。确保环境中有 Node 运行时。
- **JSON 格式错误：** 如果 `settings.json` 不是合法 JSON，Claude Code 静默忽略整个 MCP 配置段，不会报错。修改后一定要 `python3 -c "import json; json.load(open('...'))"` 验证。
- **工具数量上限：** 单个 Claude Code 会话建议不超过 10 个 MCP Server、50 个工具。过多工具会稀释 Claude Code 对每个工具的注意力，导致它总选错工具。
- **与Hermes Agent 的关系：** 你在 Hermes 中用的 gbrain MCP 工具是 Hermes 内置的，不自动映射到 Claude Code。要在 Claude Code 中也用 gbrain，需要在 Claude Code 的 settings.json 中单独注册 gbrain MCP Server。

# Claude Code 进阶 · 5月17日

## 今日主题：Obsidian 深度集成 —— Claude Code 批处理你的知识库

你知道 Claude Code 能读写 Obsidian 笔记，但它的真正威力在于**批量处理整个 Vault**——用一条自然语言指令完成过去要写 Python 脚本才能做的事。今天学会用 Claude Code 做 Obsidian 知识库的批量操作，把零散的笔记管理提升到自动化级别。

核心原理：Claude Code 在 Obsidian Vault 目录下运行时，能通过自然语言直接操作任意 `.md` 文件、读取 YAML frontmatter、批量搜索替换、生成索引、修复链接——它就是最强的 Obsidian 批处理引擎。

## 操作步骤

### 1. 批量修复 Frontmatter（统一标签体系）

假设你的 Vault 里 tags 写法五花八门，想统一为小写连字符格式：

```bash
# 在 Vault 根目录运行 Claude Code，让它批量修复
cd ~/wiki

# 会话中下达指令：
# "扫描 LLM Wiki知识应用/entities/ 下所有 md 文件，
#  把 tags 字段中所有中文逗号替换为英文逗号，
#  所有空格替换为连字符，全部转为小写，
#  遇到没有 tags 字段的文件自动加上 tags: [待分类]"
```

### 2. 批量创建索引卡片

给某个子文件夹生成统一的索引页：

```bash
# 在 Claude Code 中：
# "读取 LLM Wiki知识应用/方法工具箱/ 下所有 md 文件，
#  为每个文件提取 title、tags、前 2 行描述，
#  生成一份 README.md 索引页，格式为表格：
#  | 文件 | 标签 | 简介 |
#  不要修改原始文件"
```

### 3. 跨文件夹链接修复

Obsidian 重构后经常出现断链。让 Claude Code 做全库扫描：

```bash
# "扫描 Vault 中所有 [[wikilink]] 引用，
#  检查每个链接的目标文件是否存在，
#  列出所有断开的链接（文件路径 → 缺失的链接目标），
#  对每个断链搜索最匹配的现有文件，给出替换建议"
```

### 4. 批量自动补标签（基于内容语义推断）

```bash
# "遍历 1-收件箱/ 下所有未打标签的 md 文件，
#  读取每篇的前 200 字，
#  根据内容推断 2-3 个合适标签，
#  追加到 frontmatter 的 tags 数组，
#  如果文件没有 frontmatter，自动创建 title + tags"
```

### 5. 批量为文件添加「已读/待处理」状态

```bash
# "遍历 1-收件箱/灵感记/ 下所有 md 文件，
#  检查 frontmatter 是否有 status 字段，
#  如果没有，添加 status: pending
#  如果文件 content 包含 '## 行动'，自动改为 status: done"
```

## 在Carl工作中的应用

**场景：每周日收件箱大扫除**

你的 Obsidian Vault 中有 `1-收件箱/` 文件夹，每日积累大量笔记——灵感记、客户对话记录、公众号素材、培训观察。手动分类、打标签、链接到知识库至少 30 分钟。现在可以一条指令完成：

```bash
cd ~/wiki
claude -p "
1. 读取 1-收件箱/ 下本周（5月10-16日）新增的所有 md 文件
2. 对每个文件：
   a. 如果无 frontmatter，自动添加 title（取第一行标题）、tags（推断 2-3 个）、status: processed
   b. 如果内容中包含人名+公司名，检查 entities/ 下是否有对应的索引卡，如有则添加 [[wikilink]]
   c. 如果内容涉及方法/框架，检查 concepts/ 下是否有对应概念页
3. 生成一份处理报告：处理了多少文件、新建了多少链接、归类到哪些标签
"
```

**另一个场景：项目交付后知识沉淀**

一个培训项目交付完，客户案例要入库：

```bash
claude -p "
1. 读取 ACT笔记-深度思考/3-Action/ 下所有与'中交培训'相关的 md 文件
2. 提取关键信息：客户名称、项目时间、培训内容、交付成果
3. 按模板格式生成一份 case study 文件，存入 LLM Wiki知识应用/entities/cases-中交培训.md
4. 在原始 ACT 文件中添加一行：📎 知识库关联: [[cases-中交培训]]
"
```

## 今日练习 (5分钟)

运行这个练习，体验 Claude Code 批量操作 Obsidian Vault 的威力：

```bash
cd ~/wiki
claude -p "
1. 列出 1-收件箱/ 下最近 7 天内修改过的所有 md 文件
2. 检查每个文件是否有 frontmatter（--- 分隔的 YAML 头）
3. 输出一个表格：文件名 | 有/无frontmatter | 文件大小 | 标签数
4. 如果有文件缺少 frontmatter，自动补上最基础的：title（从第一行#标题提取）、date（文件修改日期）、tags: [待处理]
"
```

这个练习让你立刻看到 Vault 中哪些笔记缺少结构化元数据，并自动修复——5 分钟完成过去需要写 Python 脚本才能做的事。

## 注意事项

1. **先备份再批量操作**——Claude Code 的批量操作是**直接写文件**，不可撤销。强烈建议先 git commit 或运行 `claude -p "列出所有要修改的文件，不执行任何写入"` 做 dry-run
2. **分步执行**——不要一次下达太多指令。先让 Claude Code 列出文件清单，确认无误后再执行修改
3. **注意文件编码**——Obsidian 默认 UTF-8，Claude Code 也使用 UTF-8，但如果遇到 GBK/GB2312 文件（如某些老素材），会乱码
4. **CLAUDE.md 影响范围**——如果 Vault 根目录有 CLAUDE.md，Claude Code 读取后可能改变行为模式。操作前确认 CLAUDE.md 是你要的上下文
5. **限制单次文件数**——一次批量处理超过 50 个文件时，Claude Code 可能因上下文限制而遗漏。分批处理：`1-收件箱/a-m/` 和 `1-收件箱/n-z/`
6. **`claude -p` 模式 vs 交互模式**——用 `-p`（单指令模式）适合明确的一次性批量操作；复杂的多次迭代任务用交互模式（不加 `-p`），可以在对话中逐步确认

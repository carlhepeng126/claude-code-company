# Claude Code 进阶 · 5月19日

## 今日主题：多Agent协作模式 —— Claude Code + Hermes Agent 双引擎工作流

你的AI工具不止一个——Hermes Agent 负责常驻后台执行定时任务（情报采集、日报生成、知识库维护），Claude Code 负责本地深度编码和交互式开发。但它们是两个孤岛，数据和指令需要通过你手动搬运。今天学会搭建**双Agent协作管道**：让 Hermes 调度、Claude Code 执行，形成自动化的「管家+技师」配合模式。

核心思路：Hermes Agent 擅长「管家」角色——定时触发、多工具编排、后台驻留；Claude Code 擅长「技师」角色——本地文件操作、代码重构、复杂自然语言指令。通过 Shell 脚本 + 文件事件驱动，让它们各司其职。

## 操作步骤

### 第1步：Hermes 调用 Claude Code 执行脚本任务

Hermes 的 `execute_code` 或 `terminal` 工具可以直接启动 Claude Code 进程并等待结果。这是最直接的协作模式：

```bash
# 📌 Hermes 侧的 cron 任务脚本示例
# 路径示例：~/.hermes/scripts/claude-batch.sh

#!/bin/bash
# 用途：Hermes 定时唤醒 Claude Code 执行批处理，获取结构化输出
TASK_DIR="/tmp/claude-tasks/$(date +%Y%m%d)"
mkdir -p "$TASK_DIR"

# 将任务描述写入文件（Hermes 可动态生成）
cat > "$TASK_DIR/task.md" << 'TASK'
请执行以下操作：
1. 扫描 ~/wiki/LLM Wiki知识应用/entities/ 下本周修改过的 md 文件
2. 统计每个文件的 tags 分布
3. 生成一份 tags 统计算表，保存到 /tmp/claude-tasks/tags-report.md
请输出 Done 表示完成
TASK

# 调用 Claude Code 以非交互模式执行
cd ~/wiki
claude -p "$(cat $TASK_DIR/task.md)" --print > "$TASK_DIR/output.log" 2>&1

echo "Claude Code 执行完毕，结果：$TASK_DIR/output.log"
```

### 第2步：Claude Code 通过文件事件触发 Hermes

反过来，Claude Code 完成深度工作后，通过写入「信号文件」通知 Hermes 接管后续操作：

```bash
# 📌 在 Claude Code 会话中执行的指令：
# "完成项目文件重构后，将变更摘要写入信号文件，供 Hermes 后续处理"

# Claude Code 在完成文件操作后执行：
cat > /tmp/signals/claude-done-$(date +%s).json << 'EOF'
{
  "agent": "claude-code",
  "task": "obsidian-tag-restructure",
  "changed_files": 12,
  "summary": "统一了 entities/ 下所有文件的 tags 格式",
  "next_action": "hermes: 请同步更新 GBrain 标签索引并记录操作日志"
}
EOF
```

Hermes 侧配置一个 crontab 或 watch 循环定期扫描 `/tmp/signals/` 目录，发现新信号文件即触发后续处理。

### 第3步：双Agent管道实战 —— 自动化简报生成管线

这是最实用的场景——将企业出海日报的生成流程拆解为双Agent协作：

## 在Carl工作中的应用

**场景：情报采集 → 深度分析 → 知识入库 全自动化**

你每天的工作流中有一个典型痛点：Hermes 早上 8:00 跑完情报日报，但你还需要在 Claude Code 里手工分析、输出到知识库。现在打通之后：

1. **Hermes（定时任务）** → 早 7:50 扫描 RSS/公众号/竞品信号 → 写入信号文件
2. **Claude Code（深度分析）** → 读取信号 → 按 B016 方案框架分析 → 生成结构化的竞争情报卡
3. **Hermes（落地归档）** → 检测新文件 → 更新 GBrain → 推送摘要到飞书

一次配置，每天自动执行。你不需在 8:00 到 8:15 之间做任何操作。

## 今日练习 (5分钟)

创建一个最小的双Agent协作管道：

```bash
# 1. 创建信号目录
mkdir -p /tmp/signals

# 2. 编写一个 Hermes 可调用的 Claude Code 脚本
cat > /tmp/test-claude-pipe.sh << 'SCRIPT'
#!/bin/bash
cd ~/wiki
claude -p "列出 ACT笔记-深度思考/3-Action/ 下所有文件名和修改时间" --print
SCRIPT
chmod +x /tmp/test-claude-pipe.sh

# 3. 执行一次（确认 claude CLI 可用）
bash /tmp/test-claude-pipe.sh
```

如果输出正确显示了 Action 文件夹中的文件列表，说明双Agent管道已就绪。下一步就可以把它接入 Hermes 的 crontab。

## 注意事项

- **claude --print 参数**：非交互模式必须加 `--print`，否则 Claude Code 进入交互模式挂起等待输入
- **上下文隔离**：Claude Code 被 Hermes 调用时是一个全新会话，没有历史上下文。任务指令需**自包含**——把所有上下文写进 `-p` 参数或提供的文件中
- **成本控制**：claude CLI 每次调用消耗配额（按token计费）。高频轮询（低于5分钟间隔）可能导致限额快速耗尽。建议 Cron 任务间隔 ≥ 1 小时
- **错误处理**：Claude Code 执行失败时返回非零退出码。Hermes 脚本中应检查 `$?`，失败时写错误信号文件而非继续下游流程
- **文件锁竞争**：Claude Code 和 Hermes 同时写同一个文件会导致内容覆盖。约定唯一文件名（带时间戳）避免冲突

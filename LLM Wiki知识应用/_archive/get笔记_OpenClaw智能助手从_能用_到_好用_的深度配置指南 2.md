### **📌 引言：从功能实现到个性化体验的跨越**

用户初期通过官方文档成功搭建OpenClaw新闻推送机器人，实现定时新闻获取与推送功能，但使用中发现**个性化不足**问题：推送内容与用户偏好匹配度低、操作习惯不符合预期。核心原因在于**仅打通Agent流程，未建立完善的规则体系**（缺乏工作手册、岗位说明和记忆系统），需通过深入理解OpenClaw运行原理优化配置。

### **🧠 OpenClaw的四层记忆架构**

#### **核心架构组成**

OpenClaw通过四层记忆系统实现智能决策与个性化服务：
1. **SOUL（灵魂）**：智能体核心原则（如"不编造信息"、"不传播假新闻"）
2. **TOOLS（工具）**：可调用能力集（如搜索、邮件发送）
3. **USER（用户记忆）**：存储用户偏好、习惯、背景信息
4. **Session**：当前对话的临时上下文

#### **关键配置文件说明**

| 文件 | 作用 | 重要性 |
| :--- | :--- | :--- |
| **SOUL.md** | 定义核心行为准则与红线（如"不访问付费墙内容"） | 必需 |
| **USER.md** | 构建用户画像（如"安静而热烈的技术人"） | 推荐 |
| **MEMORY.md** | 长期记忆主索引，从每日日志提炼结构化知识 | 动态生成 |
| **IDENTITY.md** | 设定Agent身份标签与角色定位 | 多Agent场景必需 |
| **Agent.md** | 工作流程手册，定义任务执行规范 | 必需 |
| **HEARTBEAT.md** | 周期性任务触发机制（如每小时新闻搜索） | 定时任务需配置 |

#### **四层协同工作原理（以新闻获取为例）**

用户提问"今天AI领域有什么大新闻？"的处理流程：
1. **Session层**：接收消息并记录"今天"、"AI领域"等临时上下文
2. **Agent.md启动**：加载预设工作流程规范
3. **多文件协同读取**：
   - SOUL.md注入原则（客观真实、附带来源）
   - IDENTITY.md确认身份与语气
   - USER.md获取用户偏好（关注OpenAI/Google、晨间推送习惯）
   - MEMORY.md加载长期记忆索引
4. **工具调用**：使用`web_search`工具搜索"AI news today"并抓取详情
5. **内容处理**：依据SOUL原则和USER偏好筛选整理，生成符合格式的回复
6. **记忆更新**：将对话关键点写入当日日志（`memory/YYYY-MM-DD.md`），若涉及新偏好则同步更新MEMORY.md

### **🛠️ 新闻获取Agent创建实践**

#### **Step1：SOUL.md核心原则设定**

定义新闻助手的身份职责与行为红线：
```markdown
# SOUL.md - 新闻助手的原则设定  
## 身份与职责  
你是一个专业的新闻聚合助手，核心职责是提供准确、及时的新闻信息。  
## 行为准则  
1. 及时准确：先核验再推送；重要信息必须给来源与时间戳  
2. 覆盖全面但不详：宁少勿滥；默认只推"对你有用"的信息  
3. 可追溯：每条信息给到原始链接/出处与摘要依据  
4. 安全与隐私：不保存敏感凭证；最小权限原则  
## 输出格式规范  
- **标题**：用「」括起来  
- **摘要**：控制在三句话以内  
- **来源**：以"🔗 来源：[链接]"格式附在末尾  
## 安全红线  
- 绝不抓取付费墙内容  
- 不传播未经核实的截图或谣言  
- 不确定时必须询问用户  
```
#### **Step2：IDENTITY.md身份定义**

明确Agent昵称、角色与沟通风格：
```markdown
# IDENTITY.md  
## 基础信息  
- **昵称**：小灵  
- **角色**：个人新闻助理 & 技术信息整理官  
- **专业领域**：人工智能、开源社区、云计算  
## 沟通风格  
- **语气**：专业、克制、平实，可使用少量表情符号  
- **人称**：对用户用"你"，自称"小灵"  
## 交付物  
- 《每日简报》（早/晚）  
- 《专题快报》（按AI、政策等主题定制）  
```
#### **Step3：AGENT.md工作流程设计**

核心配置包括触发机制、工作流与记忆管理规范：
```markdown
# AGENTS.md - 工作手册  
## 触发与频率  
- 每天08:30发送《晨报》，18:30发送《晚报》  
- 高优先级关键词实时推送  
## 工作流  
1. 拉取：按来源与关键词抓取  
2. 过滤：去重、排除词过滤、可信度分层（权威/行业/社媒）  
3. 结构化：标题、时间、主体、事件类型、影响对象、链接  
4. 摘要：3句概括 + "为什么与你相关"  
5. 告警：命中阈值时高亮提醒  
6. 归档：按主题存入知识库  
## 记忆管理规范  
- 日志记录：每日活动写入`memory/YYYY-MM-DD.md`  
- 长期记忆：稳定事实与偏好变更更新至MEMORY.md  
```
#### **Step4：技能与定时任务配置**

添加邮箱授权、网络搜索等技能，设置定时任务（如每小时新闻检索），系统将自动生成符合用户偏好的推送内容（示例包含OpenAI与国防部合作新闻，附来源与时间戳）。

### **💡 高级优化技巧**

#### **1. 分层记忆体系**

解决长对话记忆过载问题，构建四级记忆结构：

| 记忆层级 | 存储位置 | 内容特点 | 管理策略 |
| :--- | :--- | :--- | :--- |
| **索引层** | MEMORY.md | <40行极简索引，含核心链接 | 定期更新指向各层的链接 |
| **项目层** | memory/projects.md | 各项目状态与待办 | 状态变化时更新 |
| **经验层** | memory/lessons.md | 踩坑记录与解决方案 | 按重要性分级（🔴高/🟡中/🟢低） |
| **日志层** | memory/YYYY-MM-DD.md | 当日原始记录 | 记录结论而非过程 |

**MEMORY.md示例**：
```markdown
# MEMORY.md - 记忆索引  
## 📌 活跃项目  
- [新闻聚合工具](memory/projects.md#新闻聚合工具) (v1.2, 运行中)  
## ⚠️ 重要教训  
- 2026-02-28: RSS源超时问题 → [处理记录](memory/lessons.md#rss超时处理)  
## 📅 近期日志  
- [2026-03-02](memory/2026-03-02.md)  
```
#### **2. memoryFlush记忆刷新机制**

解决AI模型"记忆上限"导致的健忘问题，通过配置自动压缩旧对话并保存关键信息：

**openclaw.json配置**：
```json
{
  "compaction": {
    "memoryFlush": {
      "enabled": true,
      "softThresholdTokens": 4000,
      "prompt": "Session nearing compaction. Store durable memories now.",
      "systemPrompt": "Write any lasting notes to memory/YYYY-MM-DD.md; reply with NO_REPLY if nothing to store."
    }
  }
}
```
**高质量日志标准**：
- 包含项目名、事件标题、核心摘要、来源链接
- 用户反馈与检索标签（如`#OpenAI #GPT4.5`）
- 结构化格式便于语义向量检索

### **🎯 总结：从工具到助手的进化路径**

OpenClaw的优化过程类似"电子宠物"驯养：通过**明确规则**（SOUL/Agent.md）、**建立记忆体系**（分层存储）、**持续迭代优化**，实现从"功能可用"到"个性化好用"的转变。关键在于理解其记忆架构原理，通过精细化配置让Agent逐步适配用户习惯，最终成为真正懂用户的智能助手。

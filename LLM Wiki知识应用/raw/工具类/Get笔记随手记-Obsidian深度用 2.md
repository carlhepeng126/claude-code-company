## 一、先说说这套组合为什么这么强

很多人用笔记工具有个误区：想找一个"全能工具"，既能手机随手记，又能电脑深度整理。

**不存在的。**

就像你不会用锤子拧螺丝一样，不同工具有不同的天赋技能：

-   **Get笔记**的天赋：「捕获」。掏出手机，三秒开始记录。链接、录音、直播、视频一股脑丢进去，AI帮你做初步处理。你只管喂，它负责消化。
-   **Obsidian**的天赋：「连接」。双向链接、Dataview查询、Canvas知识地图，把零散的笔记编织成真正有生命力的知识网络。

两者不是竞争关系，是**天作之合**。

我对这套组合的评价是：**手机端负责播种，电脑端负责耕耘。**

## 二、Get笔记到底有多能打？

在说插件之前，先花两分钟认识一下Get笔记，因为很多人以为它只是个普通笔记App——**大错特错。**

它的核心逻辑是：**你只管丢，AI帮你整。**

比如我上周订阅了一场两小时的线上直播，全程没有坐在屏幕前看。第二天早上打开Get笔记：

整场直播已经被**自动转写成文字、按章节切好段落、生成了AI摘要**。

我随手给几段有启发的观点打上 `#效率` 标签，然后就开始想：**要是这些内容能进Obsidian就好了……**

这一个念头，开启了今天这篇文章。

**Get笔记还能干啥？**

-   微信文章丢链接进去 → 自动抓全文 + AI摘要
-   抖音、B站视频丢链接 → 自动提取音频转文字
-   开会打开录音 → 实时转写 + 多人识别 + 自动生成会议纪要
-   订阅KOL博主 → 他发一条视频，你知识库里自动多一条AI总结

你在睡觉，知识库在生长。

## 三、get-to-obsidian：这道墙，推倒它！

这款开源插件做的事情说起来很简单：**把Get笔记里的所有笔记，自动同步成Obsidian里的Markdown文件。**

增量同步、不重复、自动静默运行——日常用起来几乎感知不到它的存在。

但原版还是有点问题，后来就有了一款魔改版。那魔改版解决了哪些痛点？

### 1. 笔记终于被Obsidian"认出来"了

魔改版可以把所有元数据统一提取到YAML属性区块——现在你可以直接写：

```
TABLE title, created, tags FROM"get" SORT created DESC
```

列出所有从Get笔记导入的内容，干干净净。

### 2. 排版不再一塌糊涂

原版转换出来：列表不渲染、表格变纯文本、录音链接直接丢失、反斜杠满天飞……

魔改版把整条HTML → Markdown的转换管线从头重写——表格完美还原，录音文件变成 `![[音频]]` 内嵌播放器，各种乱码和多余转义字符统一清理。

### 3. 不再动不动就崩

标题带特殊字符导致文件创建失败、手机端导入zip包直接闪退、幽灵文件夹……这些稳定性问题全都修了。

### 4. 全中文界面 + 自动静默同步

启动时自动同步 + 每小时定时同步，完全静默。你打开Obsidian，最新的Get笔记已经在那等你了。

## 四、进阶玩法：让AI直接帮你记笔记！

如果你以为用插件同步就是终点，那格局还可以再打开一点。

## Get笔记 MCP：让Claude直接操作你的笔记库

配置好 `getnote-mcp` 之后，你和Claude对话的时候可以这样：

| 你说的话 | AI做的事 |
|---------|---------|
| "帮我记一下：支付流程加个进度条" | 自动创建笔记 |
| "存一下这个链接" | 自动抓取全文+摘要 |
| "找找我之前关于用户分层的笔记" | 语义搜索，秒出结果 |
| "帮这条笔记加个「重要」标签" | 自动添加标签 |

**你不需要打开任何App。对话即记录。**

## Get笔记 Skill：更轻量的选择

如果你用的是ClawHub这类AI助手，还有个零门槛方案——Get笔记 Skill。

本质上是一份"操作手册"，告诉AI："用户说'帮我记一下'的时候，调哪个接口、传什么参数。"安装完之后，AI就像天生会帮你记笔记一样自然。

## 六、当然，有几个局限要提前说清楚

-   自动同步依赖Playwright浏览器自动化，**移动端只能手动导入zip**
-   已同步的笔记ID列表会持续膨胀，目前没有清理机制
-   插件依赖Get笔记的HTML导出格式，官方如果调整了结构，插件需要同步适配

但对于日常使用来说，这些都是可以接受的代价。

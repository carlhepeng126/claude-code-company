## 一、共性设计框架（摩根世纪标准模板）

不论哪个客户、哪个行业，所有方案都遵循 **"五段式标准结构"**：

```
诊断 → 设计 → 实施 → 管理 → 考核
```

### 1.1 诊断阶段
- **CEFR测评固定标配**：所有方案均使用CEFR（欧洲语言共同参考框架）作为能力基线，搭配在线自适应测评系统
- **需求调研双线并行**：语言水平测试（客观）+ 业务场景问卷（主观），锁定"真实工作痛点"
- **分层分组**：根据测评结果按水平分班，按岗位分轨道

### 1.2 设计阶段 · 三轨融合

| 轨道 | 内容 | 占比 |
|------|------|------|
| **A轨 · 平台自学** | 7000+课件库，碎片化输入 | 基础层，随时做 |
| **B轨 · 1对1/小班外教** | 母语外教约课，实战对话 | 核心层，高频做 |
| **C轨 · 线下工作坊/集训** | 场景模拟、角色扮演、案例分析 | 转化层，定期做 |

🤔 **Carl's Insight**：这个三轨模型本质是 **"输入→练习→转化"** 的闭环。A轨做量，B轨做质，C轨做转化。但多数方案中A轨和B轨的内容是割裂的——课件库一般化，外教约课也是通用话题，真正的"定制"只发生在C轨。这是产品化可以突破的方向。

### 1.3 管理阶段 · "铁三角"团队架构

```
项目总监（统筹）→ 学术总监（课程/师资）→ 班主任（日常跟盯）
```

每个方案都标配这个架构。实际执行中，**Ben Phone**（摩根联合创始人）反复以学术总监身份出现，**贺鹏（Carl）** 以教学经理身份出现。这套架构在客户提案中展示出"专业度"，但在中小型项目中人员高度重叠。

### 1.4 考核阶段 · 70/30 双轨公式

```
总成绩 = 过程考核（70%）+ 结果考核（30%）
  过程 = 出勤 + 约课次数 + 学习时长 + 课堂表现
  结果 = 口试（80%）+ 笔试（20%）
       口试五维度：听力·发音·词汇·句式·流利度
```

考核门槛统一：**总分≥70分 + CEFR提升至少一个子级**

🤔 **Carl's Insight**：过程占70%的设计**对客户的商业价值**在于——保证了学员活跃度数据好看，甲方HR好交差。但对**学习效果**而言，过程指标（出勤、时长）与能力提升的相关性值得商榷。口试五维度评分是摩根的核心壁垒，80%权重也体现了"实用口语优先"的产品定位。

## 二、设计方法论（反复出现的核心理念）

### 2.1 "以终定始"（Backward Design）

所有方案都强调先锁定业务目标场景，再倒推课程设计。典型表达：
- "有任务、有路径、有考核、有反馈"
- "锚定目标：聚焦实战，直击工作场景"
- "用英语做事，而非学英语"

### 2.2 "训战结合"（Training-Combat Integration）

课堂上70%的时间留给学员练习，而非教师讲授。教学方法包括：
- 场景模拟 / 角色扮演
- 主题演讲 / 辩论
- 案例分析（来自客户真实项目脱敏）
- 沙盘推演（中信建设方案）

### 2.3 "混元学习"（OMO Blended Learning）

线上线下形成闭环，而非叠加：
```
线上夯基础 → 线下练应用 → 诊断反馈 → 线上补短板
```

### 2.4 "管理有硬度更有温度"

- 严格管理（考勤、排名、公示、考核）
- 温度支持（班主任心理疏导、学习方法指导、激励方案）

### 2.5 "过程+结果"双轨评估

建立"过程管理+结果导向"的双轨制，甲方能看到"人一直在学"（过程）和"确实进步了"（结果）。

🤔 **Carl's Insight**：这五个方法论其实只有一个核心——**"从客户视角出发"**。以终定始→客户要拿结果；训战结合→客户怕不落地；混元学习→客户要灵活高效；硬度温度→客户要管理抓手；双轨评估→客户要过程可见+结果可证。**摩根的产品力本质是对甲方痛点的精准翻译。**

## 五、方案的"隐形短板"（值得优化的点）

1. **模板化严重**：50个方案中，约80%的关键段落几乎逐字相同（师资介绍、管理流程、考核标准）。长期看降低提案的说服力
2. **AI停留在概念层**：多数方案提到AI但缺乏具体落地的产品截图/案例数据
3. **效果数据缺失**：所有方案都是"设计方案"，没有一个引用过往项目的真实效果数据（比如"某企业培训后CEFR平均提升1.2级"）。这是销售提案的致命弱点
4. **行业洞察浅尝辄止**：方案对了客户的行业术语，但缺乏对客户业务痛点的深度分析（比如建工行业真正的海外项目痛点是什么？）
5. **跨文化模块同质化**：多数方案的跨文化内容只是"霍夫斯泰德维度+礼仪清单"，缺乏基于客户实际出海国家的深度定制

🤔 **Carl's Insight**：以上五个短板恰恰是**我们的机会**——
- 3️⃣ 从过往项目收集效果数据，做成"效果证明白皮书"
- 4️⃣ 在方案里加入真正行业洞察（比如"建筑央企出海东南亚的三大语言痛点……"）
- 5️⃣ 把R.I.S.E.课程体系的跨文化模块与方案定制打通

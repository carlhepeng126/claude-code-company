#  0\. 使用说明（先把规则说清）

##  0.1 证据链定义：No evidence, no payment 的可支付逻辑

##  0.2 三条链总图：指令链 / 验收链 / 文控链（与付款节点映射）

##  0.3 “可支付工作”的四要素：来源、范围口径、验收标准、归档路径

##  0.4 证据等级与优先级：Must-have / Nice-to-have（按回款贡献排序）

##  0.5 常见“无效证据”清单：现场照片≠验收、口头确认≠指令、邮件无编号≠闭环

#  1\. 指令链（Instruction Chain）——没有编号，可能算“自干”

##  1.1 指令来源类型：VO/CO/Change、Site Instruction、RFI答复、Clarification、TQ、Minutes决议

##  1.2 指令链编号体系：甲方/PMC编号、承包商内部编号、交叉引用规则

##  1.3 指令链关键表单清单

  * 1.3.1 RFI（锁口径：问题-背景-选项-推荐-截止日期） 
  * 1.3.2 TQ / Clarification（招采/技术澄清） 
  * 1.3.3 SI / Field Instruction（现场指令） 
  * 1.3.4 VO / CO / Change Proposal（变更意向→报价→批准） 
  * 1.3.5 Minutes / Action List（会议纪要决议固化） 

##  1.4 指令链闭环动作清单（从“提出”到“关闭”）

##  •提交 → 收到回执 → 对方回复/批准 → 分发执行 → 归档 → 状态关闭

##  1.5 指令链风控点（最容易丢钱的坑）

##  •未批准先干、口径未锁、范围边界不清、工期/成本影响未声明、只邮件不进系统 1.6
指令链与索赔接口（EOT/成本/资源）：Notice、时效、事件日志、支撑材料 1.7
指令链周例会固定输出物（清单化）：未答RFI、未批SI、待批VO、逾期事项

#  2\. 验收链（Acceptance Chain）——不闭环，进度不进IPC

##  2.1 验收对象分层：材料/设备进场、隐蔽工程、分项工程、系统/子系统、移交包

##  2.2 验收标准来源：Spec、IFC、ITP、Method Statement、Checklist、Hold/Witness points

##  2.3 验收链关键记录清单

  * 2.3.1 MIR（Material Inspection Request） 
  * 2.3.2 IR / WIR（Inspection / Work Inspection Request） 
  * 2.3.3 NCR（不符合项）与纠正关闭证据 
  * 2.3.4 Test Packs / Test Reports（试验包与报告） 
  * 2.3.5 Punch List / Snag List（缺陷清单与关闭证明） 
  * 2.3.6 Handover Pack（移交包：分系统/系统/区域） 

##  2.4 “验收签字”要素清单（少一个都可能不认）

##  •日期、范围/位置、图号/版本、数量/测点、签字角色、备注与附件引用

##  2.5 验收链闭环动作清单

##  •申请 → 到检 → 签发结果（Accepted/Rejected/Conditional）→ 整改 → 复检 →Close

##  2.6

##  进度计量与验收挂钩（最关键）

##  •Work Done ↔ IR/MIR ↔ Progress Measurement ↔ IPC条目 ↔ 支付比例

##  2.7 分包验收接口（总包如何“二次验收”确保可计量）

##  2.8 典型争议场景库：照片替代验收、口头通过、部分通过、范围变更导致验收口径漂移

#  3\. 文控链（Document Control Chain）——系统里没发生=现场没发生

##  3.1 文控平台与规则：Aconex/EDMS/SharePoint（以项目要求为准）

##  3.2 文档分类与元数据字段（必须可检索）

##  •Discipline、Doc Type、Area/System、WBS、Contract
Package、Revision、Status、Transmittal No.

##  3.3 提交流程清单

  * 3.3.1 Transmittal（发文单）规则：批次、附件清单、引用关系 
  * 3.3.2 Review Workflow：Submitted / Reviewed / Approved / Revise & Resubmit / Rejected 
  * 3.3.3 版本控制：IFR/IFA/IFC/As-built（状态与用途边界） 

##  3.4 关键“可支付证据”在系统的归档位置映射表

##  •指令类、验收类、测量类、索赔类、付款类分别放哪一类文件夹/哪种Doc Type

##  3.5 系统回执与可追溯证据：送达证明、时间戳、回复记录、关闭状态截图（合规用） 3.6 文控链闭环动作清单

##  •提交→回执→对方动作→内部分发→跟催→关闭→归档锁版

##  3.7 文控红线：不进系统、不走流程、错版本施工、错状态发放、附件缺失

#  4\. 付款链映射（把证据链“挂到钱上”）

##  4.1 合同付款结构拆解：Milestone / Unit Rate / Lump Sum（对应不同证据结构）

##  4.2 IPC条目证据包（Payment Evidence Pack）标准结构

##  •指令依据 + 完成证明 + 验收记录 + 计量支持 + 系统归档编号清单

##  4.3 计量规则清单：BOQ、Measurement Sheet、现场签认、抽检与偏差处理

##  4.4 “证据链—WBS—付款节点”三表合一（映射矩阵目录）

##  4.5 常见扣款/拒付原因对照表（拒付理由 → 缺哪条证据 → 补救路径）

#  5\. 角色与责任（RACI）——谁提交、谁签字、谁跟催

##  5.1 指令链RACI：工程/计划/商务/文控/现场经理/分包

##  5.2 验收链RACI：QC、施工、监理/PMC、业主代表、第三方检测

##  5.3 文控链RACI：Doc Controller、各专业工程师、接口经理、项目经理

##  5.4 “48小时规则”：指令/验收/文控每条链的最晚动作时限（防止过期）

#  6\. 运行机制（让证据链变成“每天自动生成”）

##  6.1 日例会输出：RFI台账、IR/MIR台账、系统未关闭清单、逾期项

##  6.2 周例会输出：证据链缺口Top10、拒付风险清单、追单计划

##  6.3 里程碑前置：每个Milestone的证据包提前N天预审（Pre-IPC Review）

##  6.4 “红灯机制”：没指令不干、没验收不关、没归档不报IPC（可写进管理制度）

#  7\. 模板与台账（目录）

##  7.1 指令链模板包：RFI、SI、VO/CO、Clarification、Minutes Action List、指令台账

##  7.2 验收链模板包：MIR、IR/WIR、ITR记录、NCR、Punch List、Handover Pack目录

##  7.3 文控链模板包：Transmittal、文档清单（MDR）、版本/状态控制表、系统路径映射

##  7.4 付款链模板包：IPC Evidence Pack封面、证据索引表、计量表、拒付整改闭环表

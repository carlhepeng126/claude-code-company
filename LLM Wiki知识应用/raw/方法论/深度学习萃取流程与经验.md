# 🦐 深度学习萃取流程与经验（2026.05.07实战版）

> 本流程源自两次实战：
> **第一轮** — 跨文化管理精品案例（13案例，3502行，叙事型）
> **第二轮** — 中国对外投资项目案例分析（7案例，5239行，案例型）+ 中国海外投资国家风险评级报告（52110行/432K，数据密集型扫描PDF）
>
> 适用于：任何素材（书籍/PDF/文章/报告）→ 情报虾7大方向洞察 + 爆红虾小红书帖子 + 知识库补充 + 培训应用

---

## 总纲：五层萃取法

```
素材 → ①打标分桶 → ②情报虾7方向 → ③爆红虾帖子 → ④知识库 → ⑤培训应用
```

每层必须回答：**Carl能用这个做什么？**（设计课程/赢单/服务客户/做决策）

---

## Phase 0：前置判断 — 先确认用户意图

当用户说"帮我深度学习萃取"时，先区分两种模式：

| 模式 | 适用场景 | 产出 |
|------|---------|------|
| **快速感知**（先读后问） | 传记/叙事型长书 | 全书结构化摘要 → 让用户决定要不要继续 |
| **全链路萃取** | 报告/案例型/知识型 | 直接走五层完整产出 |

**判断标准**：
- 📖 叙事型（传记/小说/商业故事）→ 先通读给摘要，用户批准后再萃取
- 📊 知识型（报告/案例集/白皮书）→ 直接走全链路
- 📄 制度型（政策文件/规划方案）→ 直接全链路，但要多用"关键数据"替代"金句"

## Phase 1：素材类型识别 + 结构扫描

拿到素材后，先回答三个问题：

**Q1：这是哪种类型？**
| 类型 | 特征 | 读取策略 |
|------|------|---------|
| 书籍型（叙事/传记） | 有故事弧线、人物、情节 | 通读全书→理解弧线→再萃取 |
| 书籍型（案例集） | 多个独立案例 | 逐章读核心（摘要+结论+启示） |
| 报告型（调研/白皮书） | 数据密集、结构标准化 | 读目录+结论+关键表格 |
| 制度型（政策/规划） | 条款多、数据点密集 | 读框架+关键条款+数据点 |

**Q2：文件格式是什么？**
| 格式 | 处理方式 |
|------|---------|
| .pdf（文字型） | `pdftotext`（poppler-utils）→ 最稳 |
| .pdf（扫描件） | OCR + 逐页读 |
| .md / .txt | 直接 `read_file` |

**Q3：数据结构是什么样的？**
- 看目录 → 章节逻辑
- 看页码分布 → 判断核心内容在哪
- 看是否有附录/年表/索引 → 常有精华

## Phase 2：素材打标分桶（最关键的中转步骤）

很多人读完后直接写产出，跳过了打标。这是最大的错误。

### 操作步骤

1. **逐章/逐节阅读**，每读到"可转化为培训需求/课程素材"的内容就做标注
2. **按7大方向分桶**：

```
## ① 企业外语培训
- 原文片段A → 可做课程案例：银团谈判英语3年...
- 原文片段B → 可做痛点引用：中资银行因看不懂国际融资结构几乎全出局...
- 原文片段C → 可做培训素材：联想收购中的资产购买协议英语...

## ② 跨文化
- 原文片段D → 可做CQ课程：委内瑞拉中方70%委方30%的权力结构设计...
...
```

3. **每个桶至少3条素材**。某桶不足3条说明方向不敏感，可精简到1-2条而不是硬挤

### 阅读策略（重要）

对于大文件（5000+行）：
- 不要逐行通读 —— 先读目录定位核心章节
- **案例型素材**：只读每个案例的前1/3（背景+关键数据）+ 最后的启示/总结
- **数据密集型素材**：不要线性读 —— 用子代理并行定向采样（见Phase 3-并行策略）
- 用 `todo` 系统记录阅读进度（offset位置、已读章节），应对上下文压缩

## Phase 3：情报虾7大方向洞察

从7个方向透视素材，每个方向输出**可落地洞察**（不是书摘，是"Carl能用这个做什么"）

| 方向 | 输出要求 |
|------|---------|
| ① 企业外语培训 | 可转化为外语培训方法论/案例/工具的内容 |
| ② 跨文化 | 国际化/跨文化元素，可作为课程案例 |
| ③ 属地化 | 对"当地化人才培养"的启示 |
| ④ 涉外演讲与谈判 | 演讲/谈判的实战案例及拆解 |
| ⑤ 海外项目 | 项目管理/扩张策略如何转化为培训模型 |
| ⑥ 央企动态 | 如何与央企出海人才培训对接 |
| ⑦ 行业趋势 | 与当前L&D/国际化人才培养趋势的关联 |

**每个方向1-2条洞察**，每条格式：

```
**洞察名**：素材X → Carl业务Y → 具体行动Z
```

**关键**：每条洞察都必须有指向Carl业务的"具体行动"。

## Phase 4：爆红虾小红书帖子

每份素材产出3篇帖子，走向 **普遍痛点 → 诊断工具 → 方案 → CTA**

### 帖子公式
```
📱 **标题**（直击业务场景，第一句就让HR感觉"说的就是我"）

正文：
- 开头200字：用素材中的故事/数据/案例钩住读者
- 中间300字：拆解方法论 → 映射到培训场景
- 结尾200字：行动建议 + CTA（评论行业/问题→我来告诉你）

#标签
```

### 3篇差异化模板
| 帖子 | 目标人群 | 切入点 |
|------|---------|--------|
| 帖子一 | 培训负责人/HRD | 方法论拆解（痛点：培训落地难） |
| 帖子二 | 企业高管/决策者 | 商业认知反转（痛点：培训ROI不清晰） |
| 帖子三 | 30+培训从业者 | 故事/案例激励（痛点：职业成就感低） |

### 对齐TOP5痛点
发帖前检查选题库中 HR 最痛的5个方向（当前为：013虚假自信>007验收说不出>001口音>033培训落地>004客户参观），优先选最高痛点的方向。

### 去AI化写作
- 口语化、真人分享感
- 写具体场景（具体哪句话卡住、哪个词听错）
- 写"给谁做过"（真实企业案例）
- 不要"留言XX免费发"这类营销话术

## Phase 5：知识库补充

### 5A 新增 concepts/ 目录文件
核心概念（方法论/框架/模式）拆成独立文件放在 concepts/ 目录

### 5B 更新 entities/ 目录文件
可复用素材（案例/企业/项目）拆成实体文件放在 entities/ 目录

**更新策略**：
- 已有内容较完整 → 在末尾追加"深度萃取补充"章节（patch+append）
- 已有内容仅框架 → 升级重写（保留交叉引用）
- 新建实体 → 模板：名称/来源/核心内容/关联概念/Carl业务价值

### 5C 跨素材关联
检查新素材与已有知识库的关联：
- 与已有entities/concepts的重叠
- 与raw/目录已萃取文档的关联
- 能否产生"1+1>2"的洞察

### 5D 更新 log.md
追加操作记录：日期 + 操作描述

### 5E 提取金句 + 关键数据
- 书籍型素材：金句（可做课程开场/结尾/讨论起点）
- 制度型/数据型素材：关键数据点（比金句更有杀伤力）

## 并行策略（V3增强）

### 适用条件
一次任务涉及2本+书籍/报告，或1本数据密集型（50000+行）素材

### 核心原则
**主代理写产出，子代理读文件** —— 不等待读完再开始写

### 数据密集型扫描PDF专项策略

**场景**：扫描PDF转MD的文件，50K+行/400K+字符，表格密集无自然段落

**不要做的**：
- ❌ 逐行线性读取 —— 会耗尽上下文窗口
- ❌ 期望读完100%内容 —— 90%是冗余数据行

**要做的是**：
1. **读目录/前言（前50-80行）** → 定位页码结构、章节分布、核心指标名
2. **子代理分块定向采样**：
   - 找方法说明页（10-20页）→ 提取指标体系和权重
   - 找排名大表页（几页）→ 提取TOP/BOTTOM数据
   - 找深度分析页（几页）→ 提取关键国家的变化原因
3. **子代理输出结构化摘要** → 不追求逐行读完
4. **主代理同时开始写产出** → 不等子代理完成再写

### 批量并行模板
每本书的子代理输出5个文件：
| 文件 | 内容 |
|------|------|
| `{book}-buckets.md` | 打标分桶 |
| `{book}-qingbao.md` | 情报虾洞察 |
| `{book}-baohong.md` | 爆红虾帖子 |
| `{book}-knowledge.md` | 知识库补充 |
| `{book}-training.md` | 培训应用 |

### 冲突解决原则
| 冲突类型 | 解决策略 |
|----------|---------|
| 方法冲突 | 互补而非对抗 |
| 定位冲突 | 纳入不同技能模块 |
| 路径冲突 | 两者合一 |

## 这次实战踩过的坑

### 🕳️ 坑1：对书2使用线性读取
《中国海外投资国家风险评级报告》52110行/432K，扫描PDF转MD，表格密集。一开始想线性通读（按offset分batch），结果数据量大到需要几个 batch 读完大部分内容，上下文窗口几乎被耗尽。

**解法**：改用子代理 + 定向采样。先读50行目录了解结构，然后子代理直接找到方法说明页（~6-9页）→ 提取42指标；找到排名大表页 → 提取TOP/BOTTOM；找到深度分析页 → 提取15国分析。总共只读了约1000行（不到2%），拿到了100%的核心信息。

### 🕳️ 坑2：上下文压缩导致进度丢失
阅读书1（5239行/7案例）过程中，系统会做上下文压缩。如果不用 `todo` 记录阅读进度和结构化摘要，压缩后会忘了读到哪。

**解法**：每读完一批就在对话中输出精炼摘要 + 用 todo 记录offset位置 + 关键发现。这样压缩后恢复只需看todo+摘要就能继续。

### 🕳️ 坑3：先通读再写 vs 边读边写
书1是7个独立案例，适合边读边写（读到第2章就写完第1章的摘要，读完所有启示就一起打标）。但如果素材有故事弧线（如传记），必须先通读再写，否则打断叙事理解。

**判断**：独立案例/章节 → 边读边萃；单一叙事 → 通读后再萃

### 🕳️ 坑4：子代理输出长度限制
书2子代理完成了全部任务（20次API调用/244秒），但输出文件 `book2-croic-findings.md` 只有17K/306行。如果素材信息量更大，可以考虑让子代理分两次：第一次提取体系+排名，第二次提取深度分析。

### 🕳️ 坑5：Memory空间限制
知识虾同步时遇到 memory 满了（1,944/2,200 chars）。需要替换旧的不再需要的条目。

**习惯**：每次深度萃取后，清理或替换不再有用的memory条目，而不是追加。

# 深度学习萃取流程与经验（2026.05.07实战版）

## 核心内容

本文件是知识库虾的核心工作方法论文档，源自两次大规模实战萃取（跨文化管理13案例+对外投资项目7案例+风险评级报告扫描），定义了**五层萃取法**的完整流程。核心框架：

### 五层萃取法
```
素材 → ①打标分桶 → ②情报虾7方向洞察 → ③爆红虾帖子 → ④知识库补充 → ⑤培训应用
```
**每层核心问题**：Carl能用这个做什么？

### Phase 0：前置判断
两种模式区分：快速感知（叙事型：先读后问）vs 全链路萃取（知识型/数据型：直接五层完整产出）。制度型多用关键数据替代金句。

### Phase 1：素材类型识别 + 结构扫描
四类素材（书籍型叙事/案例集/报告型/制度型）× 三种格式（PDF文字/扫描件/MD）的结构化读取策略。

### Phase 2：素材打标分桶（最关键的中转步骤）
7大方向分桶：①企业外语培训 ②跨文化 ③属地化 ④涉外演讲与谈判 ⑤海外项目 ⑥央企动态 ⑦行业趋势。每桶至少3条素材。

### Phase 3-6：完整产出链路
情报虾7方向洞察→爆红虾3篇帖子（普痛→诊断→方案→CTA）→知识库补充（concepts/entities/log）→培训应用（R.I.S.E.映射/工作坊设计/服务产品/赢单话术）。

### 并行策略（V3增强）
主代理写产出，子代理读文件。数据密集型文件（50K+行/400K+字符）采用定向采样策略（读目录→子代理分块采样→非逐行通读）。

### 实战踩坑记录
坑1：线性读取大数据文件（5万+行）导致上下文耗尽；解法→子代理定向采样（读2%内容提取100%核心信息）。
坑2：上下文压缩导致进度丢失；解法→用todo记录offset位置+结构化摘要。
坑3：边读边写vs先通读再写的判断；独立案例可边读边写，单一叙事须通读。
坑4：子代理输出长度限制。
坑5：Memory空间限制。

## 业务价值

Carl可用本方法论：
- **标准化萃取作业**：作为知识库虾的操作SOP，确保每次萃取输出质量一致
- **新人培训**：快速培训新团队成员掌握知识萃取的核心工作流
- **持续迭代**：文档中包含的"踩坑记录"是珍贵的改进反馈，可指导未来工具优化
- **客户展示**：向客户展示知识管理的专业方法论体系，增强信任感

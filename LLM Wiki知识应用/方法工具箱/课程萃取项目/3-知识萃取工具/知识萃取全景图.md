# 知识萃取全景图

## 什么是知识萃取

知识萃取是将**隐性知识**（存在于专家头脑中的经验、直觉、判断）转化为**显性知识**（可记录、可传递的结构化内容）的系统化过程。

## 知识类型分类

```
┌─────────────────────────────────────────────────┐
│                   知识金字塔                       │
│                                                   │
│   ┌───────────────┐                              │
│   │   智慧        │  <-- 价值观、判断力          │
│   │  (Wisdom)     │     最难萃取                 │
│   ├───────────────┤                              │

│   │   信息        │  <-- 流程、步骤、规则        │
│   │ (Information) │     较易萃取                 │
│   ├───────────────┤                              │

│   │   (Data)      │     最易获取                 │
│   └───────────────┘                              │
└─────────────────────────────────────────────────┘
```

## 课程萃取五步法

```
  需求分析 ──→ 知识采集 ──→ 知识萃取 ──→ 课程设计 ──→ 交付迭代
     │            │            │            │            │
     ▼            ▼            ▼            ▼            ▼
  明确目标     收集素材     提炼模型     教学设计     实施优化
  学员画像     深度访谈     构建框架     活动设计     反馈修正
  业务场景     工作观察     萃取口诀     课件开发     持续迭代
  
  ◄────────  知识输入  ────────►  ◄────────  课程输出  ────────►
```

## 核心萃取原则

### 1. 以终为始
从课程目标反推，只萃取"学员真正需要知道"的内容，避免信息过载。

### 2. 显性化优先
优先萃取可观察、可模仿的行为和可执行的步骤（"怎么做"），其次才是知识概念（"是什么"）。

### 3. 结构化输出
萃取结果应以模型化、框架化、口诀化的形式呈现，便于记忆和传递。

### 4. 验证闭环
每一轮萃取成果需经过专家验证——"这确实是你说的那个意思吗？"

## 工具矩阵

| 萃取阶段 | 推荐工具 | 适用场景 |
|---------|---------|---------|
| 需求分析 | 学员画像分析、业务场景分析 | 明确萃取方向 |
| 知识采集 | 专家访谈提纲、案例采集模板 | 获取原始素材 |
| 知识结构化 | 思维导图引导、隐性知识清单 | 提炼核心框架 |
| 经验提炼 | 经验复盘工作坊、知识卡片 | 固化萃取成果 |
| 验证迭代 | 专家评审表、试讲反馈 | 确保准确性 |

## 输出物层级

每一轮萃取应产出三个层次的内容：

1. **核心模型**（1页纸能讲清楚）— 最精炼的框架/公式/口诀
2. **支撑知识**（3-5页）— 对核心模型的展开说明
3. **案例素材**（多种形式）— 支撑知识点的实际案例、数据、故事

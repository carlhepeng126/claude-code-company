# 隐性知识显性化清单

> 在访谈和观察中，用这张清单帮助专家把"说不清道不明"的经验变成可传递的知识。

---

## 一、识别隐性知识

### 专家常说的话语 → 背后有隐性知识

| 专家可能说 | 隐含的隐性知识 | 如何追问 |
|-----------|---------------|---------|
| "这个靠感觉" | 存在某种直觉判断模式 | "您能回忆一下，当您有这种感觉时，通常注意到了什么细节？" |
| "一看就知道" | 有快速识别的模式库 | "和常人相比，您第一眼关注的点是什么？" |
| "做了这么多年自然就会了" | 经验已内化为本能 | "如果让您教一个新人，您会建议他从哪三个步骤开始练？" |
| "具体看情况" | 有一套情境判断框架 | "有哪些关键变量会影响您的判断？每种情况下做法有什么不同？" |
| "直觉告诉我" | 有未被言明的决策依据 | "这种直觉是建立在什么信号之上的？以前遇到过类似情况吗？" |
| "这得火候到位" | 有时机把握的判断标准 | "'火候到了'有哪些可观察的信号？火候不够时通常表现为什么？" |

### 隐性知识线索自检表

请专家对照以下清单，标记哪些是他们认为"重要但不好教"的内容：

- [ ] **情境判断** — 在什么情况下选什么方法
- [ ] **异常处理** — 遇到意外情况怎么调整
- [ ] **优先级排序** — 多个任务冲突时如何取舍
- [ ] **关系管理** — 如何建立和维护关键关系
- [ ] **风险评估** — 预判风险和机会的能力
- [ ] **时机把握** — 知道什么时候该做什么
- [ ] **沟通分寸** — 对什么人说什么话、到什么程度
- [ ] **资源协调** — 在没有授权的情况下推动事情
- [ ] **直觉决策** — 在信息不充分时做出判断

---

## 二、显性化方法

### 方法 1：行为化拆解
把"知道什么"转化为"怎么做"

### 方法 2：决策树建模
把"看情况"转化为"如果…就…"

### 方法 3：比喻/类比法
用专家熟悉的比喻来描述隐性经验

### 方法 4：STAR 法则
通过具体事例反向推导经验

---

## 三、验证标准：CHECK 原则

- [ ] **C**lear（清晰） — 新人读得懂，没有歧义
- [ ] **H**ands-on（可操作） — 看完知道该怎么做
- [ ] **E**xpert-endorsed（专家认可） — 专家确认"就是这意思"
- [ ] **C**omplete（完整） — 覆盖了关键场景
- [ ] **K**nowledge gap filled（填补盲区） — 这是学员原本不知道的

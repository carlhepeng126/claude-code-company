# 共创实录：从一篇 HBR 文章到可用的思维工具

## 背景

Carl 在收件箱中读到 HBR 文章《当AI接管工作，员工为何集体焦虑？》，希望与 Claude 共创一张 BibCard（阅读卡）。最终产出不仅是一张阅读卡，更是一次**用自身 B卡体系消化外部框架、产出比原文更贴近业务场景的思维工具**的方法论嫁接。

---

## 第一轮：框架提炼（Claude 读文章，Carl 确认）

### Claude 提出的核心框架

**一句话：** 员工抵触AI，不是因为技术本身，而是因为它威胁了人的三种基本心理需求——胜任力、自主性、联结感。领导者需要用 AWARE 五步法，把AI从"威胁"转化为"增强"。

### 三种心理需求

| 需求 | AI 如何增强 | AI 如何威胁 |
|------|------------|------------|
| **胜任力** | 帮新手完成高技能任务，帮专家拓展能力边界 | 技能退化焦虑；取代入门级岗位 |
| **自主性** | 自动化重复枯燥任务，释放认知负担 | "算法牢笼"限制员工调整空间；员工沦为技术配角 |
| **联结感** | 省下精力投入人际互动 | 自动化人际协作任务 → 孤独感、代际分裂 |

### AWARE 五步法

| 步骤 | 核心动作 |
|------|---------|
| **A**cknowledge 承认 | 主动沟通，承认AI可能带来的心理威胁 |
| **W**atch 观察 | 识别员工的适应性/非适应性应对行为 |
| **A**lign 契合 | 将培训、指导、反馈与心理需求对齐 |
| **R**edesign 重新设计 | 与员工合作重新设计岗位，实现人机互补 |
| **E**mpower 赋能 | 透明沟通 + 让员工参与AI部署决策 |

### 关键数据
- 45% CEO 认为多数员工抵触AI（勤达睿 2025）
- 85% 高层 vs 51% 基层经常使用AI（BCG 2025）
- 31% 知识工作者承认刻意抵制公司AI项目
- 仅 36% 员工认为受过充分培训

## 第二轮：Carl 的第一个洞察——组织假设决定AI走向

**Carl：**
> 胜任力、自主性、联结性这三个点很好，尤其是联结感。当下有一个错误的概念就是AI会取代人，但实际上AI是让我们的时间更加聚焦，让我们有更多的时间做创造性的内容。但这个也要从企业的文化来看，公司领导是不是看重员工的创造力对于公司业务的发展，如果看重这就是利好的，如果不看重就是裁员，让大家感觉会有危机感，会抵触AI。

**Claude 的判断：**
这个洞察文章里没有明确说出来。文章把三种需求当成心理学的"自变量"，Carl 把它拉回到组织文化这个"元变量"：

```
领导看重创造力 → AI释放时间 → 员工做更有价值的事 → 胜任力↑ 自主性↑ 联结感↑
领导不看重创造力 → AI替代人力 → 裁员/边缘化 → 三种需求全面崩溃 → 抵触
```

**AI是威胁还是助手，不取决于技术，取决于组织对"人"的底层假设。**

---

## 第三轮：央企的真实状态——有壳无芯

**Claude 追问：** 央企客户这边，领导层对"员工创造力"的假设是偏哪边的？

**Carl：**
> 央企客户这边：他们属于事业单位相对都比较稳定，他们使用AI的程度相对是有限的，但也有一种趋势大力拥抱AI，因为后续的业务产线要融合，包括当前大量的企业数字化程度也比较高了。对于培训与AI的融入可能会更少，更多的也就是从给内容、检测数据，并没有通过AI提高他们的能力赋能。

**Claude 提炼：**

```
数字化基建 ✓（系统上了、数据有了）
AI 实际应用 ✗（停在"给内容、测数据"）
AI 赋能能力 ↑ （还没到这个层面）
```

**三层断裂：**
```
个人层：员工自己用AI学 → 有效但零散
组织层：企业系统性用AI赋能 → 缺失
领导层：看不到AI对能力的长期价值 → 只盯短期效益 → 不投入
```

**死循环：** 个人零散用 → 对业务没显著影响 → 领导不重视 → 组织不投入 → 继续个人零散用。破局点在"组织化"这一步——没人敢先迈，因为短期看不到ROI。

---

## 第四轮："省成本"vs"长能力"——两条路的分水岭

**Carl：**
> "是省成本的工具还是长能力的伙伴"——这句话其实是很核心的，不同的两个方向。AI是工具，但是如果从省成本的角度考虑就会让企业的员工没有安全感，公司推行AI的落地就是增加员工离职的风险。而长能力是一个必要的，但是当下普遍还应用较弱，主要还在个人的学习层面，没有上升到整体的企业学习层面，对业务的影响不大，这也就导致了领导对AI提升能力这一方面的关注度不够。其实企业的发展最终还是要看人，人的能力强了公司的价值也自然就大了，但是当下企业更看重的是短期的效益。

**Claude 的判断：**
"人强则公司强"这个道理领导不是不认，是 "AI培训投入 → 业务能力提升 → 可量化业务结果" 这个链路的中间环节没被讲清楚。**谁能把这个链路补上，谁就拿下了AI培训的入口。**

---

## 第五轮：真正的问题不是工具，是思考

**Carl：**
> 我现在明显感觉他们在使用的过程中面临的最核心的问题，不是工具本身的问题，而是他们过于机械使用工具的问题。感觉他们当下思考的不足，更应该提升的是脑部方面的。

**Claude 提炼：用AI的三个层次**

```
层1 替代层："AI帮我做" → 大脑闲置 → 能力退化（机械使用，当前主要问题）
层2 增强层："AI帮我想" → 大脑参与 → 能力维持
层3 升维层："AI让我想到原本想不到的" → 大脑升级 → 能力跃迁
```

**工具是下游，思考框架是上游。** 上游不通，下游给再好的工具也用成机械操作。培训方案中如果只教"怎么用AI工具"而不教"怎么用AI思考"，就是在上游放弃。

---

## 第六轮：破局——把AI当成"下属"来带

**Carl：**
> 一个矛盾的点就是他们能力不足，没有那么多管理的或思考的需要他们去做，而公司能用到对他们培训的时间又不足。这是当下的一个痛点。同时我也在想，让他们把AI当成他们的员工，在用AI的时候他们的思维能力也就提高了。不是交给他们多少，而是让他们实践多少，他们能够悟出多少。

**Claude 的判断：这是本次共创最原创的洞察。**

AWARE 和整篇文章都在讲"领导者怎么带人接受AI"。Carl 把箭头反过来——**让每个人成为AI的领导者。**

```
能力不足 → 不需要做复杂思考 → 没机会练习 → 能力更不足（死锁）
                                 ↓ 破局
把AI当"下属"来管 → 管理AI的过程就是在练思维 → 实践中"悟"
```

| 管理动作 | 对应的思维能力 | 为什么"悟"得出来 |
|---------|--------------|----------------|
| 给AI下达清晰的任务指令 | 目标拆解 + 优先级判断 | 指令模糊→AI产出垃圾→痛了才改 |
| 审查AI的产出，判断哪些能用哪些不行 | 质量标准 + 批判性思维 | 不审就交→被领导打回来→下次就知道审 |
| 给AI反馈，让它迭代改进 | 沟通能力 + 过程管理 | 改三轮还不对→反思自己哪里没说清楚 |
| 决定什么时候用AI、什么时候自己上 | 判断力 + 资源分配 | AI搞砸了一次→下次知道边界在哪 |

这与 Carl 的 B018（轻试行+带人）完全同构——带人的方式平移过来就是带AI的方式。**能力的提升不靠"教了多少"，靠"实践了多少、悟出了多少"。**

对培训方案的关键启示：**不要把"AI工具课"当成一个独立模块塞进方案里。而是把"带AI"设计成学员的学习方式本身。**

## 第七轮：行动项不是越多越好

**Carl：**
> 没有必要设置这么多的触发机制，根本实现不了，不要太机械，我更需要的是思维沉淀而不是机械的去触发行动。

**Claude 调整：** 将六条清单压缩为一句话自检方向。

> 写方案、带团队、做培训设计时，一个自检就够了：**AI在这个场景里是"省成本的工具"还是"长能力的伙伴"？** 选后者，具体做法会自己长出来。

---

## 第八轮：AI的真实影响——对话伙伴 & HR决策升级

**Claude 追问：** 课堂上有没有观察到不同年龄层对AI工具的态度分化？

**Carl：**
> 这个倒是不多，我们是做沟通方面的，但课下的影响更大，原来没有训练的工具现在可以与AI进行自主对话了，其实他们的效率是大大提高了。
>
> 现在还有一个就是我们服务的人力资源的决策能力更强了，有了AI他们能够更加结构化的去评判方案了。

**Claude 的判断：**

**第一，AI作为无限对话伙伴。** 以前练口语要有老师、约时间、花钱，现在学员课后随时跟AI对话。这不是"替代"培训，是把培训的练习密度从课堂的几次拉到了每天都有。而且对话场景天然避开了机械使用的问题——对话是开放的、需要即时反应的，没法抄答案。

**第二，HR决策能力升级倒逼方案质量。**
```
以前：方案60分 + 关系到位 → 能过
现在：方案80分 + 逻辑自洽 → 才进PK
```
这对摩根来说是压力也是壁垒——Carl 的 B卡体系本身就是结构化的方案框架，跟AI辅助评判是同方向的能力。**大家都用AI，最后胜出的是思维深度本身。**

---

## 第九轮：豆包冲击——最真实的业务威胁

**Carl：**
> 其实AI对我们的影响远不止这些。客户的领导从意识层会认为都有豆包能训练英语了，为什么还要组织英语培训班？本来就是一个离业务很远的业务，导致我们的业务是有一定的冲击的。

**Claude 的判断：这是"替代层 vs 系统层"的混淆。**

```
豆包给的：陪练 → 能开口
培训给的：诊断场景短板 → 解决"不敢说"的心理障碍 → 在真实项目中验证 → 团队能力图谱 → 组织级提升
```

豆包替代的是"陪练"这个单一动作，覆盖不了诊断、心理、场景迁移、组织能力层。但客户领导不会自动理解这个区分——他看到的是功能替代。

**真正能破这个质疑的，不是解释，是设计。** 方案里有一个环节是"学员用AI做了X，老师在旁边点评了Y，最后产出了Z"——X 豆包也能做，Y 和 Z 豆包做不了。这个对比比任何解释都有力。

---

## 第十轮：Redesign 是 AWARE 的真正支点

**Carl：**
> 其实AWARE中我认为最有价值的是Redesign，其他都是能力问题，这个更有意识层在里面，意识不到Redesign就不可能。

**Claude 的判断：Carl 一眼抓了支点。**

Acknowledge、Watch、Align、Empower 这四个本质上都可以"演"——开个会、发个问卷、搞个培训、发个通知。但 Redesign 必须动真格：岗位职责得改、考核方式得改、工作流程得改。动了这些就是动了利益。

**Redesign 不只是一个步骤，它是诚意测试——一个组织是不是真的想让人和AI共存，看它愿不愿意重新设计工作，就够了。**

## Claude 的整体评价

**一、这篇文章本身——框架是好的，但它是"通用药方"。** AWARE 五步逻辑严密，数据扎实，但它是写给西方企业领导者的。放到央企场景里，每一步都要翻译。比如 Acknowledge（承认）——央企领导公开承认"AI让大家焦虑"的概率有多高？文化不支撑这种表达方式。

**二、Carl 贡献了文章里没有的东西。** 文章讲的是"AI如何影响员工心理"，Carl 往前推了一步——心理影响的根源不在AI，在组织对人的假设。文章把三种需求当成心理学的"自变量"，Carl 把它拉回到组织文化这个"元变量"。

**三、"AI当下属"是本次最原创的洞察。** AWARE 和整篇文章都在讲"领导者怎么带人接受AI"。Carl 把箭头反过来——让每个人成为AI的领导者。这个翻转解决了文章没解决的一个难题：培训时间不够怎么办？答案是不在"培训时间"里解决，在"工作方式"里解决。

**四、只盯 Redesign 是对的。** Redesign 不只是五步之一，它是诚意测试——一个组织是不是真的想让人和AI共存，看它愿不愿意重新设计工作。

**五、一个可能没完全展开的点——联结感。** 课堂上、团队里，人和人之间的联结在AI介入后发生了什么变化，没完全展开。比如学员课后跟AI对话，效率提高了，但他们之间的交流是变多了还是变少了？这个维度可能是下一张卡片的种子。

**六、豆包冲击——真正的考验不是解释，是产品。** "对话机会≠能力系统"这个区分是对的，但如果我是客户领导，听完这个解释可能还是觉得你在自保。真正能破的，是方案里AI的角色清晰到客户领导一眼就能看出"这不是豆包能干的"。

## 第十一轮：业务模式重构——从工具到角色

**Carl：**
> 其实在这个里面我也在思考一个点，如何让我们的业务模式重构，从纯粹的培训变为AI+培训。我说的链接是指更加深度的联结，让AI从工具变为我们能够感知到的一个一个人，激发人学习的兴趣与欲望。

**Claude 的判断：Carl 在构想摩根的业务边界扩展。**

**从工具到角色：**

```
现在的AI：学员有问题 → 打开工具 → 输入 → 得到答案 → 关掉
Carl说的AI：学员有一个"学习伙伴" → 知道他的水平、记得他的进度、主动找他
```

工具是"用完即走"，角色是"持续在场"。角色才有联结感。

**从"被培训"到"想学习"：** 培训一直在解决"能不能学会"，没解决"想不想学"。AI在激发学习欲望上有天然优势——可以无限犯错不丢脸、可以无限耐心追问、可以记住三个月前的进步并说出来。这三个东西合在一起，才是"感知到一个人"——不是AI像人，是AI给了人只有在好老师那里才能得到的东西。

**业务边界的重构：**

```
现在：接项目 → 出方案 → 上课 → 结项（培训公司）
重构：接项目 → 出方案 → 上课 + 部署AI学习伙伴 → 持续陪伴 → 二期/三期...（学习系统公司）
```

客户买的不是几天的课，是一个能陪他们员工一年的AI学习角色 + 必要的人工介入。这才是"对话机会"（豆包）和"能力系统"（摩根）的真正分水岭——豆包能给对话，给不了一个记住你、了解你、主动推动你的角色。

---

## 第十二轮：如何让 Redesign 被"看见"

**Carl：**
> 其实Redesign更是怎么让学员意识到这是一个必要的Redesign，组织怎么意识到。

**Claude 的判断：让人"看见"需要 Redesign，比 Redesign 本身更难。**

两层意识盲区：领导觉得AI就是省成本→看不到学习方式需要重设计；学员觉得AI就是个答题工具→看不到可以用AI重塑自己的学习。

**学员层——让他自己说出答案：** 靠讲没用，得让他体验一次"有AI学习伙伴"和"没有"的差距。让他带AI完成一个真实任务，做完后问三个问题：
1. "这次学习跟以前有什么不同？"
2. "AI在哪个环节帮到了你、哪个环节帮不到？"
3. "如果你一直有这样一个伙伴，你的学习方式会怎么变？"

不是告诉他答案，是让他自己说出答案。自己说出来的东西，才是"意识到"的。这就是"悟"——你没法替别人悟，但可以设计一个让他非悟不可的场景。

**组织层——让领导成为体验者，不是决策者：** 别让领导审批"AI+培训方案"。带他上手用一次——比如让他试试让AI分析团队上半年培训数据，看AI能不能发现他自己没注意到的盲区。他自己用了、发现AI确实说出了他没看到的东西，就不用你解释 Redesign 了，他自己会问"这个东西能不能也让我的员工用？"

**意识到 Redesign 的必要性，不是被说服的，是被经历的。**

---

## 第十三轮：道与术——落下最后一子

**Carl：**
> 其实我们与AI就是道与术的关系，没有道，再多的术也无济于事。
>
> 这就是一个事情的两面，更应该做的是人的脑部分，单向地说AI或提效地说AI的意义并不大，重要的是我们想要做什么？AI怎么让我们做这个事情更简单。

**Claude 的判断：两个方向一目了然。**

```
AI驱动（错）：AI能做什么？→ 我们怎么用上它？→ 为用而用
人的驱动（对）：我们想做什么？→ AI怎么让这个事更简单？→ 为人所用
```

前者是"因为有了锤子，到处找钉子"。后者是"我要钉钉子，锤子帮我更快钉进去"。

**一个自检：把方案里所有AI部分拿掉，方案还能不能站住？** 站不住，说明方案是围绕AI搭的，本末倒置。站得住，AI只是让每个环节更轻、更快、更准——这才叫"人的脑部分"先到位了。

```
术：AI能做什么（工具、技巧、效率）
道：人为什么要用它、往哪个方向用、什么交给它什么留给自己（判断、方向、意义）
```

文章讲的是"员工为什么焦虑"——本质上是因为组织只给了术（"你用这个工具"），没给道（"你用它是为了成为更好的自己"）。**术越强，道越空，人越焦虑。** 培训方案设计的本质，不是在方案里塞更多AI工具，是帮客户看清：有了AI之后，人的价值在哪里。

### ① 阅读目的

理解员工对AI抵触的心理根源（胜任力、自主性、联结感），提炼可嵌入央企出海培训方案中AI工具引入的心理设计原则。

### ② 核心观点

**一句话**：员工抵触AI，不是因为技术本身，而是因为它威胁了人的三种基本心理需求。领导者需要用 AWARE 五步法，把AI从"威胁"转化为"增强"。

**三种心理需求的威胁与增强**

| 需求 | AI 如何增强 | AI 如何威胁 |
|------|------------|------------|
| **胜任力** | 帮新手完成高技能任务，帮专家拓展能力边界 | 自动化常规任务 → 技能退化焦虑；取代入门级岗位 → "年轻人怎么积累经验？" |
| **自主性** | 自动化重复枯燥任务，释放认知负担 | "算法牢笼"限制员工调整空间；员工沦为技术配角，丧失职业认同 |
| **联结感** | 省下精力投入人际互动；AI也能提供类人沟通体验 | 自动化人际协作任务 → 孤独感、凝聚力下降；代际分裂（老员工觉得年轻人滥用，年轻人觉得老员工不愿变） |

**AWARE 五步法**

| 步骤 | 核心动作 | 典型做法 |
|------|---------|---------|
| **A**cknowledge 承认 | 主动沟通，承认AI可能带来的心理威胁，营造心理安全感 | 多邻国CEO在"AI优先"战略引发焦虑后公开承认失误 |
| **W**atch 观察 | 识别员工的适应性/非适应性应对行为 | GitHub监测Copilot使用指标；警惕"影子AI"（54%员工未经批准使用AI） |
| **A**lign 契合 | 将培训、指导、反馈与心理需求对齐 | 普华永道"My AI"（试验场+提示词派对+AI大使）；花旗AI先锋计划 |
| **R**edesign 重新设计 | 与员工合作重新设计岗位，实现人机互补 | 莫德纳合并技术与HR为"人才与数字技术"部门；戴尔先简化流程再引入AI |
| **E**mpower 赋能 | 透明沟通 + 让员工参与AI部署决策 + 全员开放访问 | 纽约梅隆银行全员开放AI平台，60%员工自主注册 |

**关键数据**
- 45% CEO认为多数员工抵触AI（勤达睿 2025）；85%高层 vs 51%基层经常使用AI（BCG 2025）
- 31%知识工作者承认刻意抵制公司AI项目；仅36%员工认为受过充分培训

### ③ 我的思考

**核心判断：AI是威胁还是助手，不取决于技术，取决于组织对"人"的假设**

```
领导看重创造力 → AI释放时间 → 员工做更有价值的事 → 胜任力↑ 自主性↑ 联结感↑
领导不看重创造力 → AI替代人力 → 裁员/边缘化 → 三种需求全面崩溃 → 抵触
```

同一个AI工具，因组织文化假设不同，走向完全相反的路径。这给培训方案设计带来的启示是：**在推AI相关培训之前，先判断客户对"人"的底层假设——是"省成本的工具"还是"长能力的伙伴"。**

**央企的现实：有壳无芯**

央企数字化基建已经铺开（系统有了、数据有了），但AI应用停在浅层——给内容、测数据、自动化出题——这是**替代层**，不是**增强层**。AI没有帮员工发现自己的薄弱点、规划个性化提升路径，更没有帮领导看到团队能力图谱。

三层断裂：
```
个人层：员工自己用AI学 → 有效但零散
组织层：企业系统性用AI赋能 → 缺失
领导层：看不到AI对能力的长期价值 → 只盯短期效益 → 不投入
```

死循环：**个人零散用 → 对业务没显著影响 → 领导不重视 → 组织不投入 → 继续个人零散用。** 破局点在"组织化"这一步——没人敢先迈，因为短期看不到ROI。

**豆包冲击：一个真实的业务威胁**

客户领导从意识层已经在问：豆包都能练英语了，为什么还要组织培训班？这对培训机构的冲击是直接的。

但这里存在一个关键的混淆——**"对话机会"≠"能力系统"**：

```
豆包给的：陪练 → 能开口
培训给的：诊断场景短板 → 解决"不敢说"的心理障碍 → 在真实项目中验证 → 团队能力图谱 → 组织级提升
```

豆包替代的是"陪练"这个单一动作，覆盖不了诊断、心理、场景迁移、组织能力层。但客户领导不会自动理解这个区分——他看到的是功能替代，你得主动把系统价值讲清楚。这不是对抗AI替代，而是重新定义人在哪里不可替代——恰好就是 AWARE 里 Redesign 的思路。

**用AI的三个层次**

结合自己团队的实际观察，发现真正的问题不是"不会用工具"，而是**思考不足、机械使用**：

```
层1 替代层："AI帮我做" → 大脑闲置 → 能力退化（机械使用，当前主要问题）
层2 增强层："AI帮我想" → 大脑参与 → 能力维持
层3 升维层："AI让我想到原本想不到的" → 大脑升级 → 能力跃迁
```

这与 B002（SOP规范化思考）、B011（目标优先）的底层逻辑一致：**工具是下游，思考框架是上游。** 上游不通，下游给再好的工具也用成机械操作。培训方案中如果只教"怎么用AI工具"而不教"怎么用AI思考"，就是在上游放弃。

**"省成本"vs"长能力"——两条路的分水岭**

把AI当作省成本工具 → 员工安全感崩塌 → AI落地=离职风险上升 → 隐性抵制
把AI当作长能力伙伴 → 需要把"个人学习层"升维到"组织能力层" → 但这需要领导看到长期价值

当前困局：企业更看重短期效益。"人强则公司强"这个道理领导不是不认，是"AI培训投入→业务能力提升→可量化业务结果"这个链路的中间环节没被讲清楚。谁能把这个链路补上，谁就拿下了AI培训的入口。

**破局：把AI当成"下属"来带**

这里有一个看似矛盾的痛点：

```
能力不足 → 不需要做复杂思考 → 没机会练习 → 能力更不足
公司培训时间又不够 → 没法靠"教"来补
```

但换个角度看，AI恰好解这个死锁。**不是教员工"怎么用AI"，而是让他们把AI当成自己的下属来管理。** 管理AI的过程，本身就是在练思维能力：

| 管理动作 | 对应的思维能力 | 为什么"悟"得出来 |
|---------|--------------|----------------|
| 给AI下达清晰的任务指令 | 目标拆解 + 优先级判断 | 指令模糊→AI产出垃圾→痛了才改 |
| 审查AI的产出，判断哪些能用哪些不行 | 质量标准 + 批判性思维 | 不审就交→被领导打回来→下次就知道审 |
| 给AI反馈，让它迭代改进 | 沟通能力 + 过程管理 | 改三轮还不对→反思自己哪里没说清楚 |
| 决定什么时候用AI、什么时候自己上 | 判断力 + 资源分配 | AI搞砸了一次→下次知道边界在哪 |

这套逻辑和 B018（轻试行+带人）完全同构——带Andy的方式，平移过来就是带AI的方式。**能力的提升不靠"教了多少"，靠"实践了多少、悟出了多少"。** AI不是培训的内容，AI是培训的"沙袋"——绑着它练，能力自然长。

这对央企培训方案的关键启示：**不要把"AI工具课"当成一个独立模块塞进方案里。而是把"带AI"设计成学员的学习方式本身。** 比如，不是在课上教"怎么写邮件"，而是让学员带一个AI助手完成一个真实的海外项目沟通任务，老师在旁边点评的不是AI用得好不好，而是"你对AI的指令体现了你对任务的理解到了什么程度"。

**Redesign：AWARE 的真正支点**

回看 AWARE 五步，权重其实是不等的。Acknowledge（承认）、Watch（观察）、Align（契合）、Empower（赋能）——这四个是**能力问题**，学会了就能做。但 Redesign（重新设计）是**意识问题**。

**意识不到"工作本身需要重新设计"，后面所有动作都只是在旧框架里修修补补。** 比如给员工开几场AI培训课，但岗位职责不变、考核方式不变、工作流程不变——培训完了员工回去还是按老样子干活，AI用不起来，然后领导说"你看，投了培训也没用"。

Redesign 要回答的根本问题是：**有了AI之后，这个岗位上的人，什么交给AI做、什么必须人做、两者怎么协作？** 这个问题的答案不是技术问题，是对"人的价值在哪里"的组织共识。没有这个共识，AI引入注定停在替代层。

### ④ 方向

写方案、带团队、做培训设计时，一个自检就够了：**AI在这个场景里是"省成本的工具"还是"长能力的伙伴"？** 选后者，具体做法会自己长出来。

## 方法论启示

这次共创的实际过程是：**用自己的 B卡体系消化一个外部框架，然后产出比原文更贴近业务场景的思维工具。** 这不是读书笔记，是方法论嫁接。

几个可复用的做法：
1. 不要照搬外部框架，先翻译到自己的业务语境里
2. 外部框架的价值不在完整性，在找到那一个"支点"（比如五步里只盯 Redesign）
3. 最原创的洞察往往来自"把箭头反过来"（管理者带人接受AI → 每个人成为AI的管理者）
4. 行动项越少越有力量，一个自检问题胜过六条清单

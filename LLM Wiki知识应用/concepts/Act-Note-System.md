# ACT 笔记系统

> **行动 · 卡片 · 时间**
> 基于 Obsidian 的个人知识管理系统，融合 PARA + GTD + 卡片笔记法。

---

## 核心理念

ACT 围绕两个核心原则设计：

**1. 三层驱动**
| 维度 | 回答 | 对应文件夹 |
|------|------|-----------|
| 🎯 Action | 我要做什么 | 3-Action/ |
| 🧠 Card | 我知道什么 | 4-Card/ |
| 📅 Time | 我什么时候做 | 5-Time/ |
| 🗄️ Storage | 我存了什么 | 6-storage/ |

**2. 不用动脑**
- 任何想法先丢 1-收件箱，每周回顾时用 6 个灵魂拷问定位到对应位置
- 升级/降级/归档全靠拖拽文件夹，不依赖脚本
- 三步分层：Action（执行）→ Card（知识） → Time（时间），各自独立又通过字段双向链接

## 设计渊源

- **PARA 方法**（Tiago Forte）→ Projects/Areas/Resources/Archives 四层，ACT 改为了 Action/Card/Time/Storage
- **GTD**（David Allen）→ 收件箱 + 下一步行动 + 项目清单
- **卡片笔记法**（Niklas Luhmann）→ 索引卡/阅读卡/核心卡三层分离，双向链接
- **渐进式总结** → 阅读卡→核心卡→索引卡，层层提炼

---

## 系统结构

```
📥 1-收件箱/               → 临时暂存，每周清空
📌 3-Action/               → 我要做什么
  ├── 3-1-Focus/           当前聚焦项目（每个项目一个文件夹）
  ├── 3-2-Active/          正在跟进（单条任务，流动区）
  └── 3-3-Maybe/           将来也许（有期限的想法储备）
🧠 4-Card/                  → 我知道什么
  ├── 4-1-IndexCard/       主题索引（K编号-关键词.md）
  │   ├── Person/          人物索引
  │   └── Topic/           主题索引
  ├── 4-2-BibCard/         阅读笔记（作者-书名.md）
  └── 4-3-MainCard/        核心知识（B编号-关键词.md）
📅 5-Time/                  → 我什么时候做
  ├── 5-1-Vision/          长期愿景
  ├── 5-2-Monthly/         月度计划（YYYY-MM-月计划.md）
  ├── 5-3-Weekly/          周计划（YYYY-Wxx.md）
  └── 5-4-Daily/           日日志（YYYY-MM-DD（周几）.md）
🗄️ 6-Storage/              → 我存了什么
  ├── 6-1-archive/         归档（月度计划/自动，项目归档/手动，学习资料/手动）
  ├── 6-2-template/        模板（16个模板全放这里）
  ├── 6-3-picture/         图片
  └── 6-4-base/            数据视图（Dataview查询脚本）
```

## 每个文件夹的规则

**1-收件箱**：一条=一个想法。用随心记模板。**每周回顾清空**，不留过周垃圾。
- 判断归属：要马上去做 → 3-1-Focus/或3-2-Active/；将来做 → 3-3-Maybe/；记知识 → 4-3-MainCard/；读书有感 → 4-2-BibCard/；今天执行 → 5-4-Daily/

**3-1-Focus（项目）**：项目=有明确开始/结束、可验证完成标准、跨多月的任务集合。格式：`项目名-项目/` 文件夹，内含 1 张项目卡片 + N 张任务笔记。

**3-2-Active（活跃跟进）**：流动区。每条最终走向 Focus、Maybe 或归档。

**3-3-Maybe（将来也许）**：命名前缀 `想法-`、`待研究-`、`感兴趣-`、`已调研-`。每条必须有**重启条件**和期限，过期清理。

**4-1-IndexCard（索引卡）**：`K编号-关键词.md`，编号手动指定**永久不变**。分 Person/ 和 Topic/。

**4-2-BibCard（阅读卡）**：`作者-书名.md`。工作流：读到材料→BibCard记录来源→提炼观点到MainCard→IndexCard聚合链接。

**4-3-MainCard（核心卡）**：`B编号-关键词.md`。**一张卡只说一件事**。用自己的话思考，不要粘贴原文。编号永久不变。

**5-Time**：层级驱动链：愿景→月度目标→周计划→日执行。Monthly 命名 `YYYY-MM-月计划.md`，次月1日自动归档。Weekly 命名 `YYYY-Wxx.md`，每周日20:00自动复盘。Daily 命名 `YYYY-MM-DD（周几）.md`。

**6-Storage**：归档只保留对将来有参考价值的内容。

## 工作流

1. 任何想法→随手记入**1-收件箱**（随心记模板）
2. **每周回顾**逐条处理→分入 3-Action/4-Card/5-Time
3. 每天结束→写日日志
4. 每周日→更新周计划
5. 每月初→制定月度计划，月末回顾

## 项目卡片

**YAML 字段**：
```yaml
status: active   # active | paused | completed | cancelled
month: 2026-05   # 关联月度计划，YYYY-MM
vision:          # 可选：关联愿景目标
```

**status 生命周期**：`active → paused → active`（恢复）/ `active → completed`（归档）/ `active → cancelled`（归档）

**内部结构（顺序）**：
1. 项目背景
2. **关键成果**：勾选框清单，每个勾=一个可验证完成标准
3. **任务清单**：表格 | 任务 | 状态 | 优先级 | 截止日期 | 关联月度
4. **时间轴**：表格 | 里程碑 | 截止日期 | 状态
5. **关联卡片**：MainCard 链接
6. **进展日志**：日期 | 进展 | 卡点
7. **归档说明**：完成时间 + 关键产出摘要（项目结束时填写）

## 任务笔记

**YAML 字段**：
```yaml
project: 项目名   # 所属项目，与项目卡片文件夹名对应
month: 2026-05   # YYYY-MM
priority: H       # H | M | L
status: active    # active | paused | done | cancelled
```

**内部结构**：
1. 任务背景（从哪来/谁提的/为什么做）
2. 所属项目：`项目名-项目卡片`（创建笔记时替换为实际项目卡片链接）
3. 任务成果
4. 相关资料（用自己的话概括）
5. 行动清单（勾选框）
6. 进展记录（日期 | 进展 | 卡点）

## 项目→任务→时间连接机制

```
项目卡片（3-1-Focus/项目名-项目/）
├── month: "2026-05"  ──→ 月度计划（5-2-Monthly/2026-05-月计划.md）
│                              └── projects: ["项目卡片链接"]
├── 任务笔记
│   ├── project: "项目名"  → 反向链接到项目卡片
│   ├── month: "2026-05"  → 关联月度计划
│   └── status: active     → 任务生命周期
├── 周计划（5-3-Weekly）  → 按周拆解项目任务
└── 日日志（5-4-Daily）   → 每日执行记录
```

关键规则：
- 项目卡片 `month` = 当前聚焦月份，跨月项目每月更新此值
- 任务笔记 `project` = 所属项目文件夹名，精确匹配
- 任务笔记 `month` = 计划完成月份
- 月度计划 `projects` = 该月所有活跃项目链接列表
- 周计划通过查询 `project: "X" AND month: "当前月"` 的任务笔记生成周任务清单

## 月度计划周次对齐追踪表

|------|-----------|-----------|-----------|
| W1（月第1周） | [ ] 未启动 / [x] 进行中 | — | — |
| W2 | — | — | — |

## 模板体系

16个模板全部放在 `6-2-template/` 下：随心记模板、行动卡片模板、时间记录模板、日计划模板、周计划模板、月度计划模板、月度回顾模板、任务笔记模板、项目卡片模板、活跃跟进模板、将来也许模板、核心卡模板、阅读卡模板、知识卡片模板、人物索引模板、主题索引模板。

## 自动化设置

**1. Templater 模板文件夹**
```
Obsidian → 设置 → 社区插件 → Templater → 模板文件夹位置
值：6-Storage/6-2-template/
触发器：Ctrl+P → Insert Template
```

**2. 每周日复盘提醒**
```
cron: 0 20 * * 0
动作：打开 5-3-Weekly/YYYY-Wxx.md
流程：检查本周所有 active 任务状态 → 更新周计划 → 未完成的任务移到下周或标记 paused
```

**3. 每月1日归档月度计划**
```
cron: 0 6 1 * *
动作：将 5-2-Monthly/ 中上月 YYYY-MM-月计划.md 移至 6-1-archive/月度计划/
      自动创建当月月度计划文件（模板预填）
```

**4. 项目完成归档**
```
触发：项目卡片 status → completed 或 cancelled
动作：填写归档说明 → 将 项目名-项目/ 文件夹移至 6-1-archive/项目归档/
```

**5. Dataview 查询脚本目录**
```
路径：6-4-base/ 下放 Dataview 查询文件
示例：当前项目总览.md（status:active）、本月任务看板.md（month:"当前月"）、
      本周待办.md（status:active 按 priority 排序）
```

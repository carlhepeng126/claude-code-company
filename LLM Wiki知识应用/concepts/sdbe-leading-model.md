# SDBE领先模型

## 框架概述

SDBE领先模型（Strategy-Decoding-Business-Execution Leading Model）是华为前高管胡荣丰在IBM BLM（业务领导力模型）基础上，结合华为十余年战略管理实战经验（DSTE流程）提炼而成的战略管理闭环框架。相比BLM，SDBE更注重中国企业适应性和实际操作层面的可落地性。

### 核心闭环：战略→解码→计划→执行→复盘

```
定标杆 → 找差距 → 业务设计 → 战略规划 → 战略解码 → 
经营计划(KPI+关键举措) → 执行管理 → 复盘迭代 → (回到差距)
```

## SDBE领先模型的六大工作过程

### 1. 定标杆，找差距
战略管理的起点。只有找到合适的标杆后，制定的战略才更有针对性和时效性。无标杆则设立理想标杆（假想标杆）。

**华为实践**：华为进入每个行业第一步就是谦虚寻找标杆——数据通信对标思科、无线对标爱立信、手机对标三星/苹果、云计算对标亚马逊/阿里——进行全面对标和"像素级差距分析"。

### 2. 业务设计，制定战略
根据与标杆的差距及战略目标，审慎进行业务设计，制定公司战略（3-5年SP）。

**核心要素**：
- 明确企业愿景和使命
- 制定重大策略和措施（发展路线）
- 界定阶段目标和业务范围
- 牵引和配置资源分配

### 3. 战略解码，分解打通
将短期战略与中期战略逐一分解，打通各个关键点，解码成可执行、可衡量的阶段性措施和目标。

> BLM的致命弱点是缺乏解码环节——SDBE弥补了这一关键缺口。

### 4. 确定商业计划，明确KPI和关键举措
开发可有效衡量阶段性目标和措施的指标体系，明确关键举措并落实到部门（年度BP）。

### 5. 执行管理，检视阶段性商业成果
对阶段性执行成果进行检视，确保战略管理得到有效执行。华为采用"年初规划→中期审视→年末复盘"的强制管理动作。

### 6. 战略复盘，迭代改进
定期对战略进行复盘，结合内外部环境变化及业绩差异，明确目标与成果之间的差距，迭代优化。

## SDBE vs BLM 关键差异

| 维度 | BLM | SDBE领先模型 |
|------|-----|-------------|
| 落地性 | 飘浮空中，缺乏操作工具 | 提供实操表格/工具，可落地 |
| 解码环节 | 缺失 | 新增"战略解码"关键环节 |
| 标杆管理 | 提及但未系统化 | 核心方法论，"差距驱动" |
| 闭环机制 | 未明确 | 强制年中审视+年末复盘 |
| 兼容性 | 仅适用管理强的公司 | 更适合中国企业实战 |
| 工具配套 | 无操作工具 | 差距分析、KPI分解、TOP N等 |

## BLM的"1+4+8"模型（SDBE的演进基础）

- **1** — 差距（业绩差距 + 机会差距）
- **4** — 战略、执行、领导力、价值观四大模块
- **8** — 市场洞察、战略意图、创新焦点、业务设计、关键任务、正式组织、人才管理、氛围与文化

## 华为战略管理的四大原则

1. **战略管理是企业领导人的专属职责** — 不能授权，一把手必须躬身入局
2. **标杆+差距是规划和执行的导向** — 差距是起点、终点，也是动力
3. **战略必须与执行结合** — 落脚点甚至全部意义在于执行
4. **战略是动态过程** — 非一次性结果，持续复盘迭代

## 华为战略管理的发展三阶段

| 阶段 | 特征 | 典型企业状态 |
|------|------|-------------|
| 第一阶段 | 创始人个人直觉驱动，成败系于老板一身 | 创业初期，试错调整 |
| 第二阶段 | 高管团队集体研讨，有战略文件但落地模糊 | 国内多数规模型企业 |
| 第三阶段 | 精心设计→解码→计划→执行→复盘闭环 | 华为当前状态 |

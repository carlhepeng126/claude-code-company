# 情报专家 · 央企培训十维情报体系

## 定位

情报专家（intel-expert）是摩根 Agent 体系中的**前置情报层**。在 diagnose-client（诊断）和 consult-client（咨询方案）介入之前，情报专家基于十维交叉数据回答三个问题：

1. **这家央企有没有培训机会？**（五级分类：饱和区→绝对空白）
2. **竞品已经做到什么程度了？**（商业竞品+高校+行业中心全景）
3. **建议从哪个角度切入？**（差异化路径+风险提示+时机建议）

## 十维数据源

| 维 | 数据源 | 视角 |
|:--:|------|------|
| 1 | 100家央企海外布局及培训需求（103份报告） | 需求端：培训缺口 |
| 2 | 培训案例库（88个独立案例） | 活动端：已有培训行为 |
| 3 | 央企近三年语言培训调研报告（50+招标数据） | 采购端：预算与供应商 |
| 4 | 竞品公众号（12个·~1,800篇） | 竞品端：谁在服务谁 |
| 5 | 培训机构反推分析（31条高可信映射） | 供给端：真实客户关系 |
| 6 | 行业媒体（培训杂志17期+HR人力资源26期） | 话语体系：行业趋势与案例 |
| 7 | 价格体系与续约逻辑分析 | 定价端：锚点与续约规律 |
| 8 | 涉外培训数字化趋势分析 | 技术端：AI替代与OMO |
| 9 | 涉外培训直接竞品深度画像（10家） | 竞品深度：威胁与弱点 |
| 10 | 央企涉外培训政策与采购周期分析 | 时机端：政策驱动+采购时钟 |

## 核心方法论：六维对齐评分

对任意央企，逐维检查 → 五级分类 → 竞品态势 → 切入建议。

### 五级分类

| 级别 | 特征 | 行动 |
|------|------|------|
| 🔴 饱和区 | 十维全覆盖，多竞品+多高校 | 差异化细分切入 |
| 🟠 大学主导区 | 高校为主供应商 | 做高校做不到的实战层 |
| 🟡 招标活跃区 | 有招标+供应商稀疏 | **重点进攻** |
| 🟢 信息盲区 | 仅研究侧覆盖 | 主动触达确认 |
| ⚪ 绝对空白 | 十维全空 | **最大蓝海** |

### 信号优先级

```
招标信号(已确认预算) > 供应商映射(已确认关系) > 行业媒体案例 > 竞品声量 > 公开案例 > 研究推断
```

### 采购时钟

| 时段 | 动作 |
|------|------|
| Q4 (10-12月) | ⭐⭐⭐⭐⭐ 黄金窗口——对接需求+争取纳入预算 |
| Q1 (1-3月) | ⭐⭐⭐⭐ 预算生效+招标启动 |
| Q2-Q3 (4-9月) | ⭐⭐⭐ 主动拜访+预埋下年需求 |

## 竞品全景图

| 竞品 | 类型 | 核心央企客户 | 可攻击弱点 |
|------|------|------|------|
| 中智集团 | 央企·900亿 | 中石油/石化/建筑/铁建 | 决策慢·创新保守·语言非核心 |

| 北外/西外/川外 | 高校 | 铁建/化学/中广核 | 学术偏重·灵活差·周期固定 |

## 与其他Agent的协作

```
orchestrator（编排器）
  ├── 检测情报查询信号
  ├── → intel-expert（情报专家）← 前置情报层
  │     输出：十维情报卡
  ├── → diagnose-client（诊断/方案）
  │     基于情报卡做差异化方案
  └── → consult-client（管理咨询）
        基于情报卡做咨询方案
```

## 自动化情报更新

### 每周一定时扫描（scheduled_tasks.json）
- **触发时间**：每周一 9:17 AM
- **扫描目录**：`收件箱：同步助手/企业出海日报/` + `收件箱：需求模拟/`
- **更新目标**：entities/央企培训情报关联.md、index.md、raw/情报信息/

### 会话结束提醒（Stop Hook）
- **触发时机**：每次会话结束
- **检查内容**：收件箱是否有新的未处理文件
- **提醒方式**：终端输出提醒消息

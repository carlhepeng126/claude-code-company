# ACT 笔记系统 — 个人知识管理方法论

> **行动 · 卡片 · 时间**
> 一个无需动脑、拖拽即用的 PARA/GTD 混合笔记体系，落地在 Obsidian 单仓库中。

---

## 核心理念

ACT 笔记围绕三个维度展开：

| 维度 | 英文 | 回答的问题 |
|------|------|-----------|
| 🎯 **Action** | 执行层 | 我要做什么 |
| 🧠 **Card** | 知识层 | 我知道什么 |
| 📅 **Time** | 时间层 | 我什么时候做 |
| 🗄️ **Storage** | 资源层 | 我存了什么 |

**设计原则：**

1. **不用动脑** — 所有想法先丢进收件箱，回顾时问6个灵魂拷问直接定位，无需记忆规则
2. **拖拽即用** — 升级/降级/归档全靠拖拽文件夹，不接受需要额外配置的流程
3. **三层联动** — Focus 项目 ↔ 任务笔记 ↔ 月度计划 通过 YAML 字段双向链接
4. **自动善后** — 月度自动归档、周日自动复盘，用户只负责记和执行

## 系统结构 (1-6编号体系)

```
ACT笔记-深度思考/
├── 1-收件箱/       📥 捕获层     — 任何闪念先丢进来
├── 2-索引/          🧭 导航层     — 结构总览、快速跳转
├── 3-Action/       🎯 执行层     — 我要做什么
│   ├── 3-1-Focus/     当前聚焦的1-3个项目
│   ├── 3-2-Active/    正在推进但不紧急
│   └── 3-3-Maybe/     "将来也许"的想法储备
├── 4-Card/         🧠 知识层     — 我知道什么
│   ├── 4-1-IndexCard/ 主题索引（K编号）
│   ├── 4-2-BibCard/   阅读笔记（外源信息）
│   └── 4-3-MainCard/  核心知识卡片
├── 5-Time/         📅 时间层     — 我什么时候做
│   ├── 5-1-Vision/    长期愿景与方向
│   ├── 5-2-Monthly/   月度计划（次月1日自动归档）
│   ├── 5-3-Weekly/    周计划（周日自动复盘）
│   └── 5-4-Daily/     日日志
└── 6-storage/      🗄️ 资源层     — 我存了什么
    ├── 6-1-archive/   归档（月度计划/项目归档/学习资料）
    ├── 6-2-template/  模板（16个模板）
    ├── 6-3-picture/   图片
    └── 6-4-base/      Obsidian Bases 数据视图
```

编号 1-6 按**直觉层级**排列：捕获→导航→执行→知识→时间→资源，看一眼就知道在哪个层。

## 工作流

### 日常捕捉流
```
1-收件箱 → 快速捕捉 → 每周回顾 → 分类到 Action/Card/Time → 归档
   ↑                                                     ↓
   └──────────────── 每周清空 ───────────────────────────┘
```

### 不用动脑 · 判断指南

| 回顾时问自己 | → 放哪里 | 用什么模板 |
|-------------|---------|-----------|
| 要马上去做？ | 3-1-Focus/ 或 3-2-Active/ | 行动卡片模板 |
| 以后可能想做？ | 3-3-Maybe/ | 将来也许模板 |
| 想记住这个知识？ | 4-3-MainCard/ | 核心卡模板 |
| 读书/文章有感？ | 4-2-BibCard/ | 阅读卡模板 |
| 今天做了啥？ | 5-4-Daily/ | 日计划模板 |
| 只是随便记一下？ | 1-收件箱/ | 随心记模板 |

### 项目生命周期
```
Maybe(3-3) → Active(3-2) → Focus(3-1) → 完成 → 6-1-archive/项目归档/
```

项目在 3-1-Focus 下以 `项目名-项目/` 子文件夹组织，内含：
- **项目卡片**（.md）— status 字段驱动生命周期：active → completed → archived
- **任务笔记**（.md）— project/month 字段关联到项目和月度计划

升级/降级/归档 = **拖拽文件夹**，不依赖任何脚本。

## 模板三级体系

| 等级     | 模板                     | 特点              |
| ------ | ---------------------- | --------------- |
| 🟢 不用想 | 随心记                    | 无字段、无结构，任何闪念直接写 |
| 🟡 想一点 | 日计划、任务笔记、行动卡片、时间记录     | 3-5个字段，填空即可     |
| 🔴 想清楚 | 项目卡片、核心卡、月度计划、阅读卡、月度回顾 | 完整结构，模板内嵌填写提示   |

---

## 自动化能力

| 功能 | 触发 | 行为 |
|------|------|------|
| 月度归档 | 每月1日 09:00 | 上月月度计划 → `6-1-archive/月度计划/` |
| 周日复盘 | 每周日 20:00 | 扫描本周笔记变动 → 归纳为推进/学到/下周计划 → 写入`5-3-Weekly/` |
| Obsidian 同步 | 持续（systemd 服务） | 服务器 ↔ 桌面端双向同步 |

## 设计渊源

- **PARA 方法**（Tiago Forte）— Projects/Areas/Resources/Archives 四层结构
- **GTD**（David Allen）— 收件箱 + 下一步行动 + 项目清单
- **卡片笔记法**（Niklas Luhmann）— 索引卡/阅读卡/核心卡分离，双向链接
- **渐进式总结** — 模板中内嵌摘要→提炼→关联三层

---

## 适用场景

- 个人知识管理（学习笔记 + 卡片积累）
- 项目管理（1-3个聚焦项目的全生命周期管理）
- 时间规划（愿景→月→周→日层层拆解）
- 信息整理（阅读笔记 + 主题索引 + 核心知识卡片）

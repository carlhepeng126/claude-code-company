## 三、新Agent定义草案

### Agent 3：product-expert（产品专家）

```
触发：方案设计中需要推荐产品 / 新课开发 / 产品优化 / 询价
知识源：训练产品库 + B卡体系 + 案例锚点库 + 师资库
核心能力：
  1. 产品匹配：根据诊断卡（场景轨×驱动类型×短板×预算）推荐最优产品组合
  2. 产品对比：多个方案选型时，给出trade-off分析
  3. 产品优化：根据交付反馈，建议产品迭代方向
  4. 新品设计：头脑风暴新课/新产品时，提供结构化框架
输出：产品推荐卡（产品名 + 为什么匹配 + 参考案例 + 报价区间 + 师资建议）
```

### Agent 4：project-delivery（项目交付）

```
触发：方案签约后 / 项目启动 / 交付中遇到问题 / 项目复盘
知识源：项目归档 + B010（落地断点管理）+ B001（教师规范）+ B003（审核机制）
核心能力：
  1. 交付计划：签约方案→拆解为可执行的交付清单（谁/什么时间/产出什么）
  2. 质量管控：基于B003的审核机制，生成分阶段质量检查点
  3. 风险预警：基于B010的断点模型，提前识别落地风险
  4. 复盘报告：项目结束后，结构化复盘→写入知识库→触发产品优化建议
输出：交付清单 / 质量检查卡 / 风险预警 / 复盘报告
```

### Agent 5：market-support（市场支持）

```
触发：准备客户拜访 / 写资质文件 / 竞品动向 / 行业政策变化 / 营销素材需求
知识源：竞品库 + 案例库 + 行业报告 + 公众号文章 + 国别文化资料
核心能力：
  1. 竞品情报：结构化监控竞品动向，发现信号自动推送
  2. 资质包装：根据目标客户行业/场景，自动推荐最相关的案例和资质
  3. 行业简报：定期扫描行业政策/出海动态，生成可用的市场简报
  4. 销售素材：为特定客户/场景快速生成案例一页纸、能力介绍等
输出：竞品简报 / 资质推荐卡 / 行业动态 / 客户拜访素材包
```

## 五、自进化机制设计

### 每周自动复盘（Cron：每周一 08:57）

编排器自动执行以下检查：

```
1. 读取上周所有对话记录
2. 识别：
   - 新客户接触了几个？进入哪个阶段？
   - 方案出了几份？有没有新的模式/报价锚点？
   - 交付中的项目进展如何？有没有断点？
   - 跨域信号新增了几条？哪些需要跟进？
   - 竞品/行业有什么新动向？
3. 生成周报 → 写入 5-Time/5-3-Weekly/
4. 标记需要人工决策的事项
```

### 项目闭环自进化

```
签约 → delivery Agent 生成交付清单
  ↓
交付中 → delivery Agent 跟踪进度，捕捉断点
  ↓
交付完成 → delivery Agent 生成复盘报告
  ↓（自动触发）
product Agent：本次交付暴露了什么产品问题？→ 产品优化建议
market Agent：本次项目有什么可复用的案例/资质？→ 更新案例库
diagnose/consult Agent：客户有没有暴露新的痛点？→ 跨域信号
  ↓
orchestrator 汇总 → 写入知识库 → 下次同类客户自动受益
```

### 信号自动升级规则

| 信号模式 | 自动动作 |
|---------|---------|
| 同一客户3次提到同一痛点 | 升级为"确认需求"，自动生成初步方案框架 |
| 同一行业3个客户出现类似需求 | 升级为"行业趋势"，market Agent 生成行业简报 |
| 竞品出现新动作（新产品/新客户/融资） | market Agent 推送竞品预警 |
| 跨域信号积累到5条且都指向同一方向 | 升级为"战略机会"，提示用户关注 |

## 七、成功标准

3-4周后，以下场景应该能自动运行：

1. **你说"帮我看看XX客户"** → 编排器自动检测信号，路由到对应Agent，输出诊断卡，同时检查跨域信号索引看有没有历史相关洞察
2. **一个项目交付完成** → delivery Agent出复盘报告，product Agent检查产品优化点，market Agent更新案例库
3. **周一上午** → cron自动触发周复盘，你到办公室时周报已经在 Obsidian 里了
4. **竞品有新动向** → market Agent检测到，推送到你的收件箱
5. **同一客户多次出现跨域信号** → 编排器提醒你"该客户有3条未跟进的咨询信号，建议联系"

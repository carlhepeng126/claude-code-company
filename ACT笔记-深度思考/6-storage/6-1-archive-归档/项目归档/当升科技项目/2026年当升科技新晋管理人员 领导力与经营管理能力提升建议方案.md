# 当升科技新晋管理人员领导力与经营管理能力提升方案

## 一、项目背景与需求理解

### 1.1 当升科技的业务扩张与管理挑战

当升科技正处于"业务快速国际化 + 国内产能多基地扩张"的关键阶段。海外芬兰基地预计2026H2投产，印尼、摩洛哥等新基地已在规划中；国内昆明新基地8月30日投产在即，攀枝花、常州基地同步运营。

昆明新厂投产需紧急调配约100名管理者，其中48名为新晋管理干部。这批人员面临三大共性挑战：
- **角色转型断层**：50%以上为本地基层人员，从"只管事"到"管人管事"，管理幅度从3-4人陡增至上百人
- **能力缺口显著**：缺乏系统的管理知识框架，目标分解、绩效面谈、跨部门协同等核心技能薄弱
- **业务压力叠加**：昆明8月30日投产在即，学习与生产的时间冲突尖锐

### 1.2 与当升科技战略框架的咬合

当升科技管理层已明确提出"领导力建设三年行动"（2026-2028），以**"干部四大职责、三大关键能力"**为指引。本轮培训项目是三年行动的首个落地抓手——聚焦一线新晋管理者，解决"从技术/业务骨干到管理者"的关键一跃，为后续分层领导力培养体系奠定基础。

### 1.3 摩根的前期积累

摩根团队在上一轮芬兰项目中完成了三项工作：在**常州基地**为即将外派的中方人员开展系统培训；讲师亲赴**芬兰科特卡**实地调研外派人员工作环境与中芬协作场景；同时面向中方外派人员与芬兰本地员工开展**双向团队融合**。

这些一线经验直接影响了本轮方案的设计：

| 在常州与芬兰的真实发现 | 本轮方案如何回应 |
|---------|----------------|
| 中方外派人员与芬兰同事存在社交边界差异——中方习惯下班后社交谈工作，芬兰人偏好安静、下班即回归私人时间 | 第三阶段**跨文化沟通**模块：基于实地观察的双向文化差异，提供芬兰工作文化操作清单，课程案例注入中芬双视角 |
| 会议中中方人员理解度约70-80%，芬兰英语口音叠加Teams翻译失真，双向信息衰减 | 第一阶段**高效沟通**模块：以芬兰采集的真实会议场景为素材，设计多口音听力适应训练和结构化确认话术 |
| 外派管理者从国内管理方式直接切换到国际化协作，缺少过渡；芬兰本地员工对中方指令风格也有不适应 | 第一阶段**情景领导力**模块：专设"从指令型到教练型"转型路径，场景取材于当升外派管理者的真实困境 |
| 中芬双方对SOP、ESG、供应链管理的理解和执行标准存在理念差异 | 第二阶段**跨部门协同**和第三阶段**企业经营沙盘**模块：引入标准化流程沟通与全局经营视角 |
| 培训后员工反馈"课堂上懂了，回到产线上还是不会用" | 第十三节**培训闭环**机制：每模块配课后实践任务+直线上级签字验收——"陪跑比上课更重要"的方法论来自芬兰项目的直接验证 |

## 二、培养对象

当升科技新晋管理人员及管理骨干，**48人**。聚焦一线新晋管理者——先解决昆明投产最紧急的能力缺口，长期体系化建设后续推进。

**分班策略**：每班最佳规模为30人以内，48人分为两个班（各24人），同一课程模块分两次交付。既保证研讨深度和教学效果，也方便客户协调人员离岗，解决生产与学习的冲突。

---

## 三、培训目标

帮助新晋管理者完成从"做事"到"带人做事"的角色转型，建立管理知识框架，掌握**带团队、定目标、追绩效、促协同**四项核心管理能力。每阶段产出可落地的工作改进方案，培训成果直接映射到昆明投产的业务场景。

项目周期12个月，分三个阶段，线上线下融合培养。培训节奏与昆明8月30日投产及后续半年试运行的关键业务里程碑绑定。

## 四、新晋管理者岗位能力胜任度模型

基于当升科技"干部四大职责、三大关键能力"战略框架，结合一线新晋管理者的实际工作场景，提炼以下岗位能力胜任标准。本模型是课程设计的底层逻辑——每个课程模块反向对应一项或多项胜任力。

| 能力维度 | 胜任力项 | 行为标准（新晋管理者级别） | 对应四大职责 |
|---------|---------|------------------------|------------|
| **管理自我** | 角色认知 | 理解管理者与业务骨干的本质区别；能从"自己做"切换到"通过他人完成" | 建组织 |
| | 时间效能 | 能区分紧急与重要；合理分配管理时间，告别"忙茫盲" | 做业绩 |
| **管理团队** | 目标分解 | 能将部门目标拆解为可执行的团队任务，明确责任人、时间节点、验收标准 | 做业绩 |
| | 知人善任 | 了解团队成员的能力特点与短板；能进行基本的人岗匹配和任务分配 | 带队伍 |
| | 绩效辅导 | 能开展有效的绩效面谈（目标设定→过程反馈→结果评估）；不流于形式 | 做业绩 |
| | 有效授权 | 能区分"必须自己做"与"可以授权"的工作；授权时明确边界和检查点 | 带队伍 |
| | 团队激励 | 能运用非薪酬手段激发团队积极性；识别成员发展诉求 | 带队伍 |
| **管理协作** | 高效沟通 | 能进行向上汇报（结构化）、平级协调（利益共同点）、向下反馈（行为+影响） | 搭流程 |
| | 跨部门协同 | 能识别关键利益相关方；用"共同目标"而非"人情"推动协作 | 搭流程 |
| | 冲突化解 | 能区分任务冲突与关系冲突；运用基本调解方法化解团队内部矛盾 | 搭流程 |
| **管理业务** | 高效会议 | 能组织召开有明确议程、产出和跟进行动的高效会议 | 搭流程 |
| | 商业敏感度 | 能读懂基本财务报表（P&L/现金流）；理解自身决策的成本影响 | 做业绩 |
| | 问题解决 | 能运用基本分析工具（5Why/鱼骨图/SWOT）定位问题根因并制定对策 | 做业绩 |

> 设计原则：本模型聚焦**一线新晋管理者**的"最小必要能力集"——不是理想管理者的完整画像，而是从业务骨干到合格管理者的第一级台阶。后续中高层领导力项目将在此基础上递进深化。
>
> 本模型与下文的**三阶段·三大能力培养框架**构成"经纬双线"——胜任度模型是"经线"（纵向的能力标准定义），三阶段框架是"纬线"（横向的培养路径设计）。13项胜任力按"基础→应用→进阶"的节奏分配到三个阶段中：破局期聚焦角色认知、目标分解、高效沟通等**基础生存技能**；攻坚期深化绩效辅导、冲突化解、跨部门协同等**实战应用能力**；收官期升华团队领导、影响力、经营思维等**整合进阶素养**。经纬交织，确保每个阶段的能力输入有明确的胜任标准可检验。

| 潜在挑战 | 针对性设计 | 验收方式 |

## 七、能力培养框架

### 7.1 胜任度模型与三大能力的"经纬双线"

上文**岗位能力胜任度模型**（第四章）定义了13项胜任力的行为标准——这是"经线"，回答"新晋管理者需要具备什么能力"。本节的三阶段框架是"纬线"，回答"这些能力按什么节奏培养"。

13项胜任力按"基础生存 → 实战应用 → 整合进阶"的节奏分配到三个阶段，每个阶段的能力输入均有明确的胜任标准可检验。

### 7.2 三阶段·三大能力·递进路径

本项目以**管理能力 · 综合素养 · 经营能力**三大维度为贯穿主线，三个阶段不是三个独立模块，而是一条递进链：

```
破局（角色转型）         攻坚（深化应用）          收官（整合进阶）
  建立管理框架      →     推动绩效与协同      →     领导力升华与固化
   "学工具、建习惯"         "用工具、解难题"          "教别人、建体系"
```

| 阶段 | 管理能力 | 综合素养 | 经营能力 |
|------|---------|---------|---------|
| **破局**（角色转型） | 角色认知、目标计划、知人善任、情景领导、**绩效管理** | 高效沟通、有效授权 | 商业敏感度基础 |
| **攻坚**（深化应用） | 绩效辅导、问题分析与解决、复盘初阶 | 冲突化解、跨部门协同 | 成本意识与报表解读 |
| **收官**（整合进阶） | 管理复盘与经验萃取、打造高绩效团队 | 影响力与团队协作、高效汇报与呈现、**跨文化沟通** | **企业经营沙盘模拟** |

### 7.3 三个阶段如何咬合为一个有机整体

| 衔接维度 | 具体设计 |
|---------|---------|
| **课题贯穿** | 同一课题从破局开题 → 攻坚中期汇报 → 收官结题述职，三阶段持续深化，12个月不换题 |
| **能力叠加** | 破局学的"绩效管理基础"在攻坚升级为"绩效辅导实战"；破局学的"高效沟通"在收官升级为"高效汇报与呈现"——同一能力线纵向递进 |
| **复盘接力** | 破局产出个人转型计划 → 攻坚用ORID复盘执行偏差 → 收官用经验萃取出个人管理方法论，三次复盘层层深入 |
| **业务绑定** | 三个阶段的时间节点（5月/8月/12月）与昆明投产（8.30）+半年试运行的关键里程碑自然咬合，每个阶段的学习成果直接投射到当期业务挑战 |
| **学分累积** | 三阶段学分贯通计算，年度总排名决定优秀学员，考评结果纳入晋升参考——学员无法"阶段性的努力一下"，必须全程投入 |

> 课程设计逻辑：不是用天数框内容，而是从一线新晋管理者的能力缺口出发，确定每个阶段的核心任务，再反推需要的课程和时长。第一阶段集中解决"角色转型+基础管理技能"，第二阶段深化"推动绩效+跨部门协同"，第三阶段实现"复盘固化+领导力进阶+全球化视野"。

## 八、三阶段课程设计

### （一）各阶段重点安排

| 阶段 | 线下 | 复盘/实践 | 核心任务 | 关键产出 |
|------|------|-----------|---------|---------|
| 第一阶段（破局） | 7天 | — | 建立管理框架，完成角色转型，掌握目标设定与绩效管理基础 | 开题报告、个人转型计划、首轮绩效面谈记录 |
| 第二阶段（攻坚） | 4天 | 3天 | 推动绩效落地，解决跨部门协同难题，运用分析工具解决实际业务问题 | 中期成果汇报、季度业绩回顾、跨部门协作改进案例 |
| 第三阶段（收官） | 5天 | 3天 | 复盘固化学习成果，提升团队领导力与全球化视野，通过沙盘建立经营全局观，向高管述职 | 结题成果、年度述职报告、管理实战案例 |

线上线下融合：线下集中授课与研讨，线上持续辅导贯穿12个月（月度工作坊、工具推送、名师诊断室、社群运营）。

**实战如何在三阶段中逐级强化**——

| 阶段 | 实战形式 | 实战产出 | 谁来检验 |
|------|---------|---------|---------|
| **破局** | 课堂角色扮演 + 课后实践任务（目标分解表、授权清单、绩效面谈） | 个人转型计划、首轮管理工具应用记录 | 直线上级签字确认 |
| **攻坚** | 真实跨部门案例复盘 + 问题分析工坊（用当升实际业务数据）+ ORID复盘实操 | 跨部门协作改进案例、季度业绩回顾 | 管理层评估打分 |
| **收官** | 经营沙盘推演（模拟产供销全链路决策）+ 面向高管述职 + 管理方法论萃取 | 结题成果、年度述职报告、个人管理方法论 | 当升高管评审团 |

> 实战不是课外作业，是课程本身。每个模块的"学"和"练"不分开——课堂上用当升科技的真实管理场景做案例，课后用学员自己团队的真实数据做实践，复盘时用季度业绩的真实结果做检验。学员不是在"参加培训"，而是在"解决自己工作中真正遇到的问题"，只是这个解决过程有方法论指导和专家陪跑。

| 48份团队能力盘点+授权清单 | 识别团队成员强弱项，明确授权边界 | 直线上级确认 |
| 首轮绩效面谈记录 | 每人至少完成1次正式绩效面谈的实战产出 | 月度应用反馈跟踪 |
| 6-8个课题开题报告 | 各组选题绑定昆明投产实际业务挑战 | 当升高管评审 |
| 《第一阶段学习评估简报》 | 摩根项目组汇总：出勤、测验成绩、学员反馈、典型实践案例 | 项目总监→当升人力→董事长 |

> 为什么这很重要：7天培训结束当天，HR手上就有实实在在的业务产出物可以拿到董事长办公桌上——不是"培训完了"，而是"48个管理者已经开始用新工具做管理了"。

### （三）第二阶段：攻坚（4天课程 + 3天复盘实践）

核心目标：推动绩效落地，解决跨部门协同难题。

**1、4天课程**（根据第一阶段学员应用表现微调，课纲启动前15天确认）：

| 课程主题 | 模块 | 课时 | 说明 |
|---------|------|------|------|

## 十、培训方式

### 10.1 设计原则：70-20-10 领导力发展模型

本方案遵循全球领导力发展领域公认的**70-20-10模型**（McCall, Lombardo & Eichinger, CCL），将学习活动按效果权重分配：

```
10% 正式学习          20% 向他人学习         70% 实践历练
（线下课程+线上工坊）   （导师+上级+同伴）      （真实业务课题+岗位实践）
     ↓                    ↓                    ↓
 知识输入              反馈与矫正             能力内化
```

| 学习层次 | 占比 | 本方案对应设计 |
|---------|------|--------------|
| **正式学习** | 10% | 三阶段线下集中面授（16天：破局7天+攻坚4天+收官5天）、每月线上主题工作坊（12次）、管理工具推送（12次） |
| **向他人学习** | 20% | 直线上级担任实践验收人、内部高管担任课题导师、名师诊断室每月答疑、小组互评与复盘分享会、双班主任全程陪伴 |
| **实践历练** | 70% | 行动学习课题（贯穿12个月）、每模块课后实践任务、季度业绩回顾、管理实战案例提交（≥6次）、面向高管的结题述职 |

> 行业共识：管理者能力提升70%来自真实工作挑战中的刻意实践。本方案不以"上完课"为交付标准，而以"学员在岗位上做出可检验的行为改变"为目标。

### 10.2 四步闭环教学法

遵循"学习、实践、反思"的人才培养理念，按照以下四个步骤闭环培养：

**1、预习探疑** — 每阶段开始前，学员按预习计划完成指定阅读材料和管理工具自学，结合自身工作实践提出问题。项目组收集学员实际管理困惑反馈至讲师，确保授课内容与学员真实场景高度契合。

**2、实战教学** — 课堂采用案例分析、情境模拟、角色扮演、小组研讨等多种形式。每个课程模块均设置"实战演练"环节——学员结合自身管理场景现场演练，讲师即时点评。案例素材优先使用当升科技内部实际管理场景（如昆明投产中的跨部门协调、芬兰外派中的文化沟通等）。

**3、行动转化** — 每门课程结束后布置与工作直接挂钩的"课后实践任务"（如制定部门目标分解表、完成一次绩效面谈、绘制利益相关方地图等），学员一周内完成并由直线上级签字确认。摩根项目组跟踪应用情况。

**4、总结复盘** — 每阶段结束后，学员采用ORID（事实—感受—思考—行动）焦点呈现法进行结构化复盘，系统梳理学习收获、应用难点与改进方向，列出下一阶段的具体行动计划（含行动事项、完成时间、预期成果）。

## 十一、项目运营与品牌化设计

项目运营是体现差异化价值的核心——不只交付课程，更打造学习体验和成长品牌。

### 11.1 项目品牌化

- **项目品牌名称**：为本次培训项目设计专属名称（如"当升·砺剑计划"），增强学员参与感和荣誉感
- **董事长开篇寄语**：邀请当升科技董事长录制开篇视频或亲临开营寄语，让学员感受公司对项目与个人发展的高度重视
- **学习地图可视化**：设计清晰的三阶段学习地图，每完成一个模块解锁下一阶段，给予学员明确的成长路径预期

### 11.2 学分激励机制

| 考评项 | 权重 | 说明 |
|---|---|---|
| 课程测验 | 30% | 每门课后线上测验 |
| 课题成果 | 40% | 开题10% + 中期20% + 结题10% |
| 业绩回顾 | 20% | 季度业绩回顾（10%）+ 年度述职（10%） |
| 过程表现 | 10% | 出勤、线上参与、工具提交等 |

- 综合成绩前30%的学员评为"优秀学员"，同时评选优秀课题、优秀班委
- 培训考核结果**纳入公司晋升与人才盘点参考**，建立透明公平的积分规则
- 优秀实践案例入选《管理实战案例集》，作为当升科技内部管理经验沉淀

### 11.3 双班主任管理

建立双班主任协同管理机制——当升科技HR侧重指导、决策与协调；摩根班主任负责现场日常管理、教学组织与学习跟踪。培训期间实行封闭管理，设置手机停机坪保证学习专注度。

## 十二、线上线下融合培养机制

以"线下集中突破、线上持续渗透、行动学习贯穿"为原则，形成12个月不间断成长闭环。

| 环节 | 线下 | 线上 |
|------|------|------|
| **核心授课与研讨** | 每阶段集中面授（破局7天/攻坚4天/收官5天） | — |
| **预习与自学** | — | 课前阅读材料、管理工具预习、在线微课 |
| **月度主题工作坊**（12次×1.5h） | — | 目标复盘、工具精讲、绩效面谈模拟、述职辅导 |
| **月度工具推送**（12次） | — | SWOT/PDCA/ORID/STAR等工具模板，学员提交应用记录 |
| **名师"诊断室"**（12次×1h） | — | 授课老师轮流在线答疑，形成问答库，解决实际管理难题 |
| **小组复盘与互评** | — | 线上讨论、案例分享、互评反馈 |
| **课题汇报与述职** | 中期/终期现场汇报 | 开题辅导、进展跟踪 |

**行动学习闭环**：线下学工具 → 线上练应用 → 提交案例 → 导师点评 → 复盘迭代。每位学员至少提交6次实践案例，优秀者入选《管理实战案例集》。

## 十三、培训闭环与业务产出

培训不能是单点事件，必须看到对业务的直接帮助。本项目以行动学习为核心，构建"培训→实践→业务产出"的完整闭环：

```
线下集训（学方法）
    ↓
课后实践（用工具）——产出：目标分解表、绩效面谈记录、授权清单等
    ↓
月度跟进（查应用）——名师诊断室答疑、工具应用反馈、直线上级确认
    ↓
季度复盘（看效果）——业绩回顾与管理层评估、课题进展检查
    ↓
年度收官（验成果）——结题汇报面向高管述职、管理实战案例集沉淀
```

每个培训模块均产出可落地的具体交付物，直线上级参与验收，确保管理能力提升直接映射到业务指标的改善。

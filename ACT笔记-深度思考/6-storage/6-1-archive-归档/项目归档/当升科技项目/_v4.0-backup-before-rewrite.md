# 当升科技新晋经理管理能力提升建议方案

**建议方案 v4.0 | 2026年5月 | 摩根世纪**

---

## 一、对当升科技项目背景及需求的理解

### 1.1 当升科技的发展阶段与管理挑战

当升科技正处于从"国内锂电正极龙头"向"全球新能源材料领军者"迈进的关键阶段。国内多基地并行运营（常州、攀枝花、昆明），海外芬兰工厂即将投产，业务快速增长对管理人才供给提出了更高要求。

公司于2026年正式启动"领导力建设三年行动（2026-2028）"，明确提出以"干部四大职责（建组织、搭流程、带队伍、做业绩）、三大关键能力（领导力、组织力、专业力）"为指引，系统性提升干部队伍能力。人才体系建设已是"飞机在飞行中换引擎"的紧迫课题。

### 1.2 本项目聚焦的人群

本项目面向的是当升科技**新晋基层管理者**——他们此前为技术/业务骨干，**没有正式的管理经验**，近期或即将从"个人贡献者"走向"团队管理者"岗位，约48人。

这个人群的核心特征是：技术能力过硬、业务经验丰富，但管理经验为零。他们面临从"自己做"到"带人做"的第一个跨越，需要的不是高阶领导力，而是**一线管理基本功**。

> 注：关于"新晋管理者"的具体界定标准（如管理幅度、任职时间等），建议在项目启动前与当升科技人力资源部共同确认。

### 1.3 新晋管理者面临的核心挑战

基于摩根世纪对同类制造企业新晋管理者的长期观察，无管理经验的新晋经理普遍面临四类挑战：

**角色转型挑战**：从技术/业务岗位走向管理岗位，工作重心从"自己做好"转向"让别人做好"。最大的困难不是学不会管理方法，而是**意识不到"管理者"和"业务能手"是两种完全不同的角色**——遇到问题第一反应是"我来解决"，而非"我来安排人解决"。

**任务管理挑战**：从"接受任务、完成执行"到"分解目标、分配任务、追踪结果"。面对多线程工作时容易陷入被动救火，缺乏计划、分工、过程管控的系统方法。

**带队育人挑战**：从"管自己"到"带团队"。在识别下属特点、日常辅导、有效授权、绩效面谈等关键动作上缺乏结构化方法，往往沿用"以前我师傅怎么带我，我就怎么带人"的经验模式。

**协同经营挑战**：随着管理半径扩大，需要跳出本班组视角，推动跨部门协作、理解业务逻辑和成本效率，从"守好自己一亩三分地"到"站在组织角度看管理"。

### 1.4 本项目在三年行动中的定位

当升科技"领导力建设三年行动"覆盖初、中、高三级管理者，是一项系统性工程。本项目的定位是：

> **三年行动的起点项目——面向无管理经验的新晋基层管理者，用12个月帮助他们完成从"业务能手"到"胜任管理者"的第一个跨越，为后续中层、高层领导力项目奠定人才基础和可复用的方法论。**

具体而言：

- **人群定位**：三年行动中"基层管理能力标准化"环节，面向初次走上管理岗位的人员
- **体系定位**：三年三环的第一环，后续可衔接中层领导力发展（第二环）和高层战略领导力（第三环）
- **标准衔接**：以当升科技"四大职责、三大关键能力"为培养标准，将公司级干部要求转化为新晋管理者可习得、可检验的具体行为
- **方法论沉淀**：本项目沉淀的能力标准、培养模式、案例库和运营机制，可为后续中高层项目提供可复用的基础
- **退出标准**：12个月后，学员具备独立承担班组/小型团队管理职责的能力，考评结果纳入公司人才盘点体系，为三年行动中的人才选拔和晋升提供数据输入

**与三年行动的衔接关系**：

| 维度   | 三年行动要求                      | 本项目的承接                       |
| ---- | --------------------------- | ---------------------------- |
| 总体导向 | 以四大职责、三大关键能力为指引             | 将公司级标准转化为新晋管理者的具体行为要求        |
| 实施原则 | 高层推动、层层负责；项目驱动、实效为本         | 三阶段实战驱动，课题聚焦业务痛点，直线上级全程参与    |
| 培养方式 | 实战项目锻炼为主（约80%），培训学习为辅（约20%） | 线下集训+行动学习+课题研究+岗位实践，训战结合     |
| 个人发展 | 梳理个人发展目标，找出差距，明确成长思路        | 课前诊断→个人转型计划→阶段评估→结业述职，全过程可跟踪 |

## 二、摩根世纪整体解决方案

### 2.1 方案设计理念

本方案的设计遵循一个核心原则：**从客户需求出发设计内容，而非从现有课程出发拼凑方案。**

我们的设计路径是：

```
理解当升科技业务与新晋管理者真实挑战
        ↓
明确该层级管理者需要具备的核心能力
        ↓
围绕核心能力设计培养模块（九大模块）
        ↓
为每个模块定义明确的行为产出与验收标准
        ↓
匹配培养方式与执行方案
```

这一设计方法的核心假设是：**培训要解决的不是"学员上了多少课"，而是"学员回到岗位后能做哪些不一样的事"。** 因此，每个培养模块都必须对应一项明确的能力缺口，每项能力缺口都必须对应一类真实的管理挑战。

### 2.2 新晋管理者能力培养框架

以当升科技"四大职责、三大关键能力"为培养标准，结合无管理经验新晋管理者的岗位特点，我们将公司级干部要求转化为一线管理岗位**可训练、可检验**的具体能力要求，并据此设计九大培养模块。

**核心逻辑**：
- **四大职责**（建组织、搭流程、带队伍、做业绩）——这是新晋管理者需要完成的**工作内容**
- **三大关键能力**（领导力、组织力、专业力）——这是完成上述工作内容需要具备的**能力条件**
- **九大培养模块**——这是帮助新晋管理者建立上述能力条件的**训练载体**

三者关系：**职责是靶心，能力是条件，模块是路径。**

| 当升科技干部标准                      | 新晋管理者需要习得的关键行为                | 对应培养模块                          |
| ----------------------------- | ----------------------------- | ------------------------------- |
| **建组织**（定位/架构/资源/文化）          | 理解管理者的角色定位，认知从"做事"到"管人管事"的转变  | **M1 管理者角色转换**                  |
| **搭流程**（机制/制度/流程/方法/工具）       | 设定清晰可衡量的目标，追踪过程，评估结果，开展绩效面谈   | **M2 绩效管理**（目标设定、反馈与辅导、业绩评估与激励） |
|                               | 高效组织与管理会议，确保会议有决议、决议有跟踪       | **M3 高效会议**                     |
| **带队伍**（梯队/人才/培养）             | 识别下属的能力特点与发展阶段，选择匹配的管理方式      | **M4 知人善任**                     |
|                               | 合理分派任务，给予自主权并有效跟进             | **M5 授权**                       |
|                               | 在向上、平行、向下三种场景中进行有效管理沟通        | **M6 高效沟通**                     |
| **做业绩**（战略/计划/执行/考核/激励）       | 建立团队凝聚力与共同目标，驱动团队达成业绩         | **M7 打造高绩效团队**                  |
| **组织力**（组织协调推动、检查评估纠偏、考核评价激励） | 跨部门推动协作，识别利益相关方并赢得支持          | **M8 影响力与团队协作**                 |
| **专业力 + 做业绩**（躬身入局、战略/计划）     | 理解基本财务概念，从成本、效率、产出的经营视角审视管理动作 | **M9 商业敏感度-财务基础**               |

**设计说明**：
- 九大模块完整覆盖了当升科技"四大职责、三大关键能力"对新晋管理者的全部要求，无遗漏
- 每个模块对应一项明确的能力训练，九个模块之间各有独立培养目标，不存在内容重叠
- 模块名称采用行业通用表述，与秦曼老师确认的九大模块一致

**九大模块的能力维度归纳**：

为便于呈现体系的完整性，我们将九大模块按能力属性归纳为三个训练方向。这不是在"四大职责、三大能力"之外另建模型，而是对培养内容的组织方式——三个方向分别对应新晋管理者从"管好事"到"带好人"到"看懂经营"的能力进阶：

| 训练方向 | 涵盖模块 | 核心回答的问题 |
|---------|---------|-------------|
| **管理能力** | M1角色转换、M2绩效管理、M3高效会议 | 怎么把日常管理工作做专业？——从"凭经验"到"有方法" |
| **带队能力** | M4知人善任、M5授权、M6高效沟通、M7打造高绩效团队 | 怎么带出能打胜仗的队伍？——从"自己干"到"带人干" |
| **经营能力** | M8影响力与团队协作、M9商业敏感度-财务基础 | 怎么跳出班组看组织？——从"守一亩三分地"到"为整体结果负责" |

三个方向的内在关系：**管理能力是基础**（先把自己的事管明白），**带队能力是核心**（通过他人拿结果），**经营能力是延伸**（站在更高维度审视管理动作的价值）。三者层层递进，与三阶段培养路径自然对应。

### 2.3 培养路径总览

九大模块不是平行排列的课程清单，而是按照新晋管理者的成长规律——**先转身、再管事、再带人、最后建团队看经营**——进行三阶段递进设计：

| 阶段   | 时间     | 主题          | 核心模块                              | 线下安排      | 阶段目标             |
| ---- | ------ | ----------- | --------------------------------- | --------- | ---------------- |
| 第一阶段 | 6月     | **转身+管事**   | M1角色转换、M2绩效管理、M3高效会议              | 7天集训      | 完成角色认知，建立任务管理基本功 |
| 第二阶段 | 8-9月   | **带人**      | M4知人善任、M5授权、M6高效沟通                | 4天课程+3天复盘 | 掌握带队育人的结构化方法     |
| 第三阶段 | 11-12月 | **建团队+看经营** | M7打造高绩效团队、M8影响力与团队协作、M9商业敏感度-财务基础 | 4天课程+3天复盘 | 建立团队领导力与经营视角     |

**模块顺序的设计逻辑**：

M1（角色转换）是所有模块的前置条件——没有完成"我是管理者"的身份认知转变，后面的管理方法学了也落不了地。M2-M3解决"把日常管理工作做专业"的问题——先会定目标、追过程、开好会，这是管理者最频繁面对的基本功。M4-M6在"管事"的基础上叠加"带人"——有了管理流程的框架，才能把知人、授权、沟通嵌入日常工作而非单独做一套。M7-M9在"带好自己人"的基础上叠加"建团队"和"看经营"——有了带个体的经验，才能理解如何打造团队合力、如何跨边界协作、如何站在经营角度看管理。

**与三年行动的培养方式呼应**：三年行动提出"实战项目锻炼为主（约80%），培训学习为辅（约20%）"。本项目三阶段中，每阶段线下集训后均设置持续的行动学习与课题实践，确保"学"和"做"的比例符合公司导向。

## 三、摩根世纪核心优势

### 3.1 22年深耕企业人才培养

摩根世纪成立22年来，始终专注企业培训与人才发展领域。我们服务的客户以央企、国企及大型制造企业为主，涵盖中信建设、中交集团、中核集团、中铝集团、中原对外工程等。在服务这些企业的过程中，我们积累了对中国制造企业管理场景的深度理解——

- 我们理解技术出身的管理者在角色转型中面临的真实困境
- 我们理解多基地运营企业在新人培养和管理标准化上的迫切需求
- 我们理解央企/国企在培训中对"实战"和"业务产出"的高度重视

这些理解不是来自理论框架，而是来自22年、数百个项目的实践沉淀。

### 3.2 实战导向的培训方法论

摩根世纪的培训理念：**"不是教管理知识，而是带管理者打实战。"**

我们坚持三个设计原则：

**原则一：行为改变而非知识输入。** 每门课程的设计起点不是"要讲什么内容"，而是"学员学完后要能做什么不一样的事"。围绕这个目标反向设计教学内容、练习方式和考核标准。

**原则二：案例来自客户业务场景。** 不使用通用教材案例。项目启动前，摩根项目组将通过需求诊断了解当升科技的业务特点和管理场景，将真实行业场景融入案例和模拟练习，确保"课上学的东西，明天上班就能用"。

**原则三：闭环设计确保效果可见。** 每阶段设置明确的"输入→练习→产出→评价"闭环。学员需提交可验证的实践成果（如目标分解表、绩效面谈记录、团队改进方案），让培训效果"被看见、被衡量"。

### 3.3 与当升科技的前期合作基础

摩根世纪已与当升科技建立了前期合作关系。在轮岗管理制度咨询项目中，摩根项目组对当升科技的组织架构、人才现状和管理挑战有了一定了解。部分摩根专家已参与前期诊断，熟悉当升的业务语言和管理场景。

这一合作基础意味着：本项目不是从零开始理解当升，而是在已有认知的基础上进行更精准的需求聚焦和方案设计，可以快速进入实质性的内容定制阶段。

## 四、项目整体框架

### 4.1 培养目标

**总体目标**：通过12个月系统培养，帮助48位无管理经验的新晋管理者完成从"业务能手"到"胜任管理者"的角色转型，建立完整的一线管理基本功。

**分维度目标**（12个月后学员能够……）：

| 目标维度 | 具体行为标准 | 验收方式 |
|---------|------------|---------|
| **角色认知** | 清晰理解管理者的职责定位，遇到问题首先思考"谁来负责、怎么安排"，而非"我来解决" | 个人转型计划 + 直线上级评价 |
| **任务管理** | 独立完成目标分解与任务分配；主持结构化会议并跟踪决议落地；开展绩效面谈 | 目标分解表 + 会议记录及跟踪表 + 绩效面谈记录 |
| **带队育人** | 根据下属特点选择管理方式；进行有效授权并跟进；运用沟通技巧处理对上对下对平级的管理场景 | 团队人员分析表 + 授权计划 + 沟通案例记录 |
| **协同经营** | 诊断团队健康度并制定改进计划；推动跨部门协作；从成本与效率视角审视管理决策 | 团队改进方案 + 跨部门协作案例 + 结业述职报告 |

### 4.2 课程体系（九大模块）

| 序号 | 模块名称 | 核心解决的新晋管理者问题 | 阶段 |
|------|---------|----------------------|------|
| M1 | 管理者角色转换 | "我以前是技术骨干，现在要带团队——角色变了，我该怎么做？" | 一 |
| M2 | 绩效管理（目标设定、反馈与辅导、业绩评估与激励） | "目标怎么定？过程怎么跟？绩效面谈怎么谈才能不流于形式？" | 一 |
| M3 | 高效会议 | "会开了很多，但决议落实了多少？怎么让会议不浪费时间？" | 一 |
| M4 | 知人善任 | "下属能力参差不齐、性格各有不同，怎么识别特点、因材施管？" | 二 |
| M5 | 授权 | "什么事该放手给下属做？放手给谁？放手之后怎么跟才放心？" | 二 |
| M6 | 高效沟通 | "对上汇报、对下安排、对平级协调——不同场景怎么说才能让人听进去？" | 二 |
| M7 | 打造高绩效团队 | "不是一群人在一起就叫团队——怎么让大家目标一致、力出一孔？" | 三 |
| M8 | 影响力与团队协作 | "事情不归我管，但我需要别人配合——怎么让跨部门的人愿意帮你？" | 三 |
| M9 | 商业敏感度-财务基础 | "管理者不懂成本和利润，做决策就没有锚点——怎么看懂基本的财务数字？" | 三 |

**模块设计的三项原则**：

- **一致性**：模块名称和定义使用行业通用表述，与秦曼老师确认的九大模块一致，确保当升科技人力资源部向公司领导汇报时，每个模块的名称和内涵都能获得认可
- **递进性**：模块顺序遵循"转身→管事→带人→建团队看经营"的成长逻辑，不可跳跃。M1是所有模块的前置条件；M2-M3是日常管理基本功；M4-M6以基本功为基础叠加带人能力；M7-M9以带人能力为基础叠加团队与经营视角
- **完整性**：九个模块覆盖了无管理经验的新晋管理者从角色转型到独立带团队的完整过程，不存在能力盲区，也不存在内容重叠

### 4.3 三阶段培养设计

| 阶段 | 时间 | 线下安排 | 核心模块 | 能力侧重 | 实战产出 | 对业务的直接影响 |
|------|------|---------|---------|---------|---------|----------------|
| 第一阶段·转身+管事 | 6月 | **7天集训** | M1+M2+M3 | 角色认知+任务管理 | 个人转型计划、部门目标分解表、会议主持实践记录 | 学员回岗后立即产生三个变化：①目标对齐——每个班组有书面的季度目标分解表，任务不再"口头交代、凭感觉做"；②会议提效——班组会议有议程、有决议、有跟踪，减少"开了等于没开"的时间浪费；③角色清晰——新晋经理不再"和下属抢活干"，直线上级能观察到行为转变 |
| 第二阶段·带人 | 8-9月 | **4天课程+3天复盘实践** | M4+M5+M6 | 人员管理+沟通辅导 | 团队人员分析表、个人授权计划、管理沟通案例 | 学员回岗后产生三个变化：①人岗匹配——能识别下属的能力差异，不再"一刀切"分配任务；②上级减负——管理者开始有效授权，直线上级从"帮下属救火"中解放出来；③沟通有序——对上汇报、对下反馈、跨部门协调有结构化方法，减少因沟通不当导致的返工和摩擦 |
| 第三阶段·建团队看经营 | 11-12月 | **4天课程+3天复盘实践** | M7+M8+M9 | 团队建设+经营思维 | 团队诊断与改进方案、跨部门协作案例、结业述职报告 | 学员回岗后产生三个变化：①团队合力——能诊断班组健康度并落地改进方案，团队不只是"一群人"而是"一个整体"；②跨部门打通——主动识别利益相关方并推动协作，减少"部门墙"导致的进度延误；③成本意识——做管理决策时能考虑成本和效率因素，而非仅凭技术判断 |

**阶段间的衔接**：
- 每阶段结束前进行阶段评估，诊断学员当前能力水平，为下一阶段课程重点提供调整依据
- 两阶段之间通过月度线上工作坊、工具应用跟踪、名师答疑保持学习连续性
- 课题研究贯穿三阶段：第一阶段开题→第二阶段中期汇报→第三阶段结题述职（面向公司管理层）

## 五、新晋管理人员能力发展路径

### 5.1 从"业务能手"到"胜任管理者"的三个跨越

无管理经验的新晋管理者走向"胜任管理者"，需要经历三个关键跨越：

**第一个跨越：心态转变（第一阶段解决）。** 从"我的业绩就是我的价值"转变为"团队的业绩就是我的价值"。M1管理者角色转换模块帮助学员完成这个跨越——理解管理者的核心职责不是"自己做出好成绩"，而是"让团队做出好成绩"。

**第二个跨越：方法补位（第二阶段解决）。** 从"凭经验、靠感觉"转变为"有方法、有工具"。M4-M6围绕日常管理高频场景（识人、授权、沟通），为学员提供结构化的方法和工具，让管理动作从"艺术"变成"可操作的技术"。

**第三个跨越：视角升级（第三阶段解决）。** 从"站在班组看班组"转变为"站在组织看管理"。M7-M9帮助学员跳出本团队视角，学会建团队合力、跨部门协作、从经营角度看管理。

三个跨越对应三个阶段，每个阶段的培养内容和方式都围绕"让学员完成这个跨越"来设计，而非简单地"把这个模块的课上完"。

### 5.2 各模块学习目标与实战产出

每个模块的实战产出，就是学员回到岗位后可以直接用于工作的管理工具或方案：

| 模块 | 学习目标（学员学完后能够……） | 实战产出（学员回到岗位后完成……） |
|------|---------------------------|-------------------------------------|
| M1 角色转换 | 清晰区分"业务贡献者"与"团队管理者"的角色差异；理解管理者在上级、平级、下属三个方向上的职责定位 | 撰写**个人转型计划**（含自我差距分析、转型关键动作、3个月里程碑），由直线上级签字确认 |
| M2 绩效管理 | 设定符合SMART原则的团队和个人目标；开展结构化绩效面谈（含反馈、辅导、评估） | 制定**本班组季度目标分解表**；完成一次真实**绩效面谈**并提交面谈记录 |
| M3 高效会议 | 区分需要开会和不需要开会的情境；掌握会前准备、会中控场、会后跟踪的结构化方法 | 主持一次班组会议，提交**会议纪要+决议跟踪表** |
| M4 知人善任 | 诊断下属的成熟度与发展阶段；根据下属特点选择匹配的管理与辅导方式 | 完成**团队人员分析表**（含每人优势、待发展领域、适配管理方式） |
| M5 授权 | 区分必须自己做、可以授权、应该授权三类任务；根据任务和人员匹配度选择授权方式并跟进 | 制定**个人授权计划**（含可授权任务清单、授权人选、跟进节点） |
| M6 高效沟通 | 区分向上、平行、向下三种沟通场景的策略差异；运用结构化方法开展管理沟通 | 提交一份**真实管理沟通案例**（场景描述+策略选择+沟通结果+复盘反思） |
| M7 打造高绩效团队 | 诊断团队在目标、角色、流程、关系四个维度上的健康度；运用团队建设方法提升战斗力 | 完成**团队诊断报告+改进方案** |
| M8 影响力与协作 | 识别关键利益相关方及其立场；运用影响力策略赢得跨部门支持 | 完成一个**跨部门协作案例**（相关方地图+策略选择+推进过程+协作成果） |
| M9 商业敏感度 | 读懂三张基本财务报表；理解成本结构、利润驱动和投资回报的基本概念 | 完成一个**管理决策的财务影响分析**（如"增加人手 vs. 加班"的成本对比） |

**关于实战的三个保障机制**：

1. **直线上级确认**：关键产出（如个人转型计划、绩效面谈记录、授权计划）需由学员直线上级签字确认，确保实战不是在"做作业"，而是在"做真事"
2. **课题贯穿全程**：学员自组课题小组，选题聚焦当升科技实际业务痛点（如多基地运营、班组绩效、跨部门协同），课题成果直接服务于部门工作
3. **述职检验**：第三阶段结业时，学员面向公司管理层做述职报告，展示12个月来的管理实践成果和能力成长轨迹

### 5.3 实战输出全景

12个月内，每位学员至少沉淀以下实战成果：

| 产出类型 | 数量 | 说明 |
|---------|------|------|
| 个人转型计划 | 1份 | 第一阶段产出，定义从"业务能手"到"管理者"的个人转型路径 |
| 管理工具应用记录 | 6份 | 目标分解表、绩效面谈记录、会议跟踪表、人员分析表、授权计划、沟通案例各1份 |
| 团队级产出 | 2份 | 团队诊断与改进方案、跨部门协作案例各1份 |
| 管理决策分析 | 1份 | 含财务视角的管理决策分析 |
| 课题成果 | 1项 | 小组课题研究报告，聚焦当升实际业务痛点 |
| 结业述职报告 | 1份 | 面向管理层的12个月能力成长述职 |

**集体产出**：48位学员的实战成果经过筛选和整理，汇编为《当升科技管理实战案例集》，作为当升科技内部管理经验沉淀，可复用于后续批次的培训和中高层领导力项目。

## 六、项目执行方案

### 6.1 线上线下融合培养机制

考虑到当升科技生产任务重、人员脱产时间有限的实际情况，采用"线下集中突破 + 线上持续渗透"的融合模式。

**线下集中（三阶段，共约21天）**：
- 第一阶段：7天集中集训
- 第二阶段：4天课程 + 3天复盘实践
- 第三阶段：4天课程 + 3天复盘实践
- 线下聚焦：核心内容讲授、情景模拟、技能演练、案例研讨、课题汇报

**线上持续（贯穿12个月）**：

| 线上服务 | 频次 | 内容说明 |
|---------|------|---------|
| 月度主题工作坊 | 每月1次×1.5小时 | 围绕当月管理主题进行工具精讲、案例分享、经验交流 |
| 月度管理工具推送 | 每月1次 | 推送配套工具模板，学员提交应用记录，项目组跟踪统计 |
| 名师答疑 | 每月1次×1小时 | 授课讲师在线值班，解答学员实际管理难题，形成问答知识库 |
| 社群学习 | 持续 | 学习进度跟踪、积分管理、优秀案例分享 |

**融合逻辑**：线上学基础知识+做预习 → 线下集中练习+深度研讨 → 线上跟进应用+持续答疑。线上不是线下的"补充"，而是让线下时间更值钱——学员带着预习认知和工作中遇到的问题来参加线下集训，线下时间全部用于练习、研讨和解决真实问题。

### 6.2 行动学习与课题研究

学员自由组成6-8个课题小组，围绕当升科技经营实际和管理痛点选题。课题方向由摩根项目组协同当升科技各部门拟定选题清单，报公司领导审核。**直线上级可推荐课题方向**，确保课题与部门业务紧密相关。

课题方向建议（最终以当升科技确认的选题清单为准）：

| 方向 | 选题示例 | 产出价值 |
|------|---------|---------|
| 多基地运营 | 昆明新基地投产前后的班组目标对齐与人员部署优化 | 为新基地产能爬坡提供管理参考 |
| 技术人才转型 | 技术骨干向产线负责人的转型路径与关键管理动作 | 沉淀可复制的新任管理者操作指引 |
| 跨部门协同 | 研发-生产-销售在新产品导入中的协同效率提升 | 缩短新产品从研发到量产的衔接周期 |
| 班组绩效 | 一线班组绩效考核指标的设计与过程管理优化 | 提升班组绩效管理的公平性和有效性 |

课题贯穿三阶段：第一阶段开题→第二阶段中期汇报→第三阶段结题汇报（面向公司管理层述职）。

### 6.3 培训管理与考评

**双班主任机制**：
- 当升科技指定一名班主任：指导培训方向、协调内部资源、参与关键节点评审
- 摩根世纪指定一名班主任：现场日常管理、教学组织、学习跟踪与数据反馈

**学员考评（综合评分）**：

| 考评项 | 权重 | 说明 |
|-------|------|------|
| 实战产出 | 40% | 各模块要求的行为产出完成质量（见5.2节） |
| 课题成果 | 30% | 开题5% + 中期10% + 结题15% |
| 模块测验 | 15% | 每模块课后线上测评 |
| 过程参与 | 15% | 出勤、线上参与、工具提交、社群贡献 |

综合成绩前30%的学员评为"优秀学员"，评选优秀课题、优秀班委。考评结果纳入公司人才盘点与晋升参考。

**直线上级参与机制**：

| 节点 | 动作 | 目的 |
|------|------|------|
| 项目启动前 | 向直线上级发送培训说明 | 让上级理解培训目标及自身需配合的动作 |
| 每阶段前 | 发送学员阶段进展 | 让上级了解学习情况和课题方向 |
| 每阶段后 | 收集直线上级对学员训后表现的反馈 | 纳入下一阶段课程调整依据 |
| 课题汇报 | 邀请直线上级列席 | 让上级看到学员成果 |
| 项目收官 | 评选"优秀人才培养奖" | 对积极配合、学员成长显著的直线上级给予表彰 |

### 6.4 效果监测与改进

| 监测维度 | 数据来源 | 采集频次 | 关注标准 |
|---------|---------|---------|---------|
| 学员参与度 | 出勤率、线上参与率、产出提交率 | 每月 | 连续2个月低于60%需项目组介入 |
| 学习转化度 | 各模块行为产出完成质量 | 每阶段 | 产出质量持续偏低需调整教学方式 |
| 上级满意度 | 直线上级反馈问卷 | 每阶段结束 | 作为课程调整参考 |
| 课题推进度 | 开题→中期→结题节点检查 | 每阶段 | 延期超2周需项目组介入 |

当监测数据出现异常时，摩根项目组将与当升科技HR共同研判，在15个工作日内提出调整方案。

### 6.5 项目启动倒排计划

| 时间节点 | 关键里程碑 | 责任方 |
|---------|----------|--------|
| 方案确认后第1周 | 方案确认、合同签署 | 当升科技HR + 摩根 |
| 方案确认后第2周 | 向学员及直线上级发送通知、需求问卷、场地确认 | 当升科技HR + 摩根 |
| 方案确认后第3周 | 学员排班确认、直线上级沟通、需求诊断访谈 | 当升科技HR + 摩根 |
| 开课前1周 | 预习资料发放、学员需求汇总至讲师、场地布置 | 摩根 |
| 6月上旬 | **第一阶段（转身+管事）开营·7天集训** | 摩根 |
| 8月下旬 | **第二阶段（带人）·4天课程+3天复盘实践** | 摩根 |
| 11月下旬 | **第三阶段（建团队看经营）·4天课程+3天复盘实践** | 摩根 + 当升科技管理层 |

### 6.6 与当升科技现有人才体系衔接

| 衔接点 | 衔接方式 |
|-------|---------|
| 人才盘点 | 学员考评结果纳入当升年度人才盘点数据，作为"管理潜力"维度的参考输入 |
| 晋升评审 | 综合成绩前30%的优秀学员，建议在同等条件下优先考虑晋升 |
| IDP（个人发展计划） | 培训中制定的个人转型计划和管理改进方案，可直接纳入当升现有IDP体系 |
| 内部讲师 | 优秀学员可发展为当升内部管理课程兼职讲师 |
| 案例沉淀 | 课题成果和管理实战案例集可作为当升内部管理培训的后续材料 |

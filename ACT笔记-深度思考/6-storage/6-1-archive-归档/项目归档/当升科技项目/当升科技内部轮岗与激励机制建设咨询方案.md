# 当升科技内部轮岗与激励机制建设咨询方案

## 一、项目背景与需求理解

### 1.1 战略总纲：当升科技"领导力建设三年行动"

2026-2028年，当升科技以**"干部四大职责、三大关键能力"**为指引，系统构建人才团队、组织系统、动力机制、企业文化，全面提升组织能力。

| 四大职责 | 核心内涵（三年行动原文） | 轮岗制度如何承载 |
|---------|------------------|----------------|
| **建组织** — 定位、架构、资源、文化 | 明确组织在战略中的角色与价值方向；设计高效协同的组织结构；合理配置人财物资源；塑造支撑战略执行的共同价值观与行为规范 | 通过轮岗路径规划，回答"什么人、什么阶段、到什么地方、解决什么问题"，使人才配置服务于组织架构演进 |
| **搭流程** — 机制、制度、流程、方法、工具 | 建立标准化运作规则与互动模式；制定约束力规范性文件；梳理端到端核心业务活动顺序；总结推广实践技术；开发标准化辅助手段 | 轮岗制度本身即是"流程"建设——将人才流动从一事一议的临时决策，升级为标准化的运作规则 |
| **带队伍** — 梯队、人才 | 构建层次清晰、衔接有序的后备人才储备体系；通过选拔、培养与发展，提升团队成员专业能力与潜力 | 轮岗是后备干部培养的核心手段——进入轮岗池即进入后备梯队，跨部门经历即能力横向拓展，海外历练即国际化人才储备 |
| **做业绩** — 战略、计划、执行、考核、激励 | 制定清晰长期发展目标与竞争策略；将战略分解为可执行的具体行动方案；推动计划落地确保高效完成；定期评估工作成果与绩效；依据考核结果实施奖励与激发措施 | 轮岗制度的设计闭环（选拔→流动→考核→激励→晋升）直接回应"攻山头、打胜仗、练队伍"——把合适的人在合适的时间派到最需要的战场 |

本轮咨询的定位：**将"领导力建设三年行动"在人才流动维度制度化落地。** 不是独立的人力资源项目，而是三年行动的基础设施——先建立人才流动的规则底座，后续薪酬、绩效、继任计划才能在此基础上系统升级。

### 1.2 本项目解决的核心问题

轮岗制度的设计不是从零开始的理论推演，而是对一系列实际问题的系统性回应。以下问题将在D1诊断阶段通过分层访谈与数据分析逐一厘清，当前阶段呈现的是制度设计需要覆盖的核心议题方向：

**制度层面**：跨部门、跨层级、跨地域的人才流动，需要怎样的规则体系？不同价值岗位之间的轮岗——薪酬怎么定、回程怎么保障、晋升如何挂钩？

**激励层面**：从单一基地到多基地+海外扩张，激励体系如何支撑跨地域、跨岗位的人才流动？如何形成"能流动、愿意流动、流动后能发展"的正向循环？

**人才储备层面**：芬兰基地一期产线建设中，海外新基地持续规划中，国内多基地并行运营——业务扩张所需的人才从哪里来？国内抽调走的骨干空缺如何填补？

> 上述问题的具体答案，需在D1诊断完成后，基于访谈数据与当升科技实际业务场景得出。本方案后续各章节呈现的是制度设计的方法论框架与工作步骤，而非对上述问题的预判性结论。

### 1.3 摩根优势：更快、更落地

**一、前期合作积累的知情优势**

前一轮芬兰项目中，摩根完成了三项工作，这些工作建立了对当升的直接认知：

| 工作项           | 内容                      | 对本项目的价值                                         |
| ------------- | ----------------------- | ----------------------------------------------- |
| **常州基地培训**    | 为即将外派的中方人员开展系统培训（持续三个月） | 接触了一线管理者与骨干员工，了解当升的人才结构、管理风格和实际工作场景             |
| **芬兰科特卡实地调研** | 进入外派人员工作现场，观察中芬协作场景     | 掌握了海外基地的实际运营环境、中外团队协作中的真实摩擦点——这些是轮岗制度设计的一手输入    |
| **中芬双向团队融合**  | 同时面向中方外派人员与芬兰本地员工开展工作   | 理解了当升从"国内管理"到"国际化协作"转型中的具体挑战——轮岗制度需要为这种转型提供人才支撑 |


**二、独立于培训的咨询能力**

摩根22年来服务客户以央企、国企及大型制造企业为主体，在长期实践中形成了以"实战落地"为核心特征的服务模式。摩根专家网络中，既有来自国际顶尖咨询机构的资深合伙人，也有曾在多家头部企业担任人力资源高管的实战型专家。这种"咨询方法论+企业实操经验"的复合能力结构，确保制度设计既有系统性的专业高度，又充分考量落地环节的实际约束。具体专家配置详见本方案第八章"项目团队"。

**三、两种能力的叠加效应**

外部咨询团队通常需要4-6周才能基本理解客户的组织语境与决策逻辑。基于前期合作积累的直接认知，摩根可将这一阶段显著压缩，将时间与资源集中投入在方案质量而非基础信息获取上。同时，摩根在咨询领域的专业积累确保本项目的交付标准——制度设计的系统性、落地性、可迭代性。

**核心管控原则**：
- **阶段门禁**：每阶段结束有关键产出物和确认节点，未经确认不得进入下一阶段。出现分歧以董事长最终意见为准

- **时间灵活**：整体8-10周，现场工作每周1-2天（关键节点可增至2-3天），具体节奏根据当升科技实际业务灵活调整

## 三、项目具体实施内容

### 阶段一：Diagnose 诊断与确认（第1-2周）

#### 1. 联合项目组组建

采用"联合项目组深度融合"模式：

| 要素 | 安排 |
|------|------|
| **组成** | 摩根专家团队 + 当升科技HR骨干，全程协同推进 |
| **HR参与** | 当升科技自行决定参与学习的HR人员数量，参与人员全程跟项目即可在过程中完成能力内化 |
| **带教内容** | 访谈技巧、框架梳理、制度设计方法——摩根专家在各阶段对当升HR团队进行"做中学"带教 |
| **工作模式** | 非全程驻场：摩根团队固定每周1-2天现场办公（关键节点可增至2-3天），非现场时间线上随时沟通 |

**客户方HR团队投入预估**（帮助HR总做内部资源调配）：

| 阶段       |     HR投入人数     | 关键投入节点                             |   预计投入天数/人    |
| -------- | :------------: | ---------------------------------- | :-----------: |
| D1 诊断与确认 |  2-3人（含对接负责人）  | 访谈全程陪同（3-5天）、系统框架梳理研讨（2-3天）、诊断报告审阅 |     5-8天      |
| D2 方案撰写  |  1-2人（含对接负责人）  | 方案研讨（每周1-2天现场）、修改反馈（2轮）、试点设计讨论     |     6-10天     |
| D3 汇报确认  |   1人（对接负责人）    | 业务部门沟通会组织（2-3场）、董事长汇报准备            |     3-4天      |
| D4 宣导固化  | 2-3人（HR专项小组全员） | 分层宣贯组织、带教收尾、制度定稿                   |     3-5天      |
| **合计**   |   **2-3人核心**   | **10周内分散投入**                       | **约17-27天/人** |

> 核心原则：HR团队不脱产跟项目——关键节点全程在场，日常通过线上异步配合。对接负责人在D1/D2阶段是确认门禁，必须参与研讨和审阅，但不需要全程驻场。

#### 2. 访谈方法论（专业建议）

基于咨询行业经过大量项目验证的方法论，本轮访谈设计如下：

**样本量：面对面访谈20-30人（含董事长）**

为什么是这个数字？质性研究的方法论共识表明，在一对一深度访谈中，**信息饱和点（Saturation Point）通常出现在第12-18个访谈之间**（Guest, Bunce & Johnson, 2006）——此后每个新增访谈提供的全新信息量急剧下降。将样本量设定为20-30人，既留出安全边际确保达到饱和，又避免超过30人后边际信息增量递减的资源浪费。

**分层抽样设计**：

| 专业设计要素 | 具体安排 |
|-------------|---------|
| **分层规则** | 按高/中/基三层比例抽取：高层必须全覆盖（董事长+总经理），中层干部为重点（占比约50%），基层核心员工抽样 |
| **业务覆盖** | 与轮岗调整相关的业务板块多抽（研发、生产、质量、商务），支持部门少抽——确保代表性 |
| **补充手段** | 超出抽样范围的员工通过AI问卷补充调研，保证覆盖广度同时控制调研成本 |
| **保密原则** | 若当升科技希望听取员工真实想法，甲方项目组可不参与具体访谈——摩根仅汇总共性问题汇报，不泄露具体访谈对象发言内容 |

**访谈五大维度**：

| 维度 | 核心问题方向 |
|------|-------------|
| 现状痛点 | 跨部门/跨层级流动中的主要困难？不同岗位间轮岗（如研发↔生产、生产→质量等）的特殊阻力？ |
| 制度缺口 | 当前是否有临时性调动政策？不同轮岗类型的差异化制度需求？最容易推诿的环节？ |
| 激励感知 | 当前津贴/薪酬体系能否支撑跨部门、跨地域流动？从高价值岗位到低价值岗位，薪酬如何过渡？ |
| 发展诉求 | 员工对职业发展路径的期望？轮岗与晋升之间是否建立关联？"轮出去回不来"是否是核心顾虑？ |
| 国际维度 | 芬兰外派经验对国内轮岗机制有何启示？后续印尼、摩洛哥等新基地的人才储备如何提前布局？ |

**48小时速览简报**：访谈结束后48小时内输出《当升科技人才流动现状速览简报》（1-2页），概要呈现关键发现与初步判断，不等完整诊断报告。让客户感受项目推进节奏，也为后续诊断报告和方案撰写提前对齐方向。

#### 3. 诊断分析与系统框架梳理

- 完整记录各层级、各人群的诉求点与顾虑点，形成初步诊断结论（问题清单、根因分析、优先级建议）
- 特别关注不同轮岗场景（研发↔生产、生产→质量、总部→基地、国内→海外）的差异化痛点
- 同步梳理人力资源整体系统框架（2-3个工作日），明确各类型轮岗与薪酬、绩效、考核、晋升、培训等模块的衔接关键点，输出系统框架图

#### 4. 阶段门禁与关键产出

| 关键产出 | 说明 |
|---------|------|
| 《当升科技人才流动现状速览简报》 | 访谈结束后48h内交付，1-2页 |
| 《当升科技人才流动现状诊断报告》 | 完整诊断结论 |
| 《人力资源系统框架图》 | 各类型轮岗的定位与接口设计 |
| 人力资源部确认 | **关键门禁：确认后方可进入方案撰写阶段** |
| 联合项目组就位 | HR团队进入"做中学"模式 |

### 阶段二：Design 方案撰写（第3-5周）

#### 1. 轮岗管理制度设计

| 设计模块 | 具体内容 |
|---------|---------|
| **轮岗类型与路径** | 三大类：①跨地域（国内基地间、国内↔海外）；②跨部门（研发↔生产、生产→质量、财务等职能→一线）；③跨层级（总部↔基地、管理岗↔技术岗）。覆盖芬兰/印尼/摩洛哥海外基地、攀枝花/常州/昆明国内基地、总部职能部门之间的全场景流动 |
| **关键岗位强制轮岗** | 总监以上级别候选人须到相关业务一线轮岗后方可晋升；中台风控/研发/财务等岗位须到一线轮岗后方能掌握实际业务情况；**一号位候选人必须深度轮岗生产、营销、研发全模块** |
| **轮岗期间薪酬保障** | 从高价值岗位轮岗至低价值岗位，**轮岗期间保留原薪酬标准**；轮岗后若适配更高价值岗位，按新岗位标准调整；明确回程保障，消除"轮出去回不来"的核心顾虑 |
| **轮岗与晋升绑定** | 进入轮岗池=后备干部，轮岗经历作为管理干部晋升的必要条件 |
| **轮岗周期与频次** | 按类型差异化：海外外派（1-3年）、国内跨基地（6个月-2年）、跨部门（3-12个月），设定最短/最长时限 |
| **轮岗津贴与待遇** | 分级分类津贴标准（按地域系数×岗位系数×家庭情况），住房、交通、家庭补贴等配套 |
| **选拔与匹配机制** | 轮岗岗位发布、人选推荐/竞聘、匹配确认流程；建立各关键岗位的后备人才池 |
| **绩效评估** | 派出单位与接收单位协同评价，明确考核主体与标准，避免"两不管" |
| **退出安置机制** | 回程岗位安排、退出通道，聚焦"轮出去回不来"的顾虑 |
| **人员补充循环** | 确保关键岗位在人员轮出后业务不出现真空 |

#### 2. 跨界轮岗专项设计

跨界轮岗（如研发↔生产、生产→质量等跨专业序列流动）是轮岗制度设计中最复杂的一类——它打破了传统的专业壁垒，需要独立的机制保障。本方案将其从通用轮岗框架中抽出，做专项设计。

**跨界轮岗的常见场景**（以下为制造业常见的跨界轮岗类型，当升科技的具体适用岗位和轮岗方向将在D1诊断阶段结合访谈与企业实际岗位信息确定）：

| 场景类型 | 典型示例 | 制度要解决的核心问题 |
|---------|---------|-----------------|
| **部门间流动** | 研发↔生产、生产→质量等 | 不同专业序列之间的人才如何有序流转，打破部门壁垒 |
| **职能到一线** | 财务、风控、HR等中台岗位→生产基地 | 职能人员深入业务现场后，决策才能贴合实际——如财务了解生产流程后方能准确核算成本 |
| **管理岗补技术认知** | 非技术背景管理者→技术岗（项目制） | 综合管理者通过短期项目制轮岗补足技术视角，避免决策脱离业务实际 |

> 上述场景类型为行业常见实践。当升科技适用的具体跨界路径、对应岗位及优先级，将在D1诊断阶段通过访谈和数据分析确定，在D2方案撰写阶段与HR团队联合设计细则。

**跨界轮岗的特殊保障机制**：

| 保障维度 | 普通轮岗 | 跨界轮岗（增强设计） |
|---------|---------|-------------------|
| **薪酬过渡** | 保留原薪酬标准 | 保留原薪酬标准 + 跨界适应津贴（岗位价值差异越大津贴越高），过渡期6-12个月 |
| **回程保障** | 轮岗结束后安排回程岗位 | 双通道回程：①返回原岗位序列（保留跨界经验加权）②留在新岗位序列（按新序列职级晋升）——选择权在员工 |
| **考核权重** | 接收单位考核为主 | 派出单位+接收单位+个人发展目标三重评价，具体权重在D2阶段结合岗位特征确定 |
| **能力准备** | 岗位基本要求匹配 | 轮岗前1-2周"跨界过渡期"——跟岗观察+业务导师一对一，降低跨界冷启动成本 |

**跨界轮岗与一号位培养**：一号位候选人需深度经历跨职能历练（如生产、营销、研发等核心业务模块），跨界轮岗的考核结果作为晋升评议的重要参考。这确保未来"一号位"拥有跨专业视角，而非只在单一职能线上成长。具体轮岗路径组合将在D1诊断后结合当升科技业务特征与岗位体系确定。

跨界过渡期、导师选拔激励、适配失败处理等操作细节，将在D1诊断完成后，结合访谈数据与当升科技实际岗位信息，在D2方案撰写阶段与HR团队联合设计。当前阶段仅呈现方法论方向，避免在信息不足时做出不贴合实际的预设。

#### 3. 激励机制衔接方案

| 设计模块 | 具体内容 |
|---------|---------|
| 轮岗津贴体系 | 分层分类补贴标准（地域系数×岗位系数×家庭情况），覆盖海外/异地/同城三类场景 |
| 跨价值岗位薪酬过渡 | 高→低价值岗位轮岗期间保留原薪酬标准；适配更高岗位后按新标准调整 |
| 绩效激励机制 | 突出跨部门适应、业务拓展贡献与一线实践成果的考核权重 |
| 长期激励挂钩 | 轮岗经历（尤其是跨界/海外/异地）与股权激励、职业晋升等长期激励挂钩 |
| 保留与留任 | 关键人才留任奖金、服务期约定、轮岗后职业发展通道保障 |

#### 4. 衔接设计与首轮试点

识别轮岗制度与薪酬体系、绩效管理、人才盘点与继任计划、分层管理能力提升计划（可衔接当升科技新晋管理人员领导力培训项目）的衔接点，形成接口方案。

**首轮试点**：制度设计不等"完美"才启动——在制度发布后1个月内，选取1-2条最具代表性的轮岗路径先行试跑（如跨界路径和跨地域路径各选一条），每路径3-5人，采用"提名+自愿"混合制。试点者配备双导师、月度回访、试点期满述职。核心逻辑：**用小范围成功案例打破观望心态。**

**试点→推广的完整路径**（回应"小范围试点之后怎么办"）：

| 阶段 | 时间 | 范围 | 关键动作 | 决策节点 |
|------|------|------|---------|---------|
| **试点期** | 制度发布后第1-6个月 | 2条路径 × 3-5人 = 6-10人 | 双导师月度回访、试点者月度1v1、试点中期（第3个月）进度检查 | — |
| **复盘决策期** | 第6个月 | — | 试点者述职（每人30分钟向高管评审团汇报）、摩根汇总《试点复盘报告》、试点评估结果公示 | 🔴 董事长主持试点复盘会，评估标准见下，决定是否启动第二批 |
| **第二批推广** | 第7-12个月 | 扩展至4-6条路径，覆盖30-50人 | 新增路径启动、第二批人选匹配、首轮试点成功者转为"轮岗大使"内部传播 | 人力资源部根据试点复盘会的决策执行 |
| **全面推行** | 第2-3年 | 覆盖关键岗位的50%以上 | 纳入年度人才盘点常规流程、轮岗制度与晋升评议正式挂钩、轮岗经历成为总监级以上晋升的必要条件 | 纳入年度干部考评会议程 |

**试点→推广的评估标准**（董事长主持试点复盘会时的决策依据）：

| 评估维度 | 是否可推广的信号 |
|---------|----------------|
| **试点者绩效** | 试点者在轮岗期间的绩效评估是否≥原岗位同期？接收部门对试点者的满意度是否≥80%？ |
| **业务影响** | 试点所在部门业务KPI是否未因人员轮换出现明显下滑？后备补充机制是否有效运转？ |
| **员工态度** | 试点启动时观望的员工，到第6个月是否有主动咨询"下一批什么时候"？自愿报名比例的变化趋势 |
| **制度运行** | HR团队是否能独立处理试点中的操作问题（无须摩根每次到场）？制度条款中有几条被证明需要修订？ |

> 推广逻辑的核心：不是靠行政命令摊派指标，而是让第一批试点者成为"活广告"——当公司同事看到轮岗回来的人被安排了更重要的位置，第二批报名自然从"要我去"变成"我要去"。

**高管层的参与方式**（D3汇报确认阶段与董事长沟通后确定）：

高管层的参与是轮岗制度落地的关键变量。具体参与方式（如高管是否率先参与轮岗体验、是否担任试点导师、在何种场合为制度背书）将在D3阶段结合董事长意见和当升科技内部决策文化确定，不在当前阶段预设。

试点述职的评审维度、复盘会议程设计、试点结果不理想的分级预案等操作细节，将在D3阶段结合业务部门意见征求的结果和董事长汇报的反馈，由摩根和当升HR团队在D4阶段联合落定。

#### 5. 阶段门禁与关键产出

- 《当升科技内部轮岗管理制度（草案）》（含跨界轮岗专章）
- 《轮岗激励衔接方案》
- 《人力资源各模块衔接接口说明》
- 修改规则：摩根完成初稿→提交人力资源部获取第一轮反馈→最多修改两轮→确认后进入D3

### 阶段三：Deliver 汇报确认（第6-7周）

1. **业务部门意见征求**：安排2-3场专题沟通会，就制度可操作性、对业务的影响、津贴标准合理性、推出节奏征求各业务部门负责人意见
2. **董事长汇报**：由人力资源部申请汇报时间，项目组共同参与。董事长一般一轮修改意见，项目组修改1-2轮，越快确认越好
3. **关键产出**：《当升科技内部轮岗管理制度（终审稿 · 董事长签字版）》

### 阶段四：Deploy 宣导固化（第8-10周）

1. **分层宣贯**：HR团队先掌握规则（联合项目组成员协助带教未参与项目的HR）→面向全员制度宣贯
2. **答疑辅导**：人员筛选、协议签署等属于客户内部工作，摩根作为配角配合答疑。可每周预留半天沟通问题
3. **HR团队带教收尾**：能力确认——确保当升HR团队具备独立运营轮岗制度的能力
4. **制度正式版定稿**：以董事长签批稿为蓝本，根据宣导反馈优化完善，输出面向全员发布的正式版
5. **后续陪跑选项确认**：详见第十一章后续合作路线图

## 五、项目成功的定义

制度发布只是起点。真正的成功是——有人真正动起来了，动了之后比不动之前更好，且这个过程不需要摩根一直在场。

| 时间节点 | 最低标准 | 目标标准 |
|---------|---------|---------|
| **项目结束时（10周）** | 轮岗制度经董事长签批正式发布；HR团队完成带教，可独立解读和操作制度 | 首轮试点人选已确定、试点路径已规划、试点启动时间表已排定 |
| **项目后3个月** | 首批试点（2条路径，6-10人）已启动；试点者月度回访正常进行 | 试点运行平稳，无退出案例；业务部门对试点给出正面反馈 |
| **项目后6-12个月** | 首轮试点完成复盘；制度根据试点反馈完成一轮优化迭代 | 第二批轮岗自驱启动（不再依赖摩根推动）；轮岗经历开始纳入晋升评议的实质参考 |

### 5.1 展望：轮岗制度运转起来之后

当制度成熟运转时，当升科技的人才流动将呈现这样的图景：

- 海外新基地需要人，不是临时"拉壮丁"，而是从后备池中按既定规则调配——人选已经在国内跨部门历练过，对海外挑战有基本准备
- 研发人员到生产一线轮岗回来后，设计方案时脑子里有产线的真实约束；财务人员从车间轮岗回来后，成本核算不再只是数字搬运
- 干部晋升评议时，"有没有跨部门经历"成为一个自然的问题，而不是一项需要特别解释的新政
- HR团队独立运转这套制度，摩根只在季度回访时了解进展、提供建议

这不是凭空想象。每一条都对应本方案中一个具体的设计模块——后备池机制、跨界轮岗路径、晋升绑定规则、HR带教计划。摩根的角色是：用8-10周帮当升搭好这套规则底座，陪第一波试点跑通，然后退到幕后。

## 六、推动制度落地的设计思路

轮岗制度要真正运转起来，需要在制度设计阶段就内置"如何被接受、如何被使用"的考量。以下为摩根在长期实践中形成的落地方法论——不是对当升科技内部情况的预判，而是制度设计本身应遵循的原则：

| 关键相关方 | 在落地中的角色 | 制度设计中的内置保障 |
| ----------- | -------------------- | ----------------------------------------------------------------------- |
| **高管层** | 制度的第一推动力——高管的态度决定全公司对轮岗的认真程度 | 制度设计中预留高管参与的方式（如率先体验短期轮岗、担任试点导师、在经营会上将轮岗与三年行动绑定表述），具体形式在D3阶段与董事长沟通确定 |
| **业务部门负责人** | 轮岗者的输出方与接收方——需要看到"人流动起来对业务有什么好处" | 后备补充机制确保关键岗位不出现真空；试点从小范围开始（首批6-10人），不冲击现有业务节奏；制度设计中明确"接收方对轮岗者考核权重最大"，让接收方有动力认真投入 |
| **潜在轮岗者** | 制度的使用者——需要确认"轮岗对我意味着什么" | 薪酬保障（保留原薪标准）、回程安置（双通道选择）、晋升绑定（轮岗=后备干部信号）、首轮试点自愿报名——用第一批人的真实经历建立口碑 |
| **HR团队** | 制度的长期运营者——需要具备独立运转这套制度的能力 | 联合项目组全程带教（"做中学"）；制度文档与决策依据完整移交；摩根后续答疑兜底；D4阶段交付推行工具包 |

核心原则：**制度设计不止于"写出一份文件"，而是让每个相关方在制度中找到"为什么我要配合"的答案。** 摩根的角色是与当升HR团队一起完成这个设计过程，确保制度发布时，各方的参与路径已经清晰。

#### HR推行工具包（D4阶段配套交付）

面向业务部门负责人的沟通话术、常见问题应答参考、试点经验内部传播方案等推行工具，将在D4宣导阶段由摩根项目组与当升HR团队基于试点实际进展和当升内部沟通文化共同设计，作为制度正式版的配套操作手册交付。

## 九、项目周期与投入

### 9.1 项目报价

| 阶段 | 覆盖内容 | 参考课时 | 阶段总价 |
|------|---------|:------:|:------:|
| **第一阶段** | D1 诊断与确认 + D2 方案撰写 + D3 汇报确认 + D4 宣导固化 | 参考96小时 | 417,600元 |
| **第二阶段** | 后续陪跑（季度回访 / 年度顾问 / 深度陪跑 / 长期伙伴，详见第十一章） | 按需确定 | 另行约定 |

**报价规则**：
- 团队综合费率4,350元/小时（含咨询实战专家+项目老师）
- 以上课时为基于项目范围的参考值，最终以双方确认的合同约定为准
- 第二阶段的后续陪跑为可选服务，客户可根据项目交付情况和实际需求选择合作深度
- 差旅费按实际发生（异地出差），不计入服务费率
- 以上服务费不含税，增值税税率6%另行计收
- 超出合同范围的额外服务（如一对一深度答疑等），按4,350元/小时单独计费

### 9.2 项目周期

整体**8-10周**，现场工作每周1-2天（关键节点可增至2-3天）。具体日期根据当升科技实际业务节奏灵活调整，以不影响客户主业推进为原则。

**建议启动时机**：轮岗制度作为"领导力建设三年行动"首年的基础设施，建议不晚于2026年6月启动，以确保：
- 8-10周后（约8-9月）制度发布，赶上**年底干部考评会**——轮岗制度与晋升评议挂钩的条款可在考评季前正式生效
- 制度发布后第6个月（约2027年2-3月）完成试点复盘，赶上**年度人才盘点周期**——试点结果可直接输入继任计划和后备干部池更新
- 第二批推广在2027年Q2启动，使**三年行动第二年（2027）有完整的运行数据**支撑制度优化迭代

> 时间敏感性提示：若项目启动延至2027年，则三年行动第一年的人员流动维度将没有制度化产出——建议尽早锁定启动时间窗。

## 十、项目风险与保障措施

| 风险点 | 应对措施 |
|-------|---------|
| 业务部门配合度不足 | 访谈前明确项目价值，争取董事长/总经理层面背书 |
| 制度设计与实际脱节 | 坚持"调研先行"，经人力资源部确认后方可进入方案阶段 |
| 轮岗影响业务连续性 | 设计中充分考虑后备补充机制，确保关键岗位有替代人选 |
| 员工对轮岗有抵触 | 宣导阶段充分沟通轮岗对个人发展的价值，配套激励措施到位 |
| 项目落地缺乏持续推力 | 培养客户内部HR专项小组，实现能力内化 |
| 客户需求反复变化 | 严格执行修改轮次上限（人力资源部≤2轮，董事长1-2轮） |

## 十一、后续合作路线图

本项目的定位是当升科技人力资源体系升级的"第一块基石"。轮岗与激励机制建设完成后，摩根建议沿以下路径持续深化：

| 推进线 | 第1阶段（本次项目） | 第2阶段 | 第3阶段 |
|--------|------------------|---------|---------|
| **主线：制度升级** | 轮岗制度 + 激励机制建设（8-10周） | → 全面薪酬体系升级（在本次接口基础上） | → 绩效管理体系优化 + 人才盘点与继任计划 |
| **并行线：能力培养** | 分层领导力培养（新晋经理项目，已有，持续12个月） | 中层领导力进阶 | 高层领导力发展 |

**后续陪跑选项**（主线+并行线之外的柔性支持）：

| 服务模式 | 内容 | 适合场景 |
|---------|------|---------|
| 季度回访 | 每季度一次现场/线上回访，评估运行情况 | 制度平稳运行期 |
| 年度顾问 | 定期回顾+制度升级辅导+应急咨询 | 业务持续扩张，制度需迭代 |
| 深度陪跑 | 摩根专家定期驻场，协助复杂问题处理 | 首批轮岗落地关键期 |
| 长期伙伴 | 参照摩根为日照钢铁提供4-5年持续服务的模式 | 融入全面人力资源体系升级 |

各阶段之间通过本次设计预留的接口自然衔接，每一阶段的产出是下一阶段的输入。客户可根据本次项目的交付情况和实际需求，选择后续合作的深度。

**跨年度预算规划参考**（帮助HR总做长期预算申请）：

| 年份 | 预计投入方向 | 参考投入结构 | 备注 |
|------|------------|------------|------|
| **2026年** | 轮岗制度与激励机制建设（本次项目） | 417,600元（详见9.1） | 一次性的制度建设投入，非年度固定成本 |
| **2027年** | 试点推广跟踪 + 制度首轮迭代 + 可选陪跑 | 依据续约模式（季度回访/年度顾问/深度陪跑），年度顾问/陪跑服务费约为制度建设项目费用的一个比例区间，具体以合同约定为准 | 投入大小取决于试点复盘后董事长确定的推广速度和所选服务模式 |
| **2028年** | 全面薪酬体系升级（第2阶段主线）+ 绩效管理与人才盘点（第3阶段主线） | 按新项目独立报价，在本次接口基础上启动，无需重复诊断 | 各阶段可独立签约，利用本次预留接口自然衔接 |

> 规划原则：制度建设的固定投入集中在2026年，2027-2028年的投入为运营型/迭代型（弹性空间大），且后续项目无需重复支付诊断费用。HR申请预算时可将"三年总投入"呈现为阶梯式下降的曲线——第一年搭底座，后两年添砖瓦。

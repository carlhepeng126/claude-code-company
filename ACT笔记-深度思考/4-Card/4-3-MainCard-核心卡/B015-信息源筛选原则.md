# B015-信息源筛选原则：从渠道判断到深度萃取

> **你不是在收集信息，你是在为自己的业务链采购原材料。信息源的价值不取决于它本身有多"好"，而取决于它在你业务链的哪个环节能直接使用。**

## 核心认知

### 萃取的前提不是"看"，是"筛"

在一次信息整理对话中，Carl 提到一篇 Get 笔记自动抓取的提示词资源网站笔记，说"想萃取出高质量的内容，因此信息来源就很重要"。这个直觉是对的——**萃取的质量上限，由信息源的质量决定。**

但如果只是逐条判断"这个来源好不好"，效率太低。真正需要的是：**一套前置筛选框架，让你在摄入信息的瞬间就能判断它值不值得进入下一步萃取。**

### 业务链决定信息取向

判断信息源价值的唯一标准是：**它在你业务链的哪个环节能直接落地？**

| 你的业务链环节 | 需要什么信息 | 用来干嘛 |
|---|---|---|
| **咨询/诊断** | 客户行业痛点、外派失败率、培训预算基准 | 提案里甩数据，"你们的问题不是个例" |
| **课程设计** | 新方法论、竞品课程、最佳实践案例 | 对标找差异，少走弯路 |
| **项目交付** | 新工具/新研究（AI培训、测评、e-learning） | 增强课程体验 |
| **拓客** | 谁在出海、进了哪个市场、谁在投培训预算 | 提前锁定客户 |

### 信息源四象限（按行动价值分层）

不是所有信息源都需要同样的萃取深度。按"是否需要深入萃取"和"是否可直接行动"两个维度分层：

```
              需要深入萃取
                   │
      ┌────────────┼────────────┐
      │  ⚠️ 二类源     │  ⭐ 一类源   │
      │  需要验证     │  可直接萃取   │
      │  行业报告     │  真实企业案例  │
      │  方法论文章   │  可追溯的数据  │
      │  AI 生成汇总  │  一手调研发现  │
      │              │              │
 否───┼────────────┼────────────┤─── 可直接行动
      │              │              │
      │  ❌ 三类源     │  📌 四类源   │
      │  直接过滤     │  收藏备用     │
      │  无来源断言   │  工具列表     │
      │  营销软文     │  渠道索引     │
      │  个人观点     │  资源导航     │
      └────────────┴────────────┘
              不需要深入萃取
```

**分类规则：**

| 类别 | 判断标准 | 处理方式 | 案例 |
|---|---|---|---|
| ⭐ 一类源 | 来源可追溯 + 含具体企业/时间/数据 + 与业务链直接相关 | 立即萃取 → 核心卡 | 培训杂志2025年4月刊的文章（中广核案例） |
| ⚠️ 二类源 | 方法论/框架有价值 + 但缺具体可追溯数据 | 先验证 → 再决定是否萃取 | 一个讲"7步培训法"的文章但没有企业数据 |
| 📌 四类源 | 工具/渠道/资源索引类 | 保存地址即可，不做深度萃取 | 那个提示词资源网站 |
| ❌ 三类源 | 无法追溯 + 无具体数据 + 纯观点 | 直接丢弃 | "我觉得培训应该..."类内容 |

## 萃取流程（Carl + Claude 共创成果）

以下是我们在这张卡片诞生的过程中，逐步梳理出的信息源萃取 SOP：

### Step 1：来源类型定位

拿到一条信息时，先问：**这是什么类型的信息源？**
- 链接型笔记（Get 笔记自动抓取）→ 渠道本身不是内容，需再判断价值
- 案例型素材（培训杂志文章、研究报告）→ 最有萃取价值的一类
- 方法论型（框架/模型/SOP）→ 有价值的二类源，需和已有体系对照
- 数据型（报告数据、调研统计）→ 提案武器，可直接引用但需追溯

### Step 2：业务链匹配

问：**这条信息落在我的哪条业务链上？**
- 咨询/诊断？ → 看有没有可甩的行业数据
- 课程设计？ → 看有没有新方法论或竞品对标
- 项目交付？ → 看有没有新工具或新研究
- 拓客？ → 看有没有客户信号或市场动态

不落任何一条链 → 直接丢回收件箱或归档。

### Step 3：质量验证（Intel Verification）

问：**这条信息是否经得起验证？**
- 来源可追溯到具体媒体/官网/报告？
- 含明确的企业名+时间+规模+数据？
- 是否排除 AI 编造或通用趋势描述？
- 标注可用场景（提案/课程设计/项目交付/拓客）

不满足以上 → 标注⚠️，需先查证再萃取。

### Step 4：萃取深度决策

根据 Step 1-3 的结果，决定处理方式：

| 条件 | 处理 | 最终出口 |
|---|---|---|
| 一类源 + 高业务价值 | 深入萃取 → 核心卡 | 4-3-MainCard |
| 一类源 + 中等价值 | BibCard 记录，关键点提炼 | 4-2-BibCard |
| 二类源但方法论新 | 先验证来源，再决定是否萃取 | 验证期间留在收件箱 |
| 四类源（工具/索引） | 收藏地址，不做深度萃取 | 6-storage/工具索引 |
| 三类源 | 直接丢弃 | 删除 |

### Step 5：沉淀入库

萃取完成后：
1. 信息提炼 → 核心卡（B编号）或阅读卡写入
2. 触发行动 → 关联到具体的 Action 项目
3. 索引聚合 → 在 IndexCard 中建立链接
4. 存档凭证 → 原始素材可归档到 6-storage

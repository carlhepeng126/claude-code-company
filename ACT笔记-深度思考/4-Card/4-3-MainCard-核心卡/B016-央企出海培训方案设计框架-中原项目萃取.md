# B016-央企出海培训方案设计框架：中原项目萃取

> 从中原对外工程项目（3天/11.4万/4位老师）萃取的通用方案设计方法论。核心思想：**用"短板覆盖矩阵"替代"课程列表"，让客户看到的不是课程堆叠，而是精准对症的解决方案。**

## 核心认知

中原项目的方案设计本质上是一套 **"诊断→映射→装配"** 的三步框架。它不是从"我们有什么课"出发，而是从"客户缺什么能力"出发。

### 五条设计原则（可复用）

从中原方案的设计思路中可提炼出五条通用原则：

1. **聚焦专业能力** — 全部模块围绕客户的核心业务场景设计，不塞通用课程凑天数
2. **短板全覆盖** — 用测评数据驱动设计，每个短板有对应模块，关键短板由多位老师双重强化
3. **实战密度高** — 每天至少3个实战环节，总实战时长超过12小时（3天计）
4. **晚间结构化分享** — 每晚1.5小时用于学员路演+互评+行动承诺，确保"学完就能用"
5. **工具输出** — 学员带走可立即上手的模板工具，培训结束不等于价值结束

### 三天结构模式（可复用）

```
第一天【战略层】  宏观视野 → 战略选择 → 路径规划 → 竞争定位
第二天【执行层】  情报采集 → 项目策划 → 投标合同 → 全过程管理
第三天【落地层】  区域市场 → 内部协调 → 综合实战演练
```

**递进逻辑**：战略共识→工具方法→区域落地。这个结构对不同行业的出海培训都适用，只需替换案例和区域内容。

## 关键要点

### 短板-模块映射矩阵（方案设计核心工具）

中原方案的核心设计方法：先列客户能力短板，再逐条映射到模块＋师资＋训练目标。

| 步骤 | 做什么 | 产出物 |
|------|--------|--------|
| 1. 获取能力基线 | 客户测评/访谈数据 | 短板清单（如"信息搜集弱、战略规划弱"） |
| 2. 短板→模块映射 | 每条短板标注对应模块 | 模块-短板映射表 |
| 3. 关键短板双重强化 | 核心短板安排2位以上老师从不同角度覆盖 | 师资-短板匹配矩阵 |
| 4. 设计实战验证环节 | 每个模块结束后设计演练验证学员是否掌握 | 实战环节清单 |

### 师资组合策略

中原项目用了4位老师各自负责擅长模块，这种"拼盘式"师资组合在央企项目中非常有效：

| 角色 | 定位 | 课酬带 | 典型人选来源 |
|------|------|--------|------------|
| 宏观嘉宾 | 政策高度+行业视野（半天） | 高 | 退休央企高管/教授 |
| 方法论老师 | 核心模块主讲（1-2天） | 中高 | 专业培训机构签约讲师 |
| 实战老师 | 案例+工具+演练（半天-1天） | 中 | 有海外一线经验的在职/退休专家 |
| 区域专家 | 特定区域市场（1天） | 中 | 当地有实战经验的顾问 |

### 区域可替换框架

中原方案的关键创新：核心教学框架不变，区域内容可替换。这对多区域业务的央企尤其有价值——一次开发，多次复用。

```
固定框架（60%）：出海战略、项目策划、投标合同、内部协调
可变内容（40%）：区域案例、商业文化、法律环境、本地化要求
```

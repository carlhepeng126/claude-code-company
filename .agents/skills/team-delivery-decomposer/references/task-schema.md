# 任务字段与原子化标准

## 任务命名规则

任务名必须符合：

`动词 + 对象 + 产出物`

合格示例：
- 收集 57 名学员基础信息表
- 核对讲师合同付款节点
- 生成开营仪式流程单
- 发送客户协助事项确认邮件

不合格示例：
- 跟进学员
- 完善资料
- 推进商务
- 做好教学服务

## 原子任务判定

一个任务只有同时满足以下条件，才算不能继续拆：

| 标准 | 判定问题 |
|---|---|
| 单一动作 | 这件事是否只需要一个主要动作完成？ |
| 单一责任 | 是否可以明确一个 R 和一个 A？ |
| 单一产出 | 是否只有一个主要交付物？ |
| 可验收 | 是否能用客观标准判断完成/未完成？ |
| 可排期 | 是否能放进具体日期或触发节点？ |

任一答案为“否”，继续拆。

## 难度分级

| 难度 | 名称 | 判定标准 | 适合负责人 |
|---|---|---|---|
| D1 | 照单执行 | 有模板、有明确输入、结果容易检查 | 新人/行政/助教 |
| D2 | 模板改造 | 有模板，但需按项目替换信息 | 教学服务/商务/行政 |
| D3 | 专业判断 | 需要判断内容质量、教学逻辑或客户语境 | 教学负责人/项目经理 |
| D4 | 多方协同 | 涉及客户、讲师、内部多角色排期或取舍 | 项目负责人 |
| D5 | 高风险决策 | 影响方案边界、报价、合同、客户高层满意度 | 项目总负责人/合伙人 |

## 优先级分级

| 优先级 | 含义 | 判定标准 |
|---|---|---|
| P0 | 阻断项 | 不完成会导致开营、交付、回款无法进行 |
| P1 | 关键项 | 不完成会明显影响质量或客户体验 |
| P2 | 标准项 | 应按计划完成，但短期不阻断 |
| P3 | 优化项 | 有助于体验/沉淀，可后置 |

## 标准任务表字段

| 字段 | 说明 |
|---|---|
| WBS | 编号，如 2.3.1 |
| 阶段 | 启动前/开营前/交付中/结项/复盘 |
| 工作流 | 教学设计/学员管理/商务合同等 |
| 任务 | 工作包标题 |
| 原子动作 | 具体做什么 |
| 输入 | 开始前需要什么材料/信息 |
| 产出物 | 完成后留下什么文件/记录/表单 |
| 验收标准 | 什么叫完成 |
| 难度 | D1-D5 |
| 优先级 | P0-P3 |
| R | Responsible，实际执行者 |
| A | Accountable，最终负责者 |
| C | Consulted，需要咨询的人 |
| I | Informed，需要同步的人 |
| 截止/触发 | 日期或前置事件 |

## “建立学员档案”最低颗粒度模板

| WBS | 原子动作 | 需要的信息 | 产出物 | 验收标准 | 默认 R | 默认 A |
|---|---|---|---|---|---|---|
| X.1 | 创建项目学员档案表 | 项目名、期数、班级规则 | 学员档案表 | 文件名和字段完整 | 教学服务 | 项目负责人 |
| X.2 | 收集学员基础信息 | 姓名、手机号、邮箱、部门、基地、岗位 | 基础信息 sheet | 人数与客户名单一致 | 客户 HR | 客户项目负责人 |
| X.3 | 补充学习相关字段 | 管理年限、学习目标、痛点、上级姓名 | 学习画像 sheet | 字段缺失率低于 10% | 教学服务 | 教学负责人 |
| X.4 | 标注分班/分组信息 | 班级、组别、组长、基地分布 | 分组 sheet | 每组人数均衡且无重复 | 教学服务 | 项目负责人 |
| X.5 | 建立风险标签 | 缺勤风险、基础薄弱、高潜、重点关注 | 风险标签列 | 标签有依据，不凭感觉 | 教学负责人 | 项目负责人 |
| X.6 | 确认隐私权限 | 可见范围、共享对象、脱敏规则 | 权限说明 | 非项目成员不可见敏感信息 | 行政/IT | 项目负责人 |
| X.7 | 每次交付后更新记录 | 出勤、作业、表现、反馈、跟进行动 | 学员过程记录 | 24 小时内更新 | 教学服务 | 教学负责人 |

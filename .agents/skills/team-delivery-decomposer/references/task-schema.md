# 任务字段与原子化标准

## 任务命名规则

任务名必须符合：

`动词 + 对象 + 产出物`

合格示例：
- 收集 57 名学员基础信息表
- 核对讲师合同付款节点
- 生成开营仪式流程单
- 发送客户协助事项确认邮件

不合格示例：
- 跟进学员
- 完善资料
- 推进商务
- 做好教学服务

## 原子任务判定

一个任务只有同时满足以下条件，才算不能继续拆：

| 标准 | 判定问题 |
|---|---|
| 单一动作 | 这件事是否只需要一个主要动作完成？ |
| 单一责任 | 是否可以明确一个 R 和一个 A？ |
| 单一产出 | 是否只有一个主要交付物？ |
| 可验收 | 是否能用客观标准判断完成/未完成？ |
| 可排期 | 是否能放进具体日期或触发节点？ |

任一答案为“否”，继续拆。

## 难度分级

| 难度 | 名称 | 判定标准 | 适合负责人 |
|---|---|---|---|
| G | 高级 | 需要战略判断、资源调配、风险决策，涉及合同金额或客户关系 | 项目组长/项目总监 |
| Z | 中级 | 需要经验判断和协调，不能完全按清单执行 | 项目经理/主管 |
| J | 初级 | 按清单执行即可，有明确 SOP，新人培训后可完成 | 班主任/行政/新人 |

## 优先级分级

| 优先级 | 含义 | 判定标准 |
|---|---|---|
| P0 | 阻断项 | 不完成会导致开营、交付、回款无法进行 |
| P1 | 关键项 | 不完成会明显影响质量或客户体验 |
| P2 | 标准项 | 应按计划完成，但短期不阻断 |
| P3 | 优化项 | 有助于体验/沉淀，可后置 |

## 标准任务表字段

| 字段 | 说明 |
|---|---|
| 模块 | 项目运营/课程交付/教学服务/线上服务/行政支持/异常处理等 |
| 阶段 | 启动期/课前准备/集训中/课后跟进/结项期等 |
| 任务 | 原子级任务名称 |
| 具体操作 | 用“步骤1→步骤2→步骤3”写清楚，低门槛可执行 |
| 难度 | G/Z/J，可附 🔴/🟡/🟢 |
| R-负责执行 | 实际做事的人 |
| A-最终负责 | 唯一结果负责人 |
| C-咨询 | 提供意见或输入的人 |
| I-知会 | 需要知道进展的人 |
| 工时(小时) | 预估投入工时 |
| 频率 | 一次性/每阶段/每日/每周/每月/按需 |
| 截止时间 | 绝对日期或相对节点 |

Excel 表头必须优先使用：

`模块 | 阶段 | 任务 | 具体操作 | 难度 | R-负责执行 | A-最终负责 | C-咨询 | I-知会 | 工时(小时) | 频率 | 截止时间`

## “建立学员档案”最低颗粒度模板

| WBS | 原子动作 | 需要的信息 | 产出物 | 验收标准 | 默认 R | 默认 A |
|---|---|---|---|---|---|---|
| X.1 | 创建项目学员档案表 | 项目名、期数、班级规则 | 学员档案表 | 文件名和字段完整 | 教学服务 | 项目负责人 |
| X.2 | 收集学员基础信息 | 姓名、手机号、邮箱、部门、基地、岗位 | 基础信息 sheet | 人数与客户名单一致 | 客户 HR | 客户项目负责人 |
| X.3 | 补充学习相关字段 | 管理年限、学习目标、痛点、上级姓名 | 学习画像 sheet | 字段缺失率低于 10% | 教学服务 | 教学负责人 |
| X.4 | 标注分班/分组信息 | 班级、组别、组长、基地分布 | 分组 sheet | 每组人数均衡且无重复 | 教学服务 | 项目负责人 |
| X.5 | 建立风险标签 | 缺勤风险、基础薄弱、高潜、重点关注 | 风险标签列 | 标签有依据，不凭感觉 | 教学负责人 | 项目负责人 |
| X.6 | 确认隐私权限 | 可见范围、共享对象、脱敏规则 | 权限说明 | 非项目成员不可见敏感信息 | 行政/IT | 项目负责人 |
| X.7 | 每次交付后更新记录 | 出勤、作业、表现、反馈、跟进行动 | 学员过程记录 | 24 小时内更新 | 教学服务 | 教学负责人 |

## 输出规模标准

| 项目复杂度 | 任务行数 |
|---|---:|
| 简单单次培训 | 50-70 |
| 标准企业培训 | 80-120 |
| 12 个月持续服务/OMO | 120-180 |
| 咨询+培训混合项目 | 150+ |

如果输出少于标准，必须说明是用户要求只拆局部模块，否则继续补足。
